# WeMusic · 本地个人音乐网站

一个仅供本地部署、个人使用的音乐网站。借助 QQ 音乐的歌单 / 歌手 / 专辑数据进行检索与管理，
播放时自动从 **Bilibili** 匹配音质最佳、非现场版的视频资源进行嵌入播放，实现免费收听体验。

> 仅供个人学习与本地使用，请勿用于商业用途。所有音乐版权归原平台与版权方所有。

---

## 功能特性

### 用户系统
- 注册 / 登录，JWT + bcrypt（cost=12）鉴权，适用于单用户保存个人数据。
- 注册完成后可在 `.env` 设置 `ALLOW_REGISTER=false` 关闭注册入口。
- 用户头像上传（自动压缩到 300px，base64 存储于 SQLite），顶栏实时显示。

### 歌单管理
- 粘贴 QQ 音乐歌单链接，自动解析并批量导入歌曲。
- 输入歌手名，一键批量添加该歌手的全部歌曲；也可按专辑整张添加。
- 新建 / 重命名（双击歌单名）/ 删除（带确认，显示歌单名与歌曲数）。
- 歌单内**拖拽排序**，持久化到数据库。
- **导出 / 导入歌单**（JSON 格式，含已缓存的 Bilibili 资源，便于备份与设备迁移）。
- 歌单详情页顶部显示**专辑封面拼图**（取前 4 张专辑封面拼成 2×2）。

### 发现音乐
- **为你推荐**：多维度兴趣模型推荐。
  - 信号体系：❤️ 红心（+3.0）、加入歌单（+2.5）、完播率 ≥ 80%（+1.5）、重复播放（+0.5/次）、时间衰减（90天线性衰减）
  - 歌手权重修正：合唱中非主唱歌手权重 ×0.4，避免误判
  - 三维推荐来源：
    1. **歌手深度挖掘** — 权重最高 3 个种子歌手搜歌
    2. **风格扩散** — 高分歌曲名 +「翻唱」关键词跨歌手搜索
    3. **热歌榜混入** — 当前热歌榜 20 首，保持新鲜度
  - 冷启动：无历史时自动展示热歌榜
  - 推荐页展示偏好歌手标签和推荐依据
- **热歌榜 / 新歌榜 / 流行指数 / 识曲榜**：实时拉取 QQ 音乐排行榜，带排名和封面，可直接播放。

### 歌曲标记
- 搜索结果 / 歌手页 / 专辑页中，已加入过任意歌单的歌曲在**时长左侧**显示 🔖 标记。
- 点 ＋ 打开歌单选择器时，已完整收录的歌单显示绿色 ✓，部分收录显示橙色 •。

### 播放
- 点击歌曲自动调用 Bilibili 搜索接口（含 WBI 签名 + buvid 激活）匹配对应资源。
- **歌手准确性优先**：优先保留目标歌手版本，过滤其他歌手演唱的同名曲。
- **播放质量排序**：过滤伴奏 / 纯音乐 / 消音；提升无损 / Hi-Res / 高音质权重。
- 自动过滤现场 / 演唱会 / Live 版本。
- 支持手动**换源**（底部播放器 + 视频面板均可触发）。
- bvid 全局缓存：匹配结果缓存到数据库，下次播放秒开。
- 风控自动重试：遇到 Bilibili 偶发风控，自动刷新 buvid 身份重试 3 次。

### 播放控制
- **视频浮窗**：可拖动、全屏、收起 / 关闭；展开 / 收起状态持久记忆。
- **播放模式**：🔁 列表循环 / 🔂 单曲循环 / 🔀 随机播放，一键切换并记忆。
- **自动连播**：按歌曲时长定时器驱动，播完自动跳下一首。
- **后台播放**：切到其他标签后持续播放，计时器切歌后新歌自动接续，无需切回前台。实现方式：切后台时销毁 iframe、用 `<audio>` 代理 B 站音频流接管（预缓冲降低切换延迟）；回前台时 iframe 带时间戳重建，交叉过渡无空档。
- **音量控制**：底部播放栏 🔊 滑块控制后台音量（前台 iframe 音量在 B 站播放器内调节）。
- **播放队列 / 历史 合并抽屉**：同框切换，支持跳播、移除、清空。
- 恢复上次播放：刷新或重启后底栏显示上次的歌曲与队列，点 ▶ 继续。
- **播放器直接操作**：底栏 ❤ 喜欢 + ··· 菜单（添加到歌单 / 复制链接）。
- 弹幕开关：视频面板标题栏切换（重载 iframe，状态持久化）。

### 播放列表工具
- **播放全部** / **随机播放** 按钮。
- **列表内筛选**：输入关键词即时过滤，不影响播放索引。

### 搜索辅助
- 搜索框聚焦自动弹出历史记录（最近 12 条），输入时按关键词建议。

### 歌曲右键菜单
右键单击任意歌曲行：▶ 播放 / ＋ 添加到歌单 / 🔗 复制 Bilibili 链接 / ✕ 从歌单移除。

### 歌词
- 点击底栏「词」或播放器封面进入**全屏歌词页**（毛玻璃背景 + 旋转封面 + LRC 同步滚动）。
- 网易云音乐**多策略搜索**（歌名+歌手 → 去括号 → 纯歌名 → 回退），匹配质量优先。
- **歌词换源**：点「⤢ 歌词」查看最多 8 个候选版本，选择后自动切换；偏好按 `song_mid` 持久化到 localStorage，下次自动使用上次选择的源。
- 歌词页底部迷你播放控制栏 + ❤ 喜欢 + ＋ 添加到歌单。

### 数据统计
- 总播放次数 / 时长 / 不重复歌曲 / 听歌天数。
- 30 天播放趋势柱状图 + 听歌时段热力图 + 最爱歌手条形图 + 最常播放 Top 10（可直接播放）。
- 播放时长真实统计（切歌时按 elapsed 上报 + 播完时按 totalDur 上报）。

### 界面
- 深色 / 浅色 / **跟随系统** 三种主题。
- 侧边栏 Logo 居中，宽度可拖拽调整（180–440 px）。
- 定时停止播放（15 / 30 / 60 分钟 / 播完当前曲）。
- 响应式设计，适配桌面与移动端（底部 Tab Bar + 简化歌曲行）。
- PWA 支持：可安装到桌面，离线可访问（Service Worker 三级缓存策略）。

### 键盘快捷键

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Node.js 20 + Express + better-sqlite3 + JWT |
| 前端 | ES Modules + Vite 打包（12 模块） |
| 音乐数据 | QQ 音乐公开 Web 接口（搜索 / 歌手 / 专辑 / 歌单 / 榜单） |
| 播放 | Bilibili 搜索 API（WBI 签名 + buvid 激活）+ 官方 iframe 播放器；后台切换时通过服务端音频流代理（`/api/play/stream`）接管播放 |
| 歌词 | 网易云音乐公开接口（多层搜索 + LRC 解析） |
| 存储 | SQLite + localStorage |

## 快速开始

```bash
npm install
npm run build          # 首次运行前必须构建一次前端
cp .env.example .env   # 按需修改 JWT_SECRET 和 PORT
npm start
```

开发模式：
```bash
npm run build:watch    # 终端 1：前端自动构建
npm run dev            # 终端 2：服务端热重载
```

启动后访问 `http://localhost:5174`，首次使用点「立即注册」。

## 使用流程

1. **发现音乐**：默认进入发现页，浏览推荐或榜单。
2. **搜索**：搜索框输入歌手 / 歌曲名。
3. **添加歌单**：点 ＋ → 选择歌单或新建。
4. **导入歌单**：左下角 ⇪ → QQ 链接或 JSON 文件。
5. **备份**：歌单详情页「⤓ 导出歌单」。

## 推荐算法

### 信号权重

| 信号 | 权重 | 说明 |
|------|------|------|
| ❤️ 红心 | +3.0 | 最强正信号 |
| 加入歌单 | +2.5 | 主动收藏 |
| 完播率 ≥ 80% | +1.5 | 真正听完 |
| 重复播放 | +0.5/次（上限 +3） | 越听越喜欢 |
| 完播率 20–80% | +0.3 | 一般兴趣 |
| 完播率 < 20% | −0.3 | 可能不喜欢 |
| 时间衰减 | 近期 ×1.0 → 90天后 ×0.5 | 近期行为优先 |

### 歌手权重修正

合唱"邓紫棋 / 艾热AIR"中，非主唱权重 ×0.4。仅独立权重 ≥ 2.0 的歌手进入种子池。

### 三维推荐来源

1. **歌手深度挖掘** — 权重最高 3 个歌手搜歌
2. **风格扩散** — 高分歌曲名 +「翻唱」关键词搜索，发现同风格跨歌手版本
3. **热歌榜混入** — 当前热歌榜 20 首，保持新鲜度

候选评分后按正分分桶（高 ≥1.5、中 0~1.5），负分不推荐（已被跳过的歌）。

| 措施 | 实现 |
|------|------|
| 点击劫持 & MIME 嗅探 & XSS | 响应头防护 |

| 错误脱敏 | 500 不暴露堆栈 |

## 已知限制

### 跨域 iframe 导致的不可修复问题

| 问题 | 原因 |
|------|------|
| 进度条无法同步视频 | iframe 跨域 → 无法读写 video.currentTime |
| 画质低 | B 站服务端识别嵌入播放，主动降级码率 |
| iframe 内置弹幕按钮无效 | 嵌入模式沙箱拦截交互事件 |
| 无法登录 B 站账号 | iframe 隔离，不传递登录态 + 嵌入模式无视登录 |
| 前台 iframe 音量无法通过 WeMusic 控制 | iframe 跨域隔离，无法读写 B 站播放器音量，需在视频内手动调节 |

### 设计限制

| 问题 | 说明 |
|------|------|
| 自动连播靠定时器非播放事件 | 视频 MV 时长不一致时可能提前/延迟切歌 |
| app.js 构建产物约 68KB（gzip 后 20KB） | 12 模块 Vite 打包，性能可接受 |
| 单用户架构 | SQLite + 单机部署，不支持多用户并发 |
