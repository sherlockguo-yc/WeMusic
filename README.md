# WeMusic · 本地个人音乐网站

一个仅供个人本地学习使用的自托管（self-hosted）音乐网站。借助 QQ 音乐的歌单 / 歌手 / 专辑数据进行检索与管理，
播放时自动从 **Bilibili** 匹配音质最佳、非现场版的视频资源进行嵌入播放，实现免费收听体验。

> 仅供个人本地学习使用，请勿用于商业用途。所有音乐版权归原平台与版权方所有。

## 功能特性

### 用户系统
- 注册 / 登录，JWT + bcrypt（cost=12）鉴权。
- 登录页浅色风格，卡片式布局 + 专辑 Logo。
- 注册完成后可在 `.env` 设置 `ALLOW_REGISTER=false` 关闭注册入口。
- 用户头像：点击头像即可上传（自动压缩到 300px，base64 存储），顶栏实时显示。

### 歌单管理
- 粘贴 QQ 音乐歌单链接，自动解析并批量导入歌曲。
- 输入歌手名，一键批量添加该歌手的全部歌曲；也可按专辑整张添加。
- 右键歌单 →「编辑歌单」→ 一个弹窗同时编辑歌单名和简介。
- 右键歌单 →「删除歌单」（带确认）。
- 歌单内拖拽排序，持久化到数据库。
- 导出 / 导入歌单（JSON 格式，含已缓存的 Bilibili 资源）。
- 歌单详情页：表头（# / 歌名 / 歌手 / 专辑 / 时长）+ 简介展示。

### 专辑收藏
- 歌手页 → 专辑详情页 →「收藏专辑」→ 永久保存在侧边栏「我的专辑」。
- 专辑详情页：左侧 200px 封面 + 右侧流派 / 发行公司 / 简介；曲目旁按钮组（播放全部 / 随机播放 / 添加到歌单 / 收藏，均带图标）。
- 我的专辑网格：每张卡片封面右上角 hover 悬浮播放按钮，流派绿底药丸标签，厂商/日期独立 meta 行（带图标），hover 上浮 + 阴影。

### 发现音乐
- **为你推荐**：多维度兴趣模型推荐。
  - 信号体系：❤️ 红心（+3.0）、加入歌单（+2.5）、完播率 ≥ 80%（+1.5）、重复播放（+0.5/次）、时间衰减（90天线性衰减）
  - 歌手权重修正：合唱中非主唱歌手权重 ×0.4，避免误判
  - 三维推荐来源：
    1. **歌手深度挖掘** — 权重最高 3 个种子歌手搜歌
    2. **风格扩散** — 高分歌曲名 +「翻唱」关键词跨歌手搜索
    3. **热歌榜混入** — 当前热歌榜 20 首，保持新鲜度
  - 冷启动：无历史时自动展示热歌榜
  - 推荐页展示偏好歌手标签和推荐依据
- **热歌榜 / 新歌榜 / 流行指数**：实时拉取 QQ 音乐排行榜。

### 歌曲标记
- 已加入过任意歌单的歌曲在时长左侧显示标记。
- 点 ＋ 打开歌单选择器时，已完整收录的歌单显示绿色勾，部分收录显示橙色点。

### 播放
- 点击歌曲自动调用 Bilibili 搜索接口（含 WBI 签名 + buvid 激活）匹配对应资源。
- 搜索优化：
  - **多 query 并行**：5 组关键词（歌名+歌手、MV、官方版等）同时发起，分两波（Wave 1 首页 + Wave 2 次页），最快 ~500ms 完成匹配。
  - **歌名片段过滤**：要求候选视频标题包含歌名至少一个 ≥2 字符片段（中文拆 2-gram，拉丁按单词），排除完全无关视频。
  - **官方 MV 最高优先级**：「官方+MV」+42 分、官版 +28 分、MV +20 分、UP 主完全匹配歌手名 +18 分。
  - **播放量分级加权**：≥1000 万 +55、≥100 万 +40、≥10 万 +25、≥1 万 +12。
- **自动过滤**：伴奏 / 纯音乐 / 消音 + 现场 / 演唱会 / Live 版本。
- 候选列表标记「当前」正在播放的视频源（绿色高亮）。
- 支持**屏蔽视频源 / 歌词源**（✕ 按钮 → 持久化黑名单 `blocked_sources` 表，以后不再出现）。
- 支持手动**换源**（底部播放器 + 视频面板均可触发）。
- bvid 全局缓存：匹配结果缓存到数据库，下次播放秒开。
- 风控自动重试：遇到 Bilibili 偶发风控，自动刷新 buvid 身份重试 3 次。
- 候选数提升至 25，搜索池上限 80；歌名含括号（如 `(笑面人)`）自动提取中文独立搜索。

### 播放控制
- **视频浮窗**：可拖动、全屏、收起 / 关闭；展开 / 收起状态持久记忆。
- **播放模式**：简约线框 SVG 图标，列表循环 / 单曲循环 / 随机播放，一键切换并记忆。
- **自动连播**：按歌曲时长定时器驱动，播完自动跳下一首。
- **后台播放**：切到其他标签后持续播放，计时器切歌后新歌自动接续，无需切回前台。切后台销毁 iframe、用 `<audio>` 代理音频流接管；回前台 iframe 带时间戳重建，交叉过渡无空档。
- **换源持久化**：手动换源后缓存到数据库（支持无 `song_mid` 的歌曲，用歌名+歌手作备用 key），下次播放自动使用上次选择。
- **音量控制**：底部播放栏竖向弹出滑块，控制后台音量。
- **播放队列 / 历史 合并抽屉**：同框切换，支持跳播、移除、清空，点击抽屉外自动关闭。
- 恢复上次播放：刷新或重启后底栏显示上次的歌曲与队列，点 ▶ 继续。
- **播放器直接操作**：底栏 ❤ 喜欢 + 💔 不喜欢 + ··· 菜单（添加到歌单 / 复制链接）。

### 播放列表工具
- **播放全部** / **随机播放** 按钮。
- **列表内筛选**：输入关键词即时过滤，不影响播放索引。

### 搜索辅助
- 搜索框聚焦自动弹出历史记录（最近 12 条），支持键盘 ↑↓ 导航 + Enter 选择。输入时末尾显示 × 清除按钮。
- 歌曲列表统一列头：# / 歌名 / 歌手 / 专辑 / 时长 / 操作，表头与数据行严格对齐（固定列宽 Grid 模板）。
- 搜索结果命中歌手时显示歌手卡片（头像 + 歌曲 / 专辑总数 + 进入歌手页按钮）。
- 搜索结果每首歌右侧喜欢 + 不喜欢按钮（与播放器底部同款 SVG，喜欢后红色高亮）。
- 专辑数量从 QQ 音乐 API 的 `total` 字段实时获取，确保准确性。

### 歌曲右键菜单
右键单击任意歌曲行：▶ 播放 / ＋ 添加到歌单 / 🔗 复制 Bilibili 链接 / ✕ 从歌单移除。

### 歌词
- 点击底栏「词」或播放器封面进入**全屏歌词页**（毛玻璃背景 + 旋转封面 + LRC 同步滚动）。
- 打开 / 关闭动画：从封面位置缩放展开 / 缩回（`transform: scale` + `transform-origin`），附带圆角过渡。
- 歌词页底部迷你播放控制栏 + ❤ 喜欢 + ＋ 添加到歌单 + **歌曲背景**卡片（专辑简介 / 风格 / 唱片公司）。
- 网易云音乐**多策略搜索**（歌名+歌手 → 去括号 → 纯歌名），并行拉取提升速度。
- **歌词换源**：并行拉取 top 20 候选歌词并预过滤空内容，返回最多 12 个有效候选，避免「暂无歌词」无效源。
- 候选列表标记「当前」使用的歌词源（绿色高亮）+ 候选带序号。
- 歌词内存缓存：同名同歌手 24h 内复用（上限 200 首）。
- 换源偏好按 `song_mid` 持久化到 localStorage，下次自动使用上次选择的源。
- 歌曲背景卡片：调用 QQ 音乐专辑详情接口，展示专辑描述、流派、语言、发行日期等元信息。

### 数据统计 · 本周 / 本月听歌报告
- 总播放次数 / 时长 / 不重复歌曲 / 听歌天数，含对上期涨跌趋势。
- 报告 Tab 切换：本周 / 本月，数据独立查询懒加载。
- 自然周（周一~周日）/ 自然月（1日~月末）。
- **历史周期浏览**：← → 箭头切换上周/上月，海报自动对应所选周期，不再被"周日晚上的窗口焦虑"困扰。
- 报告卡片：个性化洞察小结 + 音乐人设标签 + Top 歌曲 / 歌手 + 完播率堆叠色带 + 活跃时段时钟 + 趋势迷你柱状图。
- 分享海报两种方案：本地 html2canvas 简约版 + 服务端 Puppeteer 精工版（封面模糊背景 + 拼贴墙 + 缩略图），4 种主题可选。

- 搜索栏清除按钮（输入内容时自动显示 ×）。

| 层级 | 技术 |
|------|------|

## 快速开始

### Docker（推荐）

```bash
docker pull sherlockguo/wemusic
docker run -d -p 5174:5174 -v ~/wemusic-data:/app/data --name wemusic sherlockguo/wemusic
```

访问 `http://localhost:5174`，数据持久化在 `~/wemusic-data`。

### 手机部署（Android + Termux · 一键脚本）

> 手机自己跑服务，自己访问自己。不依赖电脑、不上传数据。

**① 安装 Termux**

用手机浏览器打开 [GitHub Releases](https://github.com/termux/termux-app/releases/latest)，下拉找到 **Assets**，只下载这一个：

```
termux-app_v0.118.3+github-debug_universal.apk  ← 下这个就对了
```

（选 `universal`，所有安卓手机通用，不用管 CPU 型号）

> 不要用 Google Play 版本，已停更。

**② 复制粘贴下面这一行**（打开 Termux 后长按粘贴，回车）：

```bash
pkg install wget -y && wget -qO- https://raw.githubusercontent.com/sherlockguo/WeMusic/master/setup.sh | bash
```

等待几分钟（首次安装依赖），看到 `✅ 安装完成！` 即可。

**③ 启动 + 安装到桌面**

```bash
cd ~/WeMusic && nohup npm start &
```

手机浏览器打开 `http://localhost:5174` → Chrome 菜单 → **「添加到主屏幕」**。

以后桌面点 WeMusic 图标就能用（Termux 需要在后台保持运行）。

### 源码运行（电脑）

```bash
npm install
npm run build          # 首次运行前必须构建一次前端
cp .env.example .env   # 按需修改 JWT_SECRET 和 PORT
npm start
```

开发模式：
```bash
npm run dev:full       # 同时启动服务端热重载 + 前端自动构建（推荐）
# 或者分两个终端：
npm run build:watch    # 终端 1：前端自动构建
npm run dev            # 终端 2：服务端热重载（node --watch）
```

启动后访问 `http://localhost:5174`，首次使用点「立即注册」。

## 使用流程

1. **发现音乐**：默认进入发现页，浏览推荐或榜单。
2. **搜索**：搜索框输入歌手 / 歌曲名。
3. **添加歌单**：点 ＋ → 选择歌单或新建。
4. **收藏专辑**：进入专辑详情页 →「＋ 收藏专辑」→ 侧边栏「我的专辑」查看。
5. **导入歌单**：左下角 ⇪ → QQ 链接或 JSON 文件。
6. **备份**：歌单详情页「⤓ 导出歌单」。
7. **分享报告**：我的数据 →「🎨 生成分享海报」→ 选主题 → 本地或服务端下载。

## 推荐算法

### 信号权重

| 信号 | 权重 | 说明 |
|------|------|------|
| ❤️ 红心 | +3.0 | 最强正信号 |
| 加入歌单 | +2.5 | 主动收藏 |
| 完播率 ≥ 80% | +1.5 | 真正听完 |
| 重复播放 | +0.5/次（上限 +3） | 越听越喜欢 |
| 完播率 20–80% | +0.3 | 一般兴趣 |
| 完播率 < 20% | −0.3 | 可能不喜欢 |
| 时间衰减 | 近期 ×1.0 → 90天后 ×0.5 | 近期行为优先 |

### 歌手权重修正

合唱"邓紫棋 / 艾热AIR"中，非主唱权重 ×0.4。仅独立权重 ≥ 2.0 的歌手进入种子池。

### 三维推荐来源

1. **歌手深度挖掘** — 权重最高 3 个歌手搜歌
2. **风格扩散** — 高分歌曲名 +「翻唱」关键词搜索，发现同风格跨歌手版本
3. **热歌榜混入** — 当前热歌榜 20 首，保持新鲜度

候选评分后按正分分桶（高 ≥1.5、中 0~1.5），负分不推荐（已被跳过的歌）。

## 安全措施

| 措施 | 实现 |
|------|------|
| 点击劫持 & MIME 嗅探 & XSS | 响应头防护 |
| 全局 API 限流 | 120 次/分钟/IP |

## 法律与合规

> **重要：本项目仅供个人本地使用。** 部署到公网或接受付款会产生法律风险。

| 场景 | 风险 |
|---|---|
| **个人本地使用**（默认）| 无风险，属著作权法 §24 合理使用 |
| 公开部署（不收费） | QQ 音乐 API 可能封 IP；B 站 iframe 大量使用可能被限流 |
| 公开部署 + 赞赏/收费 | 构成经营行为，丧失合理使用抗辩资格，**高风险** |
| 赞赏（仅限开发者个人支持） | 安全——定位为开源项目支持，非音乐服务付费 |
| B 站视频版权 | B 站已向音著协缴纳授权费，用户上传视频合法 |
| QQ 音乐 API | 禁止商业用途和二次分发，大规模调用有法律风险 |

> 建议本地部署或通过 Tailscale/Zerotier 等工具建立私人虚拟局域网分享给好友。

## 已知限制

### 跨域 iframe 导致的不可修复问题

| 问题 | 原因 |
|------|------|
| 进度条无法同步视频 | iframe 跨域 → 无法读写 video.currentTime |
| 画质低 | B 站服务端识别嵌入播放，主动降级码率 |
| iframe 内置弹幕按钮无效 | 嵌入模式沙箱拦截交互事件 |
| 无法登录 B 站账号 | iframe 隔离，不传递登录态 + 嵌入模式无视登录 |
| 前台 iframe 音量无法通过 WeMusic 控制 | iframe 跨域隔离，无法读写 B 站播放器音量，需在视频内手动调节 |

### 设计限制

| 问题 | 说明 |
|------|------|
| 自动连播靠定时器非播放事件 | 视频 MV 时长不一致时可能提前/延迟切歌 |
| 单用户架构 | SQLite + 单机部署，不支持多用户并发 |
| 部分 QQ 音乐专辑封面 200px/300px 尺寸不可用 | 统一使用 500px 尺寸以确保兼容性 |
