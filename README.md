# WeMusic · 本地个人音乐网站

一个仅供本地部署、个人使用的音乐网站。借助 QQ 音乐的歌单 / 歌手 / 专辑数据进行检索与管理，
播放时自动从 **Bilibili** 匹配音质最佳、非现场版的视频资源进行嵌入播放，实现免费收听体验。

> 仅供个人学习与本地使用，请勿用于商业用途。所有音乐版权归原平台与版权方所有。

---

## 功能特性

### 用户系统
- 注册 / 登录，JWT + bcrypt（cost=12）鉴权，适用于单用户保存个人数据。
- 注册完成后可在 `.env` 设置 `ALLOW_REGISTER=false` 关闭注册入口。
- 用户头像上传（自动压缩到 300px，base64 存储于 SQLite）。

### 歌单管理
- 粘贴 QQ 音乐歌单链接，自动解析并批量导入歌曲。
- 输入歌手名，一键批量添加该歌手的全部歌曲；也可按专辑整张添加。
- 新建 / 重命名（双击歌单名）/ 删除（带确认，显示歌单名与歌曲数，删除后自动跳回首页）。
- 歌单内**拖拽排序**，持久化到数据库。
- **导出 / 导入歌单**（JSON 格式，含已缓存的 Bilibili 资源，便于备份与设备迁移）。
- 歌单详情页顶部显示**专辑封面拼图**（取前 4 张专辑封面拼成 2×2）。

### 发现音乐
- **为你推荐**：基于播放历史和喜欢记录，自动推荐相关歌手的歌曲；同时混入热歌榜增加广度。
- **热歌榜 / 新歌榜 / 流行指数 / 识曲榜**：实时拉取 QQ 音乐排行榜，带排名和封面，可直接播放。

### 歌曲标记
- 搜索结果 / 歌手页 / 专辑页中，已加入过任意歌单的歌曲在时长左侧显示 🔖 标记。
- 点 ＋ 打开歌单选择器时，已完整收录的歌单显示绿色 ✓，部分收录显示橙色 •。

### 播放
- 点击歌曲自动调用 Bilibili 搜索接口（含 WBI 签名 + buvid 激活）匹配对应资源。
- **歌手准确性优先**：优先保留目标歌手版本，过滤其他歌手演唱的同名曲。
- **播放质量排序**：过滤伴奏 / 纯音乐 / 消音；提升无损 / Hi-Res / 高音质权重。
- 自动过滤现场 / 演唱会 / Live 版本。
- 支持手动**换源**（底部播放器 + 视频面板均可触发）。
- bvid 全局缓存：匹配结果缓存到数据库，下次播放秒开。
- 风控自动重试：遇到 Bilibili 偶发风控，自动刷新 buvid 身份重试 3 次。

### 播放控制
- **视频浮窗**：可拖动、缩放（拖右下角）、全屏（⛶）、收起 / 展开；展开 / 收起状态持久记忆。
- **播放模式**：🔁 列表循环 / 🔂 单曲循环 / 🔀 随机播放，一键切换并记忆。
- **自动连播**：按歌曲时长定时器驱动，播完自动跳下一首（遵循播放模式）。
- **播放队列 / 历史 合并抽屉**：播放队列 + 最近播放，同框切换，支持跳播、移除、清空。
- 恢复上次播放：刷新或重启后底栏显示上次的歌曲与队列，点 ▶ 继续。
- **播放器直接操作**：底栏 ❤ 喜欢按钮 + ··· 添加到歌单 / 复制链接菜单。

### 播放列表工具
- **播放全部** / **随机播放** 按钮。
- **列表内筛选**：输入关键词即时过滤，不影响播放索引。

### 搜索辅助
- 搜索框聚焦自动弹出历史记录（最近 12 条），输入时按关键词建议。
- 支持单条删除或一键清空。

### 歌曲右键菜单
右键单击任意歌曲行，弹出快捷操作菜单：▶ 播放 / ＋ 添加到歌单 / 🔗 复制 Bilibili 链接 / ✕ 从歌单移除（仅歌单页）。

### 歌词
- 点击底栏「词」或播放器封面进入**全屏歌词页**（毛玻璃背景 + 旋转封面 + 同步滚动）。
- 网易云音乐多策略搜索（歌名+歌手 → 去括号 → 纯歌名 → 回退）。
- 歌词页底部迷你播放控制栏（上一首 / 暂停 / 下一首 + 进度条）。
- 歌词页直接操作 ❤ 喜欢和 ＋ 添加到歌单。

### 数据统计
- **我的数据** 页：总播放次数 / 时长 / 不重复歌曲 / 听歌天数。
- 30 天播放趋势柱状图 + 听歌时段热力图 + 最爱歌手条形图 + 最常播放 Top 10（可直接播放）。
- 播放时长真实统计（按 elapsed 上报，非固定值）。

### 界面
- 深色 / 浅色 / **跟随系统** 三种主题。
- 侧边栏 Logo 居中，宽度可拖拽调整（180–440 px，记忆宽度）。
- 定时停止播放（15 分 / 30 分 / 60 分 / 播完当前曲）。
- 响应式设计，适配桌面与移动端浏览器（底部 Tab Bar + 简化歌曲行）。
- PWA 支持：可安装到桌面，离线可访问（Service Worker 缓存策略）。

### 键盘快捷键

## 技术栈

| 层级 | 技术 |

## 快速开始

```bash
# 1. 安装依赖
npm install

# 2. 准备环境变量（推荐，已有 .env 则跳过）
cp .env.example .env
# 可自定义 JWT_SECRET 和 PORT（默认 5174）

# 3. 启动
npm start
```

启动后终端会打印本机地址与局域网地址：

```
WeMusic 已启动 ✅
  本机访问:   http://localhost:5174
  局域网访问: http://192.168.x.x:5174
```

## 使用流程

1. **发现音乐**：打开默认进入发现页，浏览推荐或榜单，直接点击播放。
2. **搜索**：搜索框输入歌手 / 歌曲名，进入结构化结果页。
3. **添加歌单**：点 ＋ → 选择已有歌单或新建。
4. **导入歌单**：左下角 ⇪ 按钮 → 粘贴 QQ 音乐链接或选择 JSON 备份文件。
5. **备份**：进入歌单详情页，点「⤓ 导出歌单」保存 JSON。
6. **换源**：对匹配不满意，点底栏或视频面板「换源」手动选择。

## 安全措施

| 措施 | 实现 |
|------|------|
| 点击劫持防护 | `X-Frame-Options: DENY` |
| MIME 嗅探防护 | `X-Content-Type-Options: nosniff` |
| XSS 防护 | `X-XSS-Protection: 1; mode=block` |
| 权限策略 | 仅允许 autoplay / fullscreen，禁止摄像头 / 麦克风 / 定位 |
| 全局限流 | 120 次/分钟/IP（本机不限流） |
| 登录限流 | 10 次/15 分钟/IP（防暴力破解 + 防用户枚举） |
| Payload 限制 | 256KB 请求体上限 |
| 密码策略 | ≥8 位，≤128 位；bcrypt cost=12 |
| 注册开关 | `ALLOW_REGISTER=false` 关闭注册 |
| 错误脱敏 | 500 错误不暴露内部堆栈 |

## 已知问题与限制

### 1. 播放器进度无法同步到 iframe 视频 ⚠️

**现象**：底栏进度条是一个独立定时器，用于估算时间并触发自动连播。它无法控制或反映 Bilibili 视频的实际播放进度。

**原因**：Bilibili iframe 嵌入播放器（`player.bilibili.com`）与你的页面（`localhost:5174`）**跨域**。浏览器安全策略禁止跨域访问 iframe 内部的 DOM、JS 变量或视频元素的 `currentTime`。且 B 站未提供类似 YouTube iframe API 的 postMessage 控制接口。

**影响**：
- 拖拽进度条无法跳转视频进度
- 在 iframe 内手动暂停视频后，定时器仍会继续，可能导致提前切歌
- **缓解**：在 iframe 暂停时点底栏 ⏸ 可暂停自动连播计时器

### 2. 视频清晰度受 Bilibili 嵌入限制 ⚠️

**现象**：iframe 嵌入播放器画质明显低于直接访问 B 站网页版观看。

**原因**：Bilibili **故意对嵌入播放器降级画质**（商业策略，防止优质内容被外站蹭流）。这与登录状态无关——即使浏览器已登录 B 站账号，嵌入播放器仍以访客/低画质模式运行。

**尝试过但不可行的方案**：
- 登录 B 站账号：iframe 内无法触发登录流程，且嵌入模式不传递登录态
- 设置 B 站 cookies：iframe domain 隔离，无法注入
- 更换嵌入参数：B 站不提供画质相关的 URL 参数

**技术死局**：这是 B 站服务端根据 `Referer` / 嵌入标识主动下发的低码率流，前端无解。

### 3. 跨域带来的其他限制

| 问题 | 原因 |
|------|------|
| 无法获取视频真实播放进度 | 跨域 iframe 无权读取视频 `currentTime` |
| iframe 内置"弹"按钮无效 | 嵌入模式沙箱限制，交互事件被拦截 |
| 无法静音/调节音量 | B 站播放器音量控件在跨域沙箱中不可控 |
| 无法控制视频暂停/播放 | iframe 不支持 postMessage 跳转 |

### 4. 自动连播基于定时器而非播放事件 ⚠️

自动连播的切换时机是「定时器播完歌曲时长」，而非「视频真正播完」。如果：
- 视频是 MV 比歌曲长 → 可能视频还没播完就被切了
- 视频比歌曲短 → 可能歌曲没播到结尾就被切了
- 视频加载耗时过长 → 计时器在加载期间也在走

### 5. app.js 代码集中 ⚠️

前端核心逻辑全部位于 `public/js/app.js`，已超过 2000 行。在无打包工具（Webpack/Vite）的情况下拆分为多文件需要引入 `type="module"` 并处理依赖顺序，改动量大。当前阶段维持单文件，后续可引入 Vite 实现模块化打包。

### 6. 歌词匹配并非 100% 准确

网易云搜索策略已做到多层匹配（歌名+歌手 → 去括号 → 纯歌名 → 回退），但部分冷门歌曲或网易云无版权的曲目仍可能找不到或匹配错误。目前无 QQ 音乐官方歌词接口可用。

## 常见问题

**Q：搜索 / 解析无结果**  
QQ 音乐接口有访问频率限制，沙盒 / 数据中心 IP 可能受限。本地中国网络环境通常正常。

**Q：播放时提示"未在 Bilibili 找到视频" / 风控**  
服务会自动刷新 buvid 并重试 3 次。若仍失败，稍等几秒再次尝试，或点「换源」手动搜索。

**Q：同名歌曲播到了别的歌手版本**  
系统会优先过滤保留目标歌手版本。若仍不准，点「换源」手动选择。

**Q：搜索歌手只加载了 100 首歌**  
首批加载 100 首，列表底部有「加载更多」按钮，每次追加 100 首。

**Q：收起视频浮窗后没声音**  
「收起」（−）只隐藏画面，iframe 继续播放。只有「关闭」（✕）才会停止。

**Q：如何只允许自己使用**  
在 `.env` 中设置 `ALLOW_REGISTER=false` 并重启服务。

**Q：主题切换在哪里**  
顶栏 ⚙ → 设置面板 → 深色 / 浅色 / 跟随系统。
