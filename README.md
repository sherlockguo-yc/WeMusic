# WeMusic · 本地个人音乐网站

一个仅供本地部署、个人使用的音乐网站。借助 QQ 音乐的歌单 / 歌手 / 专辑数据进行检索与管理，
播放时自动从 **Bilibili** 匹配音质最佳、非现场版的视频资源进行嵌入播放，实现免费收听体验。

> 仅供个人学习与本地使用，请勿用于商业用途。所有音乐版权归原平台与版权方所有。

---

## 功能特性

### 用户系统
- 注册 / 登录，JWT + bcrypt 鉴权，适用于单用户保存个人数据。
- 注册完成后可在 `.env` 设置 `ALLOW_REGISTER=false` 关闭注册入口。

### 歌单管理
- 粘贴 QQ 音乐歌单链接，自动解析并批量导入歌曲。
- 输入歌手名，一键批量添加该歌手的全部歌曲；也可按专辑整张添加。
- 新建 / 重命名（双击歌单名）/ 删除（带确认，显示歌单名与歌曲数，删除后自动跳回首页）/ 去重添加 / 删除单曲（同步刷新首 / 总时长统计）。
- 歌单内**拖拽排序**，拖拽结果持久化到数据库。
- **导出 / 导入歌单**（JSON 格式，含已缓存的 Bilibili 资源，便于备份与设备迁移）。
- 歌单详情页顶部显示**专辑封面拼图**（取前 4 张专辑封面拼成 2×2）。

### 歌手检索
- 搜索歌手，结构化展示全部歌曲与专辑列表，专辑封面实时加载。
- 命中歌手时直接展示该歌手的完整曲库（第一批 100 首），支持**加载更多**（每批 100 首）。
- **精选集 / 合辑 / 节目版自动去重**：歌手页按歌名去重，保留正规专辑版本，过滤"精选""Best of""我是歌手""王牌对王牌"等重复条目。

### 已添加标记
- 搜索结果 / 歌手页 / 专辑页中，已加入过任意歌单的歌曲行末显示 🔖 标记。
- 点 ＋ 打开歌单选择器时，已完整收录的歌单显示绿色 ✓，部分收录显示橙色 •。

### 播放
- 点击歌曲自动调用 Bilibili 搜索接口（含 WBI 签名 + buvid 激活）匹配对应资源。
- **歌手准确性优先**：优先保留目标歌手版本，过滤其他歌手演唱的同名曲。
- **播放质量排序**：过滤伴奏 / 纯音乐 / 消音；提升无损 / Hi-Res / 高音质权重；兼顾播放量与时长匹配。
- 自动过滤现场 / 演唱会 / Live 版本。
- 支持手动**换源**，在候选列表中自由选择；换源结果缓存到歌单，下次播放秒开。
- 风控自动重试：遇到 Bilibili 偶发风控，自动刷新 buvid 身份重试 3 次。

### 播放控制
- **视频浮窗**：可拖动、缩放（拖右下角）、全屏（⛶）、收起 / 展开；展开 / 收起状态持久记忆，切歌不重置。
- **播放模式**：🔁 列表循环 / 🔂 单曲循环 / 🔀 随机播放，一键切换并记忆；切换新队列时自动清空随机回溯历史。
- **自动连播**：按歌曲时长计时，播完自动跳下一首（遵循播放模式）。
- **播放队列 / 历史 合并抽屉**（底栏"队列"按钮）：
  - 播放队列 Tab：查看完整队列、跳播任意一首、移出队列（移除正在播放项时自动播下一首）；清空队列同步重置底栏显示。
  - 最近播放 Tab：最近 100 首播放记录，支持单条移除和一键清空，点击直接重播。
- 恢复上次播放：刷新或重启后底栏显示上次的歌曲与队列，点 ▶ 继续。
- 切歌时浏览器标签页标题同步更新为"歌名 · 歌手"；停止播放后自动恢复默认标题。

### 播放列表工具
- **播放全部** / **随机播放** 按钮，一键将整个列表加入播放队列。
- **列表内筛选**：输入关键词即时过滤歌名 / 歌手，不影响播放索引。

### 搜索历史
- 搜索框聚焦自动弹出历史记录（最近 12 条），输入时按关键词过滤。
- 支持单条删除或一键清空。

### 歌曲右键菜单
右键单击任意歌曲行，弹出快捷操作菜单：
- ▶ 播放 / ＋ 添加到歌单
- 🔗 复制 Bilibili 链接 / 🔍 在 Bilibili 搜索
- ✕ 从歌单移除（仅歌单页显示）

### 界面
- **设置面板**（顶栏 ⚙）：外观主题（深色 / 浅色 / **跟随系统**，实时响应 macOS 日出日落）、定时停止播放、账户信息与退出登录。
- 侧边栏快捷导航：导入 / 管理、最近播放。
- 侧边栏宽度可拖拽调整（180–440 px，记忆宽度）。
- 底栏正在播放封面缩略图（QQ 音乐专辑封面）。
- 响应式设计，适配桌面与移动端浏览器。

### 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `空格` | 暂停 / 继续自动连播 |
| `→` | 下一首 |
| `←` | 上一首 |
| `/` | 聚焦搜索框 |
| `?` | 显示快捷键帮助 |
| `Esc` | 关闭弹层 / 收起视频 |

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Node.js 20 + Express + better-sqlite3 + JWT |
| 前端 | 原生 HTML / CSS / JavaScript（无需构建步骤） |
| 音乐数据 | QQ 音乐公开 Web 接口（smartbox 搜索、歌手/专辑/歌单） |
| 播放 | Bilibili 搜索 API（WBI 签名 + buvid 激活）+ 官方 iframe 播放器 |
| 存储 | SQLite（歌单 / 歌曲 / bvid 缓存）+ localStorage（主题 / 队列 / 历史 / 设置） |

## 快速开始

```bash
# 1. 安装依赖
npm install

# 2. 准备环境变量（推荐，已有 .env 则跳过）
cp .env.example .env
# 可自定义 JWT_SECRET（建议修改为随机字符串）和 PORT（默认 5174）

# 3. 启动
npm start
# 或开发模式（文件变更自动重启）
npm run dev
```

启动后终端会打印本机地址与局域网地址：

```
WeMusic 已启动 ✅
  本机访问:   http://localhost:5174
  局域网访问: http://192.168.x.x:5174（供同局域网的朋友使用）
```

访问 **http://localhost:5174**，首次使用点「立即注册」创建账户即可。

## 使用流程

1. **导入歌单**：首页粘贴 QQ 音乐歌单链接 → 「解析歌单」→「全部添加到歌单」。
2. **按歌手添加**：搜索框输入歌手名 → 进入歌手页 → 「添加已加载歌曲」或点开专辑整张添加；点「加载更多」可逐批加载该歌手全部歌曲。
3. **检索单曲**：搜索框输入歌曲名，右键或点 ＋ 添加到目标歌单。
4. **播放**：点击歌曲行（整行可点），系统自动从 Bilibili 匹配并开始播放。
5. **换源**：对匹配结果不满意，点底栏「换源」或视频浮窗「换源」手动选择。
6. **备份**：进入歌单详情页，点「⤓ 导出歌单」保存 JSON；侧边栏「⇪」按钮导入已备份歌单。

## 让朋友访问

**同一 WiFi 局域网**（最简单）：  
连接同一 WiFi 后，朋友直接访问终端打印的局域网地址，例如 `http://192.168.1.x:5174`。

**跨网络公网访问**（需要工具）：  
使用内网穿透工具（如 [cpolar](https://www.cpolar.com/)、[ngrok](https://ngrok.com/) 等），运行后获取临时公网地址发给朋友即可。服务仍运行在你的电脑上。

> **macOS 防火墙**：若局域网朋友无法访问，进入「系统设置 → 网络 → 防火墙」，给 Node.js 放行或临时关闭防火墙。

## 常见问题

**Q：搜索 / 解析无结果**  
QQ 音乐接口有访问频率限制，沙盒 / 数据中心 IP 可能受限。本地中国网络环境通常正常。相关逻辑集中在 `server/services/qqmusic.js`，便于按需维护。

**Q：播放时提示"未在 Bilibili 找到视频" / 风控**  
服务会自动刷新 buvid 访客身份并重试 3 次。若仍失败，稍等几秒再次点播放，或点「换源」手动搜索。

**Q：同名歌曲播到了别的歌手版本**  
系统会优先过滤保留目标歌手的版本（标题 / UP 主名匹配）。若仍不准，点「换源」手动选择。

**Q：搜索歌手只看到 100 首歌**  
首批加载 100 首，列表底部有「加载更多」按钮，每次追加 100 首，直至全部加载完毕。

**Q：收起视频浮窗后没声音**  
视频浮窗「收起」只隐藏画面，iframe 保持存活、声音继续。只有点 ✕（关闭）才会停止播放。

**Q：自动连播提前切歌**  
自动连播基于歌曲时长计时（无法跨域读取 B 站播放器进度）。若在浮窗里手动暂停了视频，计时仍继续，可能提前切歌——此时点底栏 ⏸ 暂停自动连播即可。

**Q：如何只允许自己使用**  
注册账号后，在 `.env` 中设置 `ALLOW_REGISTER=false` 并重启服务，注册入口即关闭。

**Q：主题切换在哪里**  
点顶栏 ⚙ 打开设置面板，可选深色 / 浅色 / 跟随系统（会实时响应 macOS 日出日落切换）。

## 安全说明

- 数据全部存储在本地 `data/wemusic.sqlite`，不上传任何第三方。
- 建议修改 `.env` 中的 `JWT_SECRET` 为随机字符串（服务首次启动会提示）。
- 本项目面向个人本地部署，未做公网部署的安全加固，请勿直接长期暴露到公网。
