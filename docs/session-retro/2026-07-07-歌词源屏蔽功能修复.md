## 复盘总结

本次会话核心是修复屏蔽功能的多个串联 bug。最耗时的分别是：

1. **数组下标 bug**：两个弹窗（lyrics + video）都存在同样的问题，好在发现后一次修复干净。

2. **屏蔽源展示格式的 3 轮迭代**：第一版只用 localStorage 存元信息，忽略了旧数据场景 → 第二版加了 DB 迁移但没回填旧数据 → 第三版加了服务端回填才彻底解决。如果一开始就设计为「服务端存储 + 旧数据回填」，可以节省约 2 轮交互。

3. **主源被屏蔽导致「当前」标记丢失**：根因是 `fetchLyrics` 不感知黑名单，返回的 `sourceId` 可能已被屏蔽。修复涉及 3 个代码路径（sourceId 分支 / cache HIT / cache MISS），每个都要处理。

4. **图片不可见**：由于模型不支持多模态，只能从代码 + 用户文字描述推断 bug 表现。用户提供了 3 张截图但无法直接读取，增加了推断成本。
