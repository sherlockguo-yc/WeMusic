# 网络概念速查

> 从 [自建服务器方案](./自建服务器方案.md) 第一章拆出，独立为速查手册。

## WAN vs LAN

| 术语 | 全称 | 作用 | 类比 |
|------|------|------|------|
| **WAN** | Wide Area Network（广域网口） | 接上级网络，通往互联网 | 房子的大门 |
| **LAN** | Local Area Network（局域网口） | 接内部设备，家里互通 | 卧室门 |

每个路由器至少一个 WAN 口（出）和多 LAN 口（入）。

## DHCP——谁分配 IP

DHCP（动态主机配置协议）= 每个路由器里自动分配 IP 的程序。新设备接进来，DHCP 就给一个空闲地址。`192.168.x.x` 是国际标准规定的私有地址，全球家庭都用，不和外网冲突。

## 你的 IP 分配链

```
移动 DHCP → 光猫 WAN：172.86.86.111（CGNAT 地址，会变）
光猫 DHCP → 小米路由器 WAN：192.168.1.105（光猫 LAN 是 192.168.1.x 网段）
小米路由器 DHCP → N150：192.168.31.102（DHCP 静态绑定，不变）
```

## 为什么是双重 NAT

你买的小米路由器接在运营商给的光猫后面——即便没有 N150，这种拓扑也是双重 NAT。每层设备都给下一层分配内部地址，两层叠加导致外部无法直接访问内部设备。

## DNS 是什么、为什么用 223.5.5.5

DNS（域名系统）= 把域名翻译成 IP 的服务。访问 `baidu.com` 时，浏览器先问 DNS 服务器「这个域名的 IP 是多少」，拿到 IP 后才真正连接。

`223.5.5.5`（阿里）和 `119.29.29.29`（腾讯）是公共 DNS，免费、快、不被运营商劫持。配**两个**是备份——第一个不可用时自动用第二个。这些 IP 背后是几百上千台服务器集群（任播技术），一个 IP 能承载海量请求。
