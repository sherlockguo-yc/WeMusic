## 二、方案：N150 小主机 + DDNS + 非标端口

### 链路（当前阶段：直接非标端口）

```
用户 → https://域名:8443 → 路由器端口转发(8443) → N150 → Nginx → Docker 服务
```

### 链路（未来升级：CF Tunnel / DNS Proxy）

**方案 A：CF Tunnel（推荐）**——无需开放入站端口
```
用户 → https://域名(标准443) → CF 边缘节点 → CF Tunnel（出站加密隧道）→ N150 → Nginx → Docker 服务
```

**方案 B：CF DNS Proxy**——仍需开放非标端口
```
用户 → https://域名(标准443) → CF 边缘节点 → N150:8443 → Nginx → Docker 服务
```

详见「常见问题 → 80/443 端口被封怎么办？」。

### 注册商选择

#### 为什么要选 Cloudflare？

| 注册商 | `.com` 续费/年 | 核心竞争力 | 自建场景适配度 |
|--------|---------------|-----------|--------------|
| **Cloudflare** ✅ | **$10.46（≈73 元）** | 成本价（零利润）；Whois 隐私免费；DDNS API 原生支持；通配符证书免费 | ⭐⭐⭐⭐⭐ |
| Porkbun | ~$10.37（≈72 元） | 界面友好、客服响应快、也送 Whois 隐私 | ⭐⭐⭐ |
| Namecheap | ~$15（≈105 元） | 老牌、功能全面 | ⭐⭐ |
| GoDaddy | ~$20（≈140 元） | 名气最大 | ⭐（贵 + 捆绑推销） |

**Cloudflare 是自建服务器场景的最优解：**

- **成本价**：CF 域名注册是战略性零利润业务，定价 = Verisign 出厂价（~$9.50）+ ICANN 税（~$0.18），不靠域名赚钱，靠生态绑定（用户用了域名的 DNS → 可能用 CF 的 CDN/Zero Trust 等付费服务）
- **DDNS 天然契合**：ddclient / Cloudflare API 直接操作 CF DNS，不需要额外配置 API 密钥权限。域名和 DNS 托管在同一平台，省掉"跨平台对接"的折腾
- **Whois 隐私免费**：注册信息不会泄露到公共 Whois 数据库（GoDaddy 同服务每年额外收 ~$10）
- **通配符证书免费**：CF 免费签发 `*.sherlockguo.com`，支持 DNS 验证，一次申请所有子域名通用

#### 域名系统的层级：为什么不能绕过注册商？

域名系统有三层角色，ICANN 规则刻意要求分离，防止垄断：

```
ICANN（互联网命名系统协调组织，非营利）
  ├── 注册局（Registry）→ Verisign 管理 .com 数据库（只批发，不零售）
  └── 注册商（Registrar）→ Cloudflare / Namecheap 等（面向用户零售）
```

| 角色 | 职责 | 能否直接向用户出售域名？ |
|------|------|-----------------------|
| ICANN | 顶层协调、委派 TLD | ❌ 不直接参与交易 |
| 注册局（Registry） | 管理某个 TLD 的权威数据库 | ❌ 规则禁止零售，防止垄断 |
| 注册商（Registrar） | 代理注册，面向终端用户 | ✅ 这是唯一的购买渠道 |

**类比**：

| 场景 | 注册局 | 注册商 | 终端用户 |
|------|--------|--------|---------|
| 域名 | Verisign | Cloudflare | 你 |
| 汽车 | 车管所 | 4S 店 | 买车的人 |
| 房子 | 不动产登记中心 | 房产中介 | 买房的人 |

车管所只登记产权，不卖车。注册局只管理数据库，不卖域名。这个"中间商"是系统设计的一环——注册局维护根 DNS 服务器集群需要成本（7×24 可用、抗 DDoS、每秒百万级查询），注册费就是这部分基础设施的维护费用。

**所有注册商查的是同一个数据库**（Verisign 的 `.com` 权威数据库），不存在"Cloudflare 这边还有、Namecheap 那边没了"的情况。搜 `sherlockguo.com` 时三家都去 Verisign 查——如果已被人注册，三家都显示"不可用"。

> Cloudflare 在这个场景不是普通中间商——它**不要这一环的利润**（定价 = 出厂价），所以「没有中间商赚差价」正好在 CF 这里实现了，虽然原因和直觉不同。

### 多服务承载方案

**结论：推荐子域名方案**（待最终确认）

| 方案 | 示例 | 优点 | 缺点 |
|------|------|------|------|
| **子域名**（推荐） | `wemusic.sherlockguo.com`<br>`blog.sherlockguo.com` | 各服务隔离清晰；独立证书/CDN；后期某服务可独立搬家不影响其他；Cookie/Service Worker 不打架 | 需通配符证书（Cloudflare 签发免费）或每个子域名单独申请 |
| 子路径（不推荐） | `sherlockguo.com/wemusic`<br>`sherlockguo.com/blog` | 配置简单；一个证书覆盖全部 | 路径前缀要写进代码；前后端都要改；Cookie 容易冲突；后期某服务想独立出来很麻烦 |

**子域名方案效果：**

```
sherlockguo.com                    → 个人主页 / 服务导航
├── wemusic.sherlockguo.com        → 音乐服务（WeMusic）
├── blog.sherlockguo.com           → 博客
├── api.sherlockguo.com            → 通用 API
└── *.sherlockguo.com              → 未来其他服务
```

**DDNS 配置**：所有子域名共用同一个 A 记录（指向公网 IP），DDNS 发现 IP 变化时自动更新。Nginx Proxy Manager 按 `Host` 头区分流量分发到不同后端。

**SSL 证书**：申请 `*.sherlockguo.com` 通配符证书一次，所有子域名通用，新增子域名零成本。

### Wi-Fi 驱动（✅ 已解决）

> **硬件**：MediaTek MT7902 Wi-Fi 6E / PCI ID `14c3:7902` / 子系统 Foxconn `105b:e13f`

#### 最终方案：morrownr/mt76

**驱动仓库**：[github.com/morrownr/mt76](https://github.com/morrownr/mt76)（支持内核 6.12–7.x，无需升级系统）

```bash
sudo apt install gcc make bc iw dkms -y
git clone https://github.com/morrownr/mt76.git
cd mt76 && sudo sh install-driver.sh NoPrompt
sudo reboot
```

重启后驱动自动绑定为 `mt7921e_git`，WiFi 网卡 `wlp3s0` 正常可用。

**成功根因**：该驱动源码为 MT7902 写了专属适配——PCI ID 表原生含 `0x7902`（自动绑定，无需 `new_id` 强制） + 专门处理了 MT7902 不同于 MT7921/7925 的 MCU ring 布局。之前尝试的内置 `mt7921e`/`mt7925e` 及其他 out-of-tree 驱动都没有这些处理。

#### 内核升级注意事项

## 六、常见问题

### 为什么需要域名？没有域名可以对外服务吗？

**直接用 IP 访问（`https://120.229.x.x:8443`）技术上可行，但实际不可用**，原因有三：

| 问题 | 影响 |
|------|------|
| **SSL 证书无解** | Let's Encrypt 不支持为 IP 地址颁发证书。没有 HTTPS → 浏览器报"不安全"→ HTTP/2、Service Worker 等现代特性全部不可用 |
| **IP 一变就失联** | 每次光猫重启换 IP，你得通知所有用户"新地址是 xxx"，然后他们改书签 |
| **没人记得住** | 一串数字 vs 一个域名，分享链接时别人会以为是钓鱼 |

域名解决了这三个问题：
- 配合 DDNS，IP 变化时自动更新 DNS 记录，用户始终用同一个域名访问
- 可以为域名申请 Let's Encrypt 免费 SSL 证书（Nginx Proxy Manager 自动续签）
- 好记、可分享、看起来专业

**域名是自建服务器里最便宜的零件**（年均几十块），但没有它整个方案就跑不起来——就像电话号码每天都变，只有你自己能用。

### 80/443 端口被封怎么办？

移动封锁 80/443 入站端口。三种解决方案：

| 方案 | URL | 原理 | 是否需要开放入站端口 | 推荐 |
|------|-----|------|-------------------|------|
| 非标端口（当前） | `https://域名:8443` | 直接使用其他端口 | ✅ 需要 | 临时用 |
| CF DNS Proxy | `https://域名`（标准 443） | CF 代理模式（橙色云朵），回源到 8443 | ✅ 需要 | 升级选项 |
| **CF Tunnel** | `https://域名`（标准 443） | N150 主动向 CF 建立加密出站隧道，用户访问 CF 边缘节点 | ❌ **不需要** | ⭐ 最推荐 |

**CF Tunnel 详解**（推荐）：

```
用户 → https://wemusic.sherlockguo.com (标准 443)
        ↓
   Cloudflare 边缘节点
        ↓  cloudflared 加密隧道（N150 主动向外连接）
   N150 → WeMusic (localhost:3000)
```

- **不需要开放任何入站端口**：隧道是 N150 主动向外连 CF，不是外面连进来
- 用户体验完美：标准 443，无端口号
- 免费：一个账号最多 50 个 Tunnel
- Cloudflare 承担 DDoS 防护
- 在 N150 上安装 `cloudflared` 客户端即可，一个 YAML 配置完成
- **关键优势**：不依赖端口转发、不依赖 DDNS、不暴露公网 IP，运营商无法封锁（本质是出站 HTTPS 长连接，和浏览器访问网站一样）

### IP 变了怎么办？

**为什么 IP 会变**：中国移动家用宽带给的是「动态公网 IP」——每次拨号上网时，运营商的 DHCP 服务器从地址池里临时分配一个 IP。重启光猫/路由器 = 重新拨号 = 大概率换 IP。即使不重启，DHCP 租期到了也可能换（移动的租期通常几天到几周，不稳定）。固定 IP 是企业宽带才有的，价格通常是家宽的 10 倍以上。

**解决方案**：DDNS（动态 DNS）客户端每 5 分钟检查一次公网 IP，发现变化后自动更新 DNS 记录。域名始终解析到正确的 IP，用户无感知。

**为什么 IP 变了还能立刻生效**：DDNS 更新后，DNS TTL（生存时间）通常设为 120 秒，即全球 DNS 缓存最多 2 分钟就过期，之后所有请求都会拿到新 IP。

### 停电/断网怎么办？

个人项目可接受偶尔中断。如需更高可用性，可加个 UPS（不间断电源），~200 元。

### 安全措施

- SSH 密钥登录 + 禁用密码
- ufw 防火墙只开放必要端口
- 服务间通过 Docker 内网通信
- 定期更新系统和 Docker 镜像

### 合规与法律风险

> ⚠️ 以下信息仅供参考，不构成法律建议。做任何涉及法律合规的决定前，建议咨询专业人士。

#### 自用的法律边界

**个人用家宽跑自建服务（自用、不对外盈利）属于灰色地带，但普遍如此**——全国成百上千的 NAS 玩家、HomeLab 爱好者都在这个边界内运行。只要保持以下边界，通常是安全的：

- ✅ 自己用、家人朋友用 → OK
- ✅ 不收费、不签合同、不开发票 → OK
- ❌ 公开招揽用户、投放广告、商业运营 → 跨线
- ❌ 帮他人托管服务并收费 → 非法经营 IDC

#### 三条正规合规路径

| 方案 | 月费 | 备案 | 固定 IP | 80/443 | 适合谁 |
|------|------|------|---------|--------|--------|
| **家宽 + CF Tunnel**（推荐） | ~30 元 | 模糊 | 不需要 | ✅（通过 CF） | 个人自用 |
| 家宽升级商宽 | ~500 元 | ✅ 支持 | ✅ | ✅ | 个人创业 |
| 企业专线 | ~3,000+ 元 | ✅ | ✅ | ✅ | 企业生产 |

**商宽路径细节**：
- 需要营业执照（个体工商户即可，去工商局免费办理）
- 月费 300-800 元（带宽通常比家宽更低，上行 10-50M）
- 需要代理记账 ~2,400 元/年（个体户即使零收入也要报税）
- 年总成本 ~6,000 元
- 可以申请 ICP 备案 + 公安备案

**结论**：个人自建服务器的合规性和低成本在国内是互斥的。维持「自用、不盈利」的边界，灰色地带足够安全。

#### ICP 备案与公安备案

| 备案类型 | 谁管 | 什么情况需要 | 家宽能做吗 |
|---------|------|------------|----------|
| ICP 备案 | 省通信管理局 | 国内服务器提供 Web 服务 | ❌ 家宽不支持 |
| 公安备案（网安备案） | 公安机关 | 交互式网站（评论、注册、上传） | 理论上需要，但家宽场景不明确 |

> WeMusic 有用户登录/播放行为，理论上属于交互式网站。但家庭宽带不在现行备案体系的覆盖范围内（运营商不支持家宽备案），存在法规与实践的落差。

#### 核心风险

| 风险 | 等级 | 说明 |
|------|------|------|
| 运营商断网/限速 | 🔴 高 | 移动家宽 TOS 禁止建站，检测到大流量可能采取措施 |
| 被入侵做跳板 | 🟡 中 | 暴露端口 = 暴露攻击面，可能被扫到漏洞后变成 DDoS 肉鸡/挖矿节点 |
| 他人内容连带责任 | 🔴 高 | 如果让他人在你服务器上部署服务，对方放涉黄/涉赌/涉政内容，你是服务器所有者，可能被追究**帮助信息网络犯罪活动罪**（帮信罪） |
| 无证经营罚款 | 🟡 中 | 一旦有收费行为被查出，可没收违法所得 + 3-5 倍罚款 |

**最重要的原则**：服务器只自用，不给他人部署服务，不收费。这是保护自己最有效的方式。
