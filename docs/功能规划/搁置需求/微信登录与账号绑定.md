# 微信登录与账号绑定 ⛔ 阻塞

## 阻塞原因（2026-07-04 确认）

以下三项硬性前提条件**均不具备**，当前无法推进：

| 前提条件 | 说明 | 状态 |
|---|---|---|
| 微信开放平台开发者账号 | 需要企业主体或个体工商户，不支持个人开发者 | ❌ 无 |
| 已备案域名 + HTTPS | OAuth 回调 `redirect_uri` 必须是线上域名，微信会校验 | ❌ 无 |
| 本地开发调试 | OAuth 需要线上回调，本地开发依赖内网穿透（ngrok/frp） | ❌ 公司安全策略禁止使用 |

微信 OAuth 2.0 协议本身决定了这三项缺一不可：没有线上回调地址，整个流程就无法完成。**此功能在获得上述前提条件前处于阻塞状态，暂时不进行实现。**

## 替代思路（未验证）

- 如果未来有域名和服务器，可以部署一个极简的回调代理服务（如 Cloudflare Worker / Vercel Function）转发 OAuth 回调到内网服务器，绕过 ngrok/frp 的安全限制。但这仍需域名+备案。
- 微信以外是否有更轻量的第三方登录？（如 GitHub OAuth — 但面向国内用户不适用；手机验证码登录 — 需要短信服务商，也有成本）

## 概述（原规划）

支持用户通过微信扫码登录，以及已有账号绑定微信，实现多端/多方式登录。微信是国内用户最常用的账号体系，接入微信登录可以大幅降低注册/登录门槛，为后续社交功能（评论、关注、分享）打好用户账号基础。

## 功能方向（原规划）

- **微信扫码登录**：Web 端展示微信扫码页面，用户微信扫码授权后自动登录/注册。基于微信开放平台 OAuth 2.0。
- **绑定微信**：已注册用户可在设置中绑定微信账号，绑定后支持微信扫码登录同一账号。
- **解绑微信**：已绑定微信的用户可在设置中解绑，解绑后不影响通过其他方式登录。
- **微信用户信息**：登录后获取微信昵称、头像等基础信息，作为用户默认头像和昵称。

## 技术考量（原规划）

- 需要微信开放平台开发者账号（需企业主体或个体工商户）。
- 需申请 `snsapi_login` 权限（微信扫码登录）。
- 服务端需新增微信 OAuth 流程：获取 code → 换取 access_token → 换取 openid/unionid → 登录/注册。
- 数据库需扩展 users 表，新增 `wechat_openid`、`wechat_unionid`、`wechat_nickname`、`wechat_avatar` 字段。
- 已有 JWT 认证体系需兼容微信登录：微信登录后同样签发 JWT token。

## 未讨论的设计决策

- [ ] 账号模型：纯微信登录 vs 绑定现有账号

- [ ] 绑定/解绑/换绑逻辑
