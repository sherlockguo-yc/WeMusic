# 主题系统（明星主题）

> 状态：📋 规划中 | 日期：2026-07-22

## 概述

在现有「自定义主题色」（仅修改 `--accent` 强调色）之上，构建完整的**可组合主题系统**。用户可以选择预设明星主题（一键应用完整视觉包），也可以进入「主题编辑器」逐个配置页面组件/元素，每个组件可从已有预设中选择，也可自行上传素材。

目标视觉效果：页面背景是明星写真 + 毛玻璃叠加层，播放器封面框有定制形状，角落有专属装饰元素，整体氛围与明星风格一致。

## 与现有系统的关系

| 现有功能 | 新主题系统 |
|---------|----------|
| `--accent` 强调色（25 预设 + 8 自定义） | 主题选中后**覆盖** `--accent`，用户无需单独调色 |
| `--font` 字体选择 | 主题可携带字体偏好，覆盖现有字体设置 |
| 深色/浅色模式 | 每个主题需同时定义深色和浅色两套值（或声明仅支持某种模式） |
| 自定义颜色 CRUD（`/auth/custom-palettes`） | 不受影响，未启用主题时仍可独立使用 |

## 核心设计

### 主题 = 一组 Slot 配置的集合

将页面可定制的视觉区域抽象为 **Slot（槽位）**，每个 Slot 是一个独立的可配置维度。预设主题 = 所有 Slot 填好的完整配置；自定义主题 = 用户逐个 Slot 选择/上传。

### Slot 清单（8 个）

| # | Slot | 说明 | 取值类型 |
|---|------|------|---------|
| 1 | **页面背景** (`bg`) | 主内容区背景，纯色 / 渐变 / 图片 + 遮罩 | color / gradient / image |
| 2 | **强调色** (`accent`) | 覆盖 `--accent`，影响按钮、链接、高亮等 | color |
| 3 | **字体** (`font`) | 覆盖 `--font`，影响全站文字 | font（从现有 FONTS 列表选） |
| 4 | **侧边栏** (`sidebar`) | 左侧导航区背景/装饰 | color / gradient / image |
| 5 | **播放器底栏** (`player`) | 封面框形状、进度条样式、按钮风格 | preset（预设样式名） |
| 6 | **卡片样式** (`card`) | 歌单卡片、歌手卡片的背景/边框/阴影/圆角 | preset |
| 7 | **装饰元素** (`decorations`) | 页面角落花纹、边框装饰、水印图案等 | preset（SVG/CSS 装饰模板） |
| 8 | **歌词面板** (`lyrics`) | 歌词全屏/桌面歌词的背景、文字颜色、高亮色 | color / image |

### Slot 取值模式

每个 Slot 支持三种来源：

| 来源 | 说明 | 适用 Slot |
|------|------|----------|
| **系统预设** | 官方预制的选项列表（如「毛玻璃卡片」「圆角封面」「音符装饰」） | 所有 Slot |
| **从预设主题复用** | 从某个完整的预设主题中「借」一个 Slot 的值 | 所有 Slot |
| **用户上传** | 用户自行上传图片 / 输入色值 | `bg`、`sidebar`、`lyrics`（image 类 Slot） |

### 双模式工作流

```
预设主题（一键）
  ┌─ 选择预设 → 全部 Slot 自动填充 → 应用
  │
自定义主题（逐个配置）
  └─ 进入编辑器 → 逐个 Slot 选择来源（预设/复用/上传）→ 实时预览 → 保存
```

## 数据模型

### 预设主题

```json
{
  "id": "jay-chou",
  "name": "周杰伦主题",
  "artist": "周杰伦",
  "thumbnail": "/data/themes/jay-chou/thumb.jpg",
  "mode": "dark",
  "slots": {
    "bg":       { "type": "image", "value": "/data/themes/jay-chou/bg.jpg", "overlay": "rgba(0,0,0,0.55)" },
    "accent":   { "type": "color", "value": "#FF6B9D" },
    "font":     { "type": "font",  "value": "serif" },
    "sidebar":  { "type": "color", "value": "rgba(13,15,18,0.75)" },
    "player":   { "type": "preset", "value": "rounded-cover" },
    "card":     { "type": "preset", "value": "glass-morphism" },
    "decorations": { "type": "preset", "value": "music-notes-corner" },
    "lyrics":   { "type": "color", "value": "#FF6B9D" }
  }
}
```

字段说明：
- `id`：唯一标识，预设用 kebab-case，用户自定义用 `custom-{uuid}`
- `mode`：`dark` | `light` | `both`——主题适用的颜色模式
- `slots.{key}.type`：`color` | `gradient` | `image` | `font` | `preset`
- `slots.{key}.value`：对应类型的值
- `slots.{key}.overlay`（可选）：image 类 Slot 的半透明遮罩

### 用户自定义主题

```json
{
  "id": "custom-a1b2c3",
  "name": "我的主题",
  "mode": "dark",
  "slots": {
    "bg":       { "type": "image", "value": "/data/uploads/user_xxx/bg_abc.jpg", "overlay": "rgba(0,0,0,0.4)" },
    "accent":   { "type": "color", "value": "#FF5733" },
    "font":     { "type": "font",  "value": "default" },
    "sidebar":  { "type": "image", "value": "/data/uploads/user_xxx/sidebar_abc.jpg" },
    "player":   { "type": "preset", "value": "rounded-cover" },
    "card":     { "type": "preset", "value": "glass-morphism" },
    "decorations": { "type": "preset", "value": "none" },
    "lyrics":   { "type": "color", "value": "#FF5733" }
  }
}
```

### 存储位置

| 数据 | 存储 | 说明 |
|------|------|------|
| 预设主题定义 | `data/themes/presets.json` | 服务端维护，随代码部署 |
| 预设主题素材 | `data/themes/{id}/` | 图片等静态资源，随代码部署 |
| 用户自定义主题 | `user_preferences.data` JSON 的 `themes` 字段 | 与现有偏好共存，跨设备同步 |
| 用户上传素材 | `data/uploads/{userId}/` | 服务端 `data/` 目录，受 `rsync --exclude 'data'` 保护 |
| 当前激活主题 | `user_preferences.data` JSON 的 `activeTheme` 字段 | `null` = 不使用主题，回退到独立颜色/字体设置 |

### 用户偏好扩展

```json
// user_preferences.data 新增字段
{
  "activeTheme": "jay-chou",           // 当前激活的主题 ID，null = 无主题
  "themes": [                           // 用户自定义的主题列表
    { "id": "custom-a1b2c3", ... }
  ]
}
```

激活主题时：`activeTheme` 设为主题 ID，`wemusic_palette` 等独立 localStorage 值被忽略（主题覆盖一切）。
取消主题时：`activeTheme` 设为 `null`，恢复使用 localStorage 中的独立颜色/字体设置。

## CSS 变量扩展

现有变量体系（12 个变量）不足以表达重度主题。需新增以下变量：

```css
:root {
  /* === 现有（保持不变）=== */
  --bg, --bg-soft, --bg-card, --border, --text, --text-dim,
  --accent, --accent-2, --danger, --radius, --shadow, --font, --font-size;

  /* === 新增：主题背景 === */
  --theme-bg-image: none;              /* 背景图片 URL */
  --theme-bg-overlay: transparent;     /* 背景遮罩层颜色 */
  --theme-bg-size: cover;
  --theme-bg-position: center;

  /* === 新增：侧边栏 === */
  --theme-sidebar-bg: transparent;
  --theme-sidebar-image: none;

  /* === 新增：卡片 === */
  --theme-card-bg: var(--bg-card);     /* 卡片背景（可被 preset 覆盖） */
  --theme-card-border: var(--border);
  --theme-card-radius: var(--radius);
  --theme-card-shadow: var(--shadow);
  --theme-card-backdrop: none;         /* 毛玻璃效果：backdrop-filter */

  /* === 新增：播放器 === */
  --theme-player-cover-radius: 8px;    /* 封面圆角 */
  --theme-player-progress-height: 4px;
  --theme-player-progress-radius: 2px;

  /* === 新增：歌词 === */
  --theme-lyrics-bg: transparent;
  --theme-lyrics-highlight: var(--accent);
}
```

### 背景实现方案

背景图片通过伪元素实现，放在 `body` 或主内容区：

```css
body[data-theme]::before {
  content: '';
  position: fixed; inset: 0; z-index: -1;
  background-image: var(--theme-bg-image);
  background-size: var(--theme-bg-size);
  background-position: var(--theme-bg-position);
}

body[data-theme]::after {
  content: '';
  position: fixed; inset: 0; z-index: -1;
  background: var(--theme-bg-overlay);
}
```

> **注意**：`body::before/after` 的 `position: fixed` 需要确认不会被祖先的 `backdrop-filter` / `transform` 等属性改变 containing block（参照项目 Rule：CSS position:fixed 的 containing block 陷阱）。如 `.dt-root[data-bg="blur"]` 等问题场景，需将背景层移到 `body` 的直接子元素。

### 主题编辑器（Slot-by-Slot 配置）

```
┌──────────────────────────────────┐
│  编辑主题                  [保存] │
│                                  │
│  预览区（实时反映当前配置）        │
│  ┌──────────────────────────┐   │
│  │  (模拟页面外观的缩略预览)   │   │
│  └──────────────────────────┘   │
│                                  │
│  Slot 列表：                     │
│  ┌─ 页面背景 ─────────────────┐  │
│  │ ○ 纯色  ○ 图片  ○ 预设复用  │  │
│  │ [上传图片] 或 从预设选取     │  │
│  │ 遮罩透明度：━━━━━○──── 55%  │  │
│  └────────────────────────────┘  │
│  ┌─ 强调色 ───────────────────┐  │
│  │ 色板选择 / HEX 输入 / 取色器 │  │
│  └────────────────────────────┘  │
│  ┌─ 播放器 ───────────────────┐  │
│  │ 预设：圆角封面 │ 经典 │ 无边框│  │
│  └────────────────────────────┘  │
│  ...（共 8 个 Slot，可折叠）     │
└──────────────────────────────────┘
```

每个 Slot 的编辑器根据类型不同：
- `color` → 色板 + HEX 输入 + 取色器（复用现有 HSL 编辑器）
- `image` → 上传按钮 + 预设图片选择 + 遮罩滑块
- `preset` → 预设选项列表（可视化缩略图）
- `font` → 字体列表（复用现有字体选择器）

### 实时预览

编辑器中修改任意 Slot 时，CSS 变量即时更新，页面背景/强调色等**实时变化**，所见即所得。

## 状态机

| 状态 | 触发 | 表现 |
|------|------|------|
| 无主题 | 默认 / 取消主题 | 使用独立颜色/字体设置，`body[data-theme]` 不存在 |
| 预览预设 | 在主题选择器中 hover/点击预设 | `body[data-theme]` 临时应用，取消或确认前不持久化 |
| 已激活预设 | 确认应用预设 | `activeTheme = presetId`，CSS 变量全量应用 |
| 编辑自定义 | 打开主题编辑器 | `body[data-theme]` 实时更新，编辑器内修改即时反映 |
| 已激活自定义 | 保存自定义主题 | `activeTheme = customId`，slot 配置持久化 |

### 与现有设置的关系（覆盖规则）

```
if (activeTheme) {
  // 主题覆盖一切
  applyThemeSlots(activeTheme.slots);    // 设置所有 CSS 变量
  // 忽略 localStorage 中的独立设置
} else {
  // 回退到现有逻辑
  applyPalette(localStorage.wemusic_palette);
  applyFont(localStorage.wemusic_font);
  applyTheme(localStorage.wemusic_theme);
}
```

用户激活预设主题后，设置面板中的「强调色」「字体」「深色/浅色」选项**灰掉**并标注「由主题控制」，防止用户困惑。

## 预设主题体系

### 预设 Slot 选项

每个 Slot 需要预先设计一组可选值：

**`player` Slot 预设**：
| key | 名称 | 效果 |
|-----|------|------|
| `default` | 经典 | 现有样式不变 |
| `rounded-cover` | 圆角封面 | 封面 `border-radius: 12px` |
| `pill-cover` | 胶囊封面 | 封面 `border-radius: 50%`（圆形） |
| `borderless` | 无边框 | 去除播放器边框 |

**`card` Slot 预设**：
| key | 名称 | 效果 |
|-----|------|------|
| `default` | 经典 | 现有卡片样式 |
| `glass-morphism` | 毛玻璃 | `backdrop-filter: blur()` + 半透明背景 |
| `flat` | 扁平 | 无阴影、直角、纯色背景 |
| `outlined` | 线框 | 仅边框、透明背景 |

**`decorations` Slot 预设**：
| key | 名称 | 效果 |
|-----|------|------|
| `none` | 无 | 不添加装饰 |
| `music-notes-corner` | 音符角落 | 页面四角出现音符 SVG 装饰 |
| `wave-bottom` | 底部波浪 | 页面底部有波浪形装饰线 |
| `star-dust` | 星尘 | 散布的星光粒子（CSS animation） |
| `vinyl-record` | 黑胶唱片 | 侧边栏或角落有黑胶唱片装饰图 |

### 首批预设主题（建议 3 个）

1. **周杰伦 Jay Chou** — 粉色系，手写体，暖色调，音符角落装饰
2. **林俊杰 JJ Lin** — 蓝紫色系，现代感，毛玻璃卡片，星尘装饰
3. **经典 WeMusic** — 绿色系，默认风格，相当于「无主题」但以主题形式呈现

## 边界条件

| 场景 | 处理 |
|------|------|
| 预设主题被删除（服务端更新） | 已激活该预设的用户自动回退到「无主题」+ 上次的独立设置 |
| 用户上传的图片过大 | 前端限制 ≤ 5MB，> 1920px 宽的服务端压缩 |
| 用户上传的图片加载失败 | 回退到对应 Slot 的默认值（纯色背景） |
| 激活主题后切换深色/浅色模式 | 如果主题 `mode` 是 `both` → 应用对应模式的配置；如果仅 `dark` → 强制深色模式（切换按钮灰掉） |
| 同时存在 activeTheme 和自定义颜色 | activeTheme 优先，自定义颜色隐藏但保留（取消主题后恢复） |
| 主题编辑中关闭面板 | 未保存的修改丢弃，恢复到上次保存的状态 |
| 自定义主题数量上限 | 每人最多 10 个自定义主题 |
| N150 部署时主题素材更新 | `data/themes/` 在 `rsync --exclude 'data'` 保护范围内，不会被部署覆盖。预设主题素材随代码在 `public/themes/` 或通过 git 管理 |

## 风险

| 风险 | 等级 | 说明 |
|------|------|------|
| CSS 变量爆炸 | 中 | 从 12 个变量扩展到 ~25 个，需管理好默认值和回退链 |
