# 主题系统（明星主题）

> 状态：📋 规划中 | 日期：2026-07-22 | 更新：2026-07-22（头脑风暴 v2）

## 概述

在现有「自定义主题色」（仅修改 `--accent` 强调色）之上，构建完整的**可组合主题系统**。用户可以选择预设明星主题（一键应用完整视觉包），也可以进入「主题编辑器」逐个配置页面组件/元素，每个组件可从已有预设中选择，也可自行上传素材。

目标视觉效果：页面背景是明星写真 + 毛玻璃叠加层，播放器封面框有定制形状，角落有专属装饰元素，整体氛围与明星风格一致。

### 适用范围

- ✅ **听歌主页面**（首页、搜索、歌单、歌词、推荐等）：主题全量生效
- ❌ **管理面板**（admin）：不适用主题，始终保持默认样式
- ❌ **登录页**：不适用主题
- 🔶 **桌面歌词 PiP 窗口**：不适用主题背景。PiP 窗口应显示**当前播放歌曲的封面**作为背景，而非主题的明星图片

## 与现有系统的关系

| 现有功能 | 新主题系统 |
|---------|----------|
| `--accent` 强调色（25 预设 + 8 自定义） | 主题选中后**覆盖** `--accent`，用户无需单独调色 |
| `--font` 字体选择 | 主题可携带字体偏好，覆盖现有字体设置 |
| 深色/浅色模式 | 每个主题需同时定义深色和浅色两套值（或声明仅支持某种模式） |
| 自定义颜色 CRUD（`/auth/custom-palettes`） | 不受影响，未启用主题时仍可独立使用 |

## 核心设计

### 主题 = 一组 Slot 配置的集合

将页面可定制的视觉区域抽象为 **Slot（槽位）**，每个 Slot 是一个独立的可配置维度。预设主题 = 所有 Slot 填好的完整配置；自定义主题 = 用户逐个 Slot 选择/上传。

### Slot 分级（两级）

为避免用户面对过多配置项产生疲劳，Slot 分为两级：

#### 核心 Slot（5 个，默认展开）

这些是用户感知最强的维度，主题选择器/编辑器中**始终可见**：

| # | Slot | 说明 | 取值类型 |
|---|------|------|---------|
| 1 | **页面背景** (`bg`) | 主内容区背景，纯色 / 渐变 / 图片 + 遮罩 | color / gradient / image |
| 2 | **强调色** (`accent`) | 覆盖 `--accent`，影响按钮、链接、高亮等 | color |
| 3 | **字体** (`font`) | 覆盖 `--font`，影响全站文字 | font（从现有 FONTS 列表选） |
| 4 | **播放器底栏** (`player`) | 封面框形状、进度条样式、按钮风格 | preset（预设样式名） |
| 5 | **卡片样式** (`card`) | 歌单卡片、歌手卡片的背景/边框/阴影/圆角 | preset |

#### 高级 Slot（默认折叠，点击「高级设置」展开）

面向深度定制用户，普通用户跳过不影响体验：

| # | Slot | 说明 | 取值类型 |
|---|------|------|---------|
| 6 | **侧边栏** (`sidebar`) | 左侧导航区背景/装饰 | color / gradient / image |
| 7 | **装饰元素** (`decorations`) | 页面角落花纹、边框装饰、水印图案等 | preset（SVG/CSS 装饰模板） |
| 8 | **歌词面板** (`lyrics`) | 歌词全屏的背景、文字颜色、高亮色 | color / image |
| 9 | **滚动条** (`scrollbar`) | 滚动条颜色/宽度 | color |
| 10 | **表格行** (`row`) | 歌曲列表行 hover/playing 态的背景色、斑马条纹 | color |

### Slot 取值模式

每个 Slot 支持三种来源：

| 来源 | 说明 | 适用 Slot |
|------|------|----------|
| **系统预设** | 官方预制的选项列表（如「毛玻璃卡片」「圆角封面」「音符装饰」） | 所有 Slot |
| **从预设主题复用** | 从某个完整的预设主题中「借」一个 Slot 的值 | 所有 Slot |
| **用户上传** | 用户自行上传图片 / 输入色值 | `bg`、`sidebar`、`lyrics`（image 类 Slot） |

### 双模式工作流

```
预设主题（一键）
  ┌─ 选择预设 → 全部 Slot 自动填充 → 应用
  │
自定义主题（逐个配置）
  └─ 进入编辑器 → 核心 Slot 默认展开 → 可选展开高级 Slot → 实时预览 → 保存
```

## 智能感知（Smart Theme）

### 场景 1：按播放歌曲自动切换主题

用户可启用「智能切换」——当播放某歌手/明星的歌时，自动切换到对应主题。

**条件**：用户必须同时拥有该明星的主题（预设或自定义均可），且主题的 `artistIds` 与当前歌曲歌手匹配。

**数据模型扩展**——预设主题增加 `artistIds` 字段：

```json
{
  "id": "jay-chou",
  "artist": "周杰伦",
  "artistIds": ["qqmusic:0025NhlN2yWrP4", "netease:6452"],
  ...
}
```

`artistIds` 是 `platform:id` 格式的数组，用于跨平台匹配歌手。

**行为**：
- 歌曲切换 → 查找歌手 ID → 匹配主题的 `artistIds` → 找到则自动切换
- 切换时有 **0.5 秒 CSS transition**（背景图 crossfade + 强调色渐变），避免生硬闪烁
- 用户在设置中可关闭「智能切换」（默认开启）
- 手动选择主题后，24 小时内智能切换暂停（尊重用户主动选择，避免「我刚选了 A 你怎么又切回 B」）

**自定义主题的歌手关联**：用户在编辑器里可手动输入歌手名（搜索匹配），主题自动关联歌手 ID。

### 场景 2：主题的日/夜双模式变体

一个主题可以有 `dayVariant`（浅色变体）和 `nightVariant`（深色变体），随系统深色/浅色模式自动切换。

```json
{
  "id": "jay-chou",
  "mode": "both",
  "dayVariant": {
    "slots": {
      "bg": { "type": "image", "value": "jay-light.jpg", "overlay": "rgba(255,255,255,0.3)" },
      "accent": { "type": "color", "value": "#FF8FAB" }
    }
  },
  "nightVariant": {
    "slots": {
      "bg": { "type": "image", "value": "jay-dark.jpg", "overlay": "rgba(0,0,0,0.55)" },
      "accent": { "type": "color", "value": "#FF6B9D" }
    }
  }
}
```

- `mode: "both"` → 主题有两套变体，跟随系统模式切换
- `mode: "dark"` → 仅深色变体，锁定深色模式
- `mode: "light"` → 仅浅色变体，锁定浅色模式
- 切换时同样有 CSS transition 过渡

**自定义主题**：用户可以为自己的主题分别配置日/夜变体，也可以只配一套（mode: dark/light）。

## 背景图片处理

### 处理管线（4 步）

用户上传或预设提供的背景图片，经过以下处理管线：

```
原图 → 用户裁剪（可选）→ 模糊 + 遮罩 → 渐变融合 → 亮度自适应
```

### 1. 模糊 + 遮罩（必须）

图片作为全屏背景时，模糊处理避免干扰前景内容：

```css
body[data-theme]::before {
  content: '';
  position: fixed; inset: 0; z-index: -1;
  background-image: var(--theme-bg-image);
  background-size: cover;
  background-position: center;
  filter: blur(20px) saturate(1.2);
  transform: scale(1.1); /* 避免模糊边缘白边 */
}
body[data-theme]::after {
  content: '';
  position: fixed; inset: 0; z-index: -1;
  background: var(--theme-bg-overlay);
}
```

> **注意**：`filter: blur()` 会使 `body` 成为 fixed 后代的 containing block。参照项目 Rule（CSS position:fixed 的 containing block 陷阱），背景层放在 `body` 直接子元素，其他 fixed 弹窗（modal、右键菜单）不受影响。

### 2. 用户裁剪工具（image 类 Slot 必须）

用户上传背景图后，弹出裁剪对话框：

- 显示一个**按当前窗口比例**的裁剪框（宽高比 = 视口宽高比）
- 用户可拖动裁剪框选择保留区域，缩放调整大小
- 也支持切换比例：`窗口比例` | `16:9` | `4:3` | `1:1` | `自由`
- 裁剪后的坐标（x, y, w, h，百分比）保存在 Slot 配置中：

```json
"bg": {
  "type": "image",
  "value": "/data/uploads/user_xxx/bg.jpg",
  "overlay": "rgba(0,0,0,0.55)",
  "crop": { "x": 10, "y": 5, "w": 80, "h": 90 }
}
```

裁剪通过 CSS `background-position` + `background-size` 或服务端图片处理实现。

### 3. 渐变融合（必须）

背景图边缘用渐变过渡到纯色，消除图片边界突兀感：

```css
body[data-theme] .theme-bg-fade {
  position: fixed; inset: 0; z-index: -1;
  background: radial-gradient(ellipse at center, transparent 60%, var(--theme-bg-fade-color) 100%);
}
```

- `--theme-bg-fade-color` 自动取主题强调色的极暗/极亮版本，或主题 mode 对应的默认底色（深色→`#0d0f12`，浅色→`#f4f6f9`）
- 渐变覆盖范围：中心 60% 透明 → 边缘过渡到纯色

### 4. 亮度自适应（必须）

背景图如果太亮，在深色模式下前景浅色文字会看不清。需要自动调整遮罩深度。

**实现**：用户上传图片后，前端用 Canvas 采样图片平均亮度（0-255）：

| 图片平均亮度 | 深色模式遮罩 | 浅色模式遮罩 |
|-------------|------------|------------|
| 暗（< 85） | `rgba(0,0,0,0.3)`（轻遮罩） | `rgba(255,255,255,0.2)` |
| 中等（85-170） | `rgba(0,0,0,0.5)`（中等） | `rgba(255,255,255,0.1)` |
| 亮（> 170） | `rgba(0,0,0,0.7)`（重遮罩） | `rgba(255,255,255,0.05)`（几乎不加） |

用户仍可在编辑器中手动微调遮罩透明度滑块（0%-100%），覆盖自动值。

### 存储位置

| 数据 | 存储 | 说明 |
|------|------|------|
| 预设主题定义 | `data/themes/presets.json` | 服务端维护，随代码部署 |
| 预设主题素材 | `data/themes/{id}/` | 图片等静态资源，随代码部署 |
| 用户自定义主题 | `user_preferences.data` JSON 的 `themes` 字段 | 与现有偏好共存，跨设备同步 |
| 用户上传素材 | `data/uploads/{userId}/` | 服务端 `data/` 目录，受 `rsync --exclude 'data'` 保护 |
| 当前激活主题 + 智能切换配置 | `user_preferences.data` JSON 的 `activeTheme` + `smartSwitch` 字段 | — |

### 用户偏好扩展

```json
// user_preferences.data 新增字段
{
  "activeTheme": "jay-chou",
  "smartSwitch": {
    "enabled": true,
    "pauseUntil": null
  },
  "themes": [
    { "id": "custom-a1b2c3", ... }
  ]
}
```

- `smartSwitch.enabled`：智能切换开关
- `smartSwitch.pauseUntil`：手动切主题后 24h 内暂停智能切换（ISO 8601 datetime），`null` = 不暂停

## 素材来源策略

三种来源并行，互补：

| 来源 | 做法 | 适用场景 |
|------|------|---------|
| **A. CC0 风格化素材** | 从 Unsplash / Pexels 等获取免费高质量图片，做「风格化」主题（如「复古胶片」「霓虹都市」），不绑定特定明星 | 通用主题、氛围主题 |
| **B. 用户自行上传** | 用户上传自己喜欢的明星图片，责任在用户 | 自定义主题 |
| **C. 官方收集明星高清照** | 从网络公开资源（豆瓣、Google Images、粉丝站）收集明星高清写真，作为预设主题素材 | 明星专属预设主题 |
| **AI 生成风格化插画** | 用 AI 生成「周杰伦风格」的抽象插画/纹理，替代真实照片，版权风险更低 | 不想用真实照片时的替代方案 |

> 预设主题中的明星照片从网络公开资源获取，用于个人非商业项目。不考虑版权风险。

```
│    ┌─ 侧边栏 ─────────────────┐  │
│    │ ...                       │  │
│    └────────────────────────────┘  │
│    ┌─ 装饰元素 ───────────────┐  │
│    │ 预设：无│音符│波浪│星尘    │  │
│    └────────────────────────────┘  │
│    ...（歌词、滚动条、表格行）    │
└──────────────────────────────────┘
```

**预览窗口**：编辑器顶部有一个 380×240 的静态缩略图，模拟页面外观。修改 Slot 时预览图同步更新（客户端渲染）。点击「全屏预览」关闭编辑器直接在真实页面上看效果。

上传背景图后弹出裁剪对话框：

```
│                                  │
│  ┌──────────────────────────┐   │
│  │                          │   │

│  │     └──────────┘         │   │
│  │                          │   │
│  └──────────────────────────┘   │
```

## 状态机

| 状态 | 触发 | 表现 |
|------|------|------|
| 无主题 | 默认 / 取消主题 | 使用独立颜色/字体设置，`body[data-theme]` 不存在 |
| 预览预设 | 在主题选择器中 hover/点击预设 | `body[data-theme]` 临时应用，确认前不持久化 |
| 已激活预设 | 确认应用预设 | `activeTheme = presetId`，CSS 变量全量应用 |
| 编辑自定义 | 打开主题编辑器 | 编辑器预览窗口实时更新，全屏预览时页面实时变化 |
| 已激活自定义 | 保存自定义主题 | `activeTheme = customId`，slot 配置持久化 |
| 智能切换 | 歌曲切换 → 歌手匹配主题 | 自动切主题（0.5s transition），手动切后 24h 暂停 |
| 日/夜切换 | 系统模式切换 / 手动切换深浅色 | dayVariant ↔ nightVariant 平滑过渡 |

### 覆盖规则

```
if (activeTheme) {
  applyThemeSlots(activeTheme.slots);  // 设置所有 CSS 变量
  // 忽略 localStorage 中的独立颜色/字体/深浅设置
  // 设置面板中相关选项灰掉，标注「由主题控制」
} else {
  applyPalette(localStorage.wemusic_palette);
  applyFont(localStorage.wemusic_font);
  applyTheme(localStorage.wemusic_theme);
}
```

## 边界条件

| 场景 | 处理 |
|------|------|
| 预设主题被删除 | 已激活该预设的用户自动回退到「无主题」+ 上次独立设置 |
| 用户上传图片过大 | 前端限制 ≤ 10MB，> 1920px 宽的服务端压缩 |
| 用户上传图片加载失败 | 回退到对应 Slot 默认值（纯色） |
| 背景图太亮 → 文字看不清 | 自动亮度检测 → 调整遮罩深度（见「亮度自适应」） |
| 激活主题后手动切换深浅色 | 如果 mode=`both` → 应用对应变体；mode=`dark/light` → 灰掉切换按钮 |
| 智能切换时目标主题不存在 | 不切，保持当前主题 |
| 手动切主题后智能切换暂停 | 24h 后自动恢复（`pauseUntil`） |
| 主题编辑中关闭面板 | 未保存修改丢弃 |
| 自定义主题数量上限 | 10 个 |
| 管理页面访问时 | `body[data-theme]` 移除，CSS 变量恢复默认 |
| N150 部署时素材更新 | `data/themes/` 受 `rsync --exclude 'data'` 保护，预设素材随代码在 repo 中管理 |

## 风险

| 风险 | 等级 | 说明 |
|------|------|------|
| CSS 变量爆炸 | 中 | 从 12 个扩展到 ~25 个，需管理默认值和回退链 |
| 背景图片性能 | 中 | 大图 + blur filter 可能导致首屏渲染慢，需懒加载 + 渐进式 |
| `position: fixed` 背景层被裁切 | 中 | `filter: blur()` 使 body 成为 fixed 后代的 containing block |
| 智能切换误切 | 中 | 歌手 ID 匹配不准确 / 翻唱版本误匹配 → 需充分测试 |
| 裁剪工具UX | 中 | 拖拽裁剪在移动端体验差，需专门适配 |
| 预设主题素材维护 | 低 | 仅 3 个首批，素材量不大 |
| 用户上传图片安全 | 低 | 校验类型+大小，禁止 SVG（XSS），服务端压缩 |
