# 音量控制统一

## 概述

当前存在三个独立的音量控制入口（WeMusic 底部滑块、B站播放器内部滑块、系统音量），用户每次前后台切换都感到音量不一致。目标：消除 app 内的音量控制，统一到系统音量。

**状态**：🔲 待实现

> 相关需求：「[音量归一化](./音量归一化.md)」— 自动标准化不同 B站源之间的响度（已实现）。

---

## 现状问题

### 三个音量控制

| 控制点 | 位置 | 作用范围 | 可控性 |
|--------|------|----------|--------|
| WeMusic 底部滑块 | 播放器底栏 `.volume-wrap` | 后台 bgAudio（GainNode） | ✅ 完全可控 |
| B站 iframe 音量 | B站播放器内部（cross-origin） | 前台视频音频 | ⚠️ 可 postMessage 设值，但用户能手动改 |
| 系统音量 | OS 级别 | 全局 | ✅ 用户手动 |

### 核心矛盾

两条独立的音频管线各有一套音量乘数，前后台切换时必然有落差。

```
前台：B站 CDN → iframe → [B站音量滑块] → [系统音量] → 耳朵
后台：B站 CDN → bgAudio → [GainNode(_bgVolume × safeGain)] → [系统音量] → 耳朵
```

归一化（`safeGain`）只通过 GainNode 对后台 bgAudio 生效，前台 iframe 不受影响。

## 方案探索

### 各方案概述

| 方案 | 一句话 |
|------|--------|
| **现在** | 底部滑块 + B站滑块 + 系统音量，三个独立控制，前后台切换音量必跳 |
| A | iframe 静音（muted=1），bgAudio 唯一音频源，前后台都走它 |
| B | 两端都设 100%，隐藏 WeMusic 按钮，系统音量接管 |
| C | 同 B，但保留按钮改为只读指示器（显示增益信息） |
| D | 保留现有 cross-fade 切换架构，两端设最大 |
| D+ | D + 前台 iframe 也跟随归一化增益 |
| E | iframe 静音 + 不销毁（隐藏替换销毁），bgAudio 唯一音源 |
| F | 自建 `<video>` 标签 + 服务端代理 B站视频流，彻底甩掉 iframe |

### 关键否决因素

**方案 A**（E 同理）：要求两个跨域不可控的播放器保持同步，但 B站 iframe 对我们是黑盒：
- 能发：`setVolume`（已知，在用）
- 能收：播放器就绪心跳（已知，在用）
- 不能发：seek / pause / resume（API 未公布，社区无文档）
- 不能收：currentTime / playback state（消息格式不明）
- 结构性矛盾 → 否决

**方案 F**：理论最优（一条管线、无跨域、原生 seek / 同步），但：
- 服务端视频流代理 → 带宽 10-50x 音频流
- N100/N150 家宽上传带宽不匹配 → 否决

**方案 B/C**：解决方式偏弱（仍然有前后台音量差），C 额外多了维护成本。

### 选定：方案 D-lite

D 与 D+ 在主场景（后台听歌，归一化覆盖）表现一致。差异仅在前台（次要场景）：
- D+ 的前台归一化依赖 `setVolume(safeGain*100)`，B站播放器音量算法未知，不能保证与 GainNode 线性对齐；
- D 的前台无归一化是可接受的——用户在前台看视频时听到原始音量，符合"看视频"的预期。

D-lite = D + 一个零成本锦上添花：回前台时尝试用归一化增益反推 B站音量值（`setVolume(round(clamp(safeGain, 0, 1)*100))`），设不准也无所谓（现在前台压根没归一化），设准了就白赚一致性。

## 实现计划

### 核心改动

```
删除 _bgVolume（硬编码为 1.0）
  → _gainNode.gain.value = skipNorm ? 1.0 : safeGain
  → 归一化仍正常工作（safeGain 自动随歌曲切换）

删除 WeMusic 底部音量按钮
  → HTML: 移除 .volume-wrap
  → CSS:  移除 #volBar 相关样式
  → JS:   移除 volBar/volBtn 事件绑定

B站 iframe 音量锁死
  → 每次 mountVideoAt 后 setTimeout → _setIframeVolume(100)
  → 回前台时尝试 _setIframeVolume(round(clamp(safeGain, 0, 1)*100))
```

### 不改动的部分

- 前后台切换逻辑（`visibilitychange`、cross-fade）—— 完全不动
- 归一化分析管线（`_fetchGain`、`_applyNormVol`、服务端分析队列）—— 完全不动
- 设置面板的归一化开关 —— 继续工作

### 涉及文件

| 文件 | 改动类型 | 内容 |
|------|----------|------|
| `src/player.js` | 删+改 | 删 `_bgVolume`、`volBar`/`volBtn` 事件；`_applyNormVol` 简化；`_setIframeVolume` 适配 |
| `public/index.html` | 删 | 移除 `.volume-wrap` 整块（6 行） |
| `public/css/style.css` | 删 | 移除 `.volume-wrap` / `#volBar` 样式（~50 行） |
