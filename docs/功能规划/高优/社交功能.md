# 社交功能

> 基于 WeMusic 现有 JWT 用户体系，构建站内社交功能。无需依赖微信登录，用户通过邮箱/用户名注册即可使用全部社交能力。微信登录仅是降低注册门槛的可选增强，非社交功能的前置条件。

## 概述

利用已有用户系统，构建站内社交以提高用户黏性：歌单分享与发现、评论互动、关注/粉丝、动态时间线、一起听。类似小红书等站内社交模型——用户体系自闭环，不依赖第三方账号。

## 功能方向

- **歌单分享与发现**：公开分享歌单，热门歌单/编辑推荐/分类浏览。
- **评论与互动**：对歌曲/歌单评论、点赞、回复。
- **用户主页**：展示公开歌单、最近播放、收藏歌曲、关注/粉丝机制。
- **一起听**：实时同步播放，进度同步，创建听歌房间邀请好友。
- **动态/时间线**：展示好友最近听的歌、创建的歌单、分享的歌曲。

## 技术考量

- 已有 JWT 认证 + 用户注册/登录体系，无需额外接入第三方账号系统即可启动。
- 需要数据库新增表：用户关系表（关注/粉丝）、评论表、动态表。
- WebSocket 用于「一起听」实时同步。
- 建议分阶段：歌单公开分享 + 歌曲分享 → 评论/关注/用户主页 → 动态时间线 → 一起听。
- 后续可接入微信登录（见搁置需求）降低注册门槛，但不影响社交功能本身的实现和上线。
