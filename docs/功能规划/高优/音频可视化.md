# 音频可视化

> 状态：✅ 已完成 · 2026-07-21

## 概述

在歌词详情页底部显示实时音频频谱，增强沉浸感。纯前端实现，使用 Web Audio API 的 `AnalyserNode`。

## 实现细节

### 数据链路

```
bgAudio → GainNode → AnalyserNode → EQ → destination
                            ↑
              player.js: getAnalyserNode()
                            ↑
              lyrics.js: startSpectrum() → _drawSpectrum() (rAF 循环)
```

### 频谱计算（`lyrics.js` `_computeBars`）

1. **对数频率映射**：40Hz–16kHz 按指数均匀分配 64 根柱子（`minHz * exp(logRange * i/(n-1))`）
2. **峰值保持衰减**：快速跟随上升（系数 0.4），慢速指数衰减（每帧 × 0.92 ≈ 0.5 秒回落到底）
3. **空间平滑**：1:2:1 加权平均，消除帧间抖动但不抹平柱间差异
4. **Canvas HiDPI**：同步 `devicePixelRatio`，确保 Retina 屏幕不模糊

### 三种视觉样式

| 样式 | key | 特征 |
|------|-----|------|
| 渐变镜像 | `gradient`（默认） | 圆角柱 + 垂直渐变（顶亮底暗）+ 底部倒影 |
| 霓虹辉光 | `glow` | 柱体 `shadowBlur` 外发光 + 顶部白色高亮线 |
| 波形填充 | `wave` | `quadraticCurveTo` 平滑曲线 + 渐变填充 + 描边辉光 |

### 设置（`settings.js`）

- 开关：`localStorage.wemusic_spectrum`（`1`/`0`）
- 样式：`localStorage.wemusic_spectrum_style`（`gradient`/`glow`/`wave`）
- 移动端（≤ 720px）自动禁用

### Canvas 同步

`_drawSpectrum` 首帧检查 `canvas.offsetWidth * devicePixelRatio`，不一致时更新 `canvas.width`/`height` 属性。避免 CSS `width: 100%` 拉伸造成的模糊。

## 已知限制

- 展开视频（B 站 iframe）不可用：音频由 iframe 播放器独立输出，Web Audio 管线无法介入
- FFT size = 256 → 128 个有效 bins，对短音频响应略有延迟
