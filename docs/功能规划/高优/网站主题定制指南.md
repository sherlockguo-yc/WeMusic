## 1. 场景差异：Electron vs Web

| | Electron（本项目方案） | Web 网站 |
|---|---|---|
| 资源修改能力 | 解包 ASAR，完全控制 | 只能注入 CSS/JS，不能改服务端 HTML |
| 持久性 | 写入文件，永久生效 | 注入脚本随页面加载生效 |
| 安装方式 | Shell 脚本 + 原子替换 | UserScript / 浏览器扩展 / Stylus |
| 版本稳定性 | 选择器锁定在特定版本 | 选择器可能随网站更新而变化 |
| 备份恢复 | 文件级备份 | 不需要（关掉脚本即恢复） |

**核心结论**：CSS 变量覆盖、暗色模式、字体策略、视觉技巧、状态监听——这些方法论 100% 可复用。不可复用的只有 Electron 特有的打包/安装/备份体系。

## 2. 选择器分析：Web 上的挑战

### 与 Electron 的相同之处

目标网站如果用了 CSS Modules、CSS-in-JS 或动态 class 名，选择器同样不稳定。需要使用：

- **属性部分匹配**：`[class*="_status_"]`、`[class*="trailingStatus"]`
- **data 属性**：`[data-testid="sidebar"]`（如果网站保留了测试属性）
- **层级关系**：`.parent > .child`（在固定结构中才稳定）
- **ARIA 属性**：`[role="dialog"]`、`[aria-label="搜索"]`

### 与 Electron 的不同之处

网站无法解包分析所有源码。只能通过 DevTools 查看**编译后的** DOM 和 CSS。

**分析流程**：

1. 打开目标网站 → F12 DevTools
2. Elements 面板：找到 5~10 个结构稳定的区域（顶栏、侧栏、主内容区、输入框等）
3. Styles 面板：在 `:root` 或 `<html>` 上查看所有 `--` 开头的变量
4. Computed 面板：验证哪些变量实际被使用
5. 在 Console 中粘贴试改：
   ```js
   document.documentElement.style.setProperty('--bg-primary', '#dceef9');
   ```
   看效果是否正确。

### 选择器稳定性评级（Web 版）

| 策略 | 稳定性 | 说明 |
|------|--------|------|
| CSS 变量覆盖 | ★★★★★ | 最优，不依赖 DOM 结构 |
| `[data-testid]` | ★★★★☆ | 如果网站为测试保留了这些属性 |
| `[aria-label]` | ★★★★☆ | 无障碍属性通常稳定 |
| `[role="..."]` | ★★★★☆ | 语义角色很少变 |
| 固定 class 名 | ★★★☆☆ | 取决于前端框架和重构频率 |
| class 部分匹配 | ★★☆☆☆ | CSS Modules 的 hash 会变，但前缀可能保留 |
| `:nth-child()` | ★☆☆☆☆ | 极其脆弱 |

## 3. CSS 变量覆盖策略

### 三步法

**第一步：收集变量清单**

在 DevTools Console 中执行：

```js
// 列出页面中所有 CSS 自定义属性
const allVars = [];
for (const sheet of document.styleSheets) {
  try {
    for (const rule of sheet.cssRules) {
      if (rule.style) {
        for (const prop of rule.style) {
          if (prop.startsWith('--')) allVars.push(prop);
        }
      }
    }
  } catch(e) {}
}
console.log([...new Set(allVars)].sort());
```

**第二步：分类变量**

按前缀分组：
```
--vscode-*  → VS Code 系
--wb-*      → WorkBuddy 系
--cb-*      → CodeBuddy 系
--td-*      → TDesign 系
--color-*   → 通用颜色系
```

**第三步：编写覆盖块**

```css
:root {
  /* 按系列分组，便于维护 */
  /* ---- 背景 ---- */
  --bg-primary: #dceef9 !important;
  --bg-secondary: #f1f7fb !important;
  --bg-card: #ffffff !important;

  /* ---- 文字 ---- */
  --text-primary: #263d50 !important;
  --text-secondary: #617b8e !important;
  --text-dim: #7b91a1 !important;

  /* ---- 强调色 ---- */
  --accent: #2d83c6 !important;
  --accent-hover: #1f73b5 !important;
  --accent-active: #17649e !important;

  /* ---- 边框 ---- */
  --border-default: #aac9dc !important;
  --border-subtle: #d0e2ed !important;
  --border-focus: #3187c7 !important;

  /* ---- 阴影 ---- */
  --shadow-card: 0 1px 4px rgba(43, 94, 129, .12) !important;
  --shadow-dialog: 0 6px 18px rgba(40, 78, 104, .18) !important;

  /* ---- 圆角 ---- */
  --radius-sm: 4px !important;
  --radius: 6px !important;
  --radius-lg: 9px !important;
  --radius-xl: 12px !important;

  /* ---- 字体 ---- */
  --font-ui: Tahoma, "Microsoft YaHei", "PingFang SC", Arial, sans-serif !important;
  --font-mono: "SFMono-Regular", Menlo, Consolas, monospace !important;
}
```

### 验证方法

写完后在 Console 逐条验证：

```js
// 1. 检查变量是否被读取
getComputedStyle(document.documentElement).getPropertyValue('--bg-primary');
// 预期输出: "#dceef9"

// 2. 检查一个使用了该变量的元素是否正确渲染
const el = document.querySelector('.some-card');
getComputedStyle(el).backgroundColor;
// 预期输出背景色
```

## 4. 暗色模式适配

### 检测网站的暗色模式触发方式

在 DevTools 中切换网站自带的暗色模式开关，观察变化：

| 常见触发方式 | 如何检测 |
|-------------|---------|
| `<html data-theme="dark">` | 检查 `html` 的 `data-theme` 属性 |
| `<body class="dark">` | 检查 `body` 的 classList |
| `prefers-color-scheme: dark` | 系统级，看 `@media` 查询 |
| `<html class="dark">` | 检查 `html` 的 classList |

### 编写暗色覆盖块

```css
/* 根据网站的触发方式选择对应的选择器 */
html[data-theme="dark"],
body.dark {
  --bg-primary: #102a3a !important;
  --bg-secondary: #0f202c !important;
  --bg-card: #142936 !important;

  --text-primary: #d8e9f3 !important;
  --text-secondary: #91abba !important;
  --text-dim: #718b9a !important;

  --border-default: #315b71 !important;
  --border-subtle: #203d4d !important;
  --border-focus: #49a9de !important;

  --shadow-card: 0 1px 4px rgba(0, 0, 0, .3) !important;
  --shadow-dialog: 0 8px 22px rgba(0, 0, 0, .48) !important;
}
```

### 明→暗色映射规律（备忘）

| 属性类型 | 明色 | 暗色 | 规律 |
|---------|------|------|------|
| 背景 | `#fff` ~ `#dceef9` | `#0b1822` ~ `#142936` | 深蓝黑系 |
| 文字 | `#263d50` | `#d8e9f3` | 反转，HSL 中 L 值对调 |
| 边框 | `#aac9dc` | `#315b71` | 降低亮度、降低饱和度 |
| 阴影 | `rgba(蓝, .12)` | `rgba(黑, .3)` | 从明亮弥散 → 深色穿透 |
| 高光线 | `inset 0 1px #fff` | `inset 0 1px rgba(蓝, .16)` | 降低不透明度 |

## 5. 字体替换规则

### 不破坏代码和图标的核心原则

❌ **错误**：
```css
* { font-family: Tahoma !important; }
```
→ 图标变乱码、代码对齐错乱

✅ **正确**：
```css
/* 排除列表 */
:is(div, span, p, a, button, input, textarea, select, h1, h2, h3, h4, h5, h6, li, th, td, label)
  :not([class*="icon"])
  :not([class*="codicon"])
  :not(.katex):not(.katex *)
  :not(pre):not(code)
  :not([class*="CodeMirror"])
  :not([class*="monaco"]) {
  font-family: var(--font-ui) !important;
}

/* 代码区保持等宽 */
pre, code, [class*="CodeMirror"], [class*="monaco"] {
  font-family: var(--font-mono) !important;
}

/* 图标字体原样保留 */
[class*="icon"], [class*="codicon"] {
  font-family: inherit !important;
}
```

### 排除列表维护

每次发现被误伤的图标/代码区域，加入排除列表：
- `codicon` — VS Code / CodeBuddy 图标
- `iconfont` / `IconFont` / `[class*="icon"]` — 通用图标
- `katex` — 数学公式
- `CodeMirror` / `monaco` — 代码编辑器
- `pre` / `code` — 代码块

## 7. 状态监听与交互增强

### 7.1 通用模板：MutationObserver 监听

```js
(() => {
  "use strict";

  // ---- 配置区 ----
  const TARGET_SELECTOR = ".task-card";           // 要监听的元素
  const STATE_SELECTOR = '[class*="_status_"]';    // 状态指示器
  const ACTIVE_STATES = new Set(["running", "pending"]);

  // ---- 状态存储（用 data-id 或 DOM 路径作 key） ----
  const stateMap = new Map();

  function getKey(el) {
    // 优先用业务 ID，退而求其次用 DOM 路径
    return el.dataset.id || el.dataset.taskId
        || el.closest("[data-id]")?.dataset.id
        || el;
  }

  function readState(el) {
    const status = el.querySelector(STATE_SELECTOR);
    if (!status) return "unknown";
    const text = status.textContent.trim().toLowerCase();
    if (/完成|done|complete/.test(text)) return "completed";
    if (/失败|错误|error|fail/.test(text)) return "failed";
    if (/运行|处理|working|running/.test(text)) return "running";
    if (/等待|排队|pending/.test(text)) return "pending";
    return "other";
  }

  // ---- 防抖 ----
  let lastActionAt = 0;
  const DEBOUNCE_MS = 1200;

  function onStateChange(el, from, to) {
    const now = Date.now();
    if (now - lastActionAt < DEBOUNCE_MS) return;
    lastActionAt = now;

    // 在此处定义状态变化的响应
    if (ACTIVE_STATES.has(from) && to === "completed") {
      console.log(`[Theme] 任务完成: ${getKey(el)}`);
      // playSound("complete");
      // showNotification("任务完成");
    }
  }

  // ---- 扫描 ----
  function scan() {
    const seen = new Set();
    for (const el of document.querySelectorAll(TARGET_SELECTOR)) {
      const key = getKey(el);
      seen.add(key);
      const current = readState(el);
      const previous = stateMap.get(key);
      stateMap.set(key, current);
      if (previous && previous !== current) {
        onStateChange(el, previous, current);
      }
    }
    // 清理已移除的元素
    for (const key of stateMap.keys()) {
      if (!seen.has(key)) stateMap.delete(key);
    }
  }

  // ---- 双层监听 ----
  let scanTimer = 0;
  function scheduleScan() {
    clearTimeout(scanTimer);
    scanTimer = setTimeout(scan, 60);
  }

  function start() {
    scan(); // 首次扫描建基线，不触发动作
    // MutationObserver：实时
    new MutationObserver(scheduleScan).observe(document.body, {
      childList: true,
      subtree: true,
      characterData: true,
      attributes: true,
      attributeFilter: ["class"]
    });
    // setInterval：兜底（1500ms）
    setInterval(scan, 1500);
  }

  if (document.readyState === "loading") {
    document.addEventListener("DOMContentLoaded", start, { once: true });
  } else {
    start();
  }
})();
```

### 7.2 音效注入

```js
const successSound = new Audio("data:audio/wav;base64,..."); // 或 URL
successSound.preload = "auto";

function playSound(type) {
  const enabled = localStorage.getItem("THEME_SOUND_ENABLED") !== "false";
  if (!enabled) return;
  successSound.volume = 0.8;
  successSound.play().catch(() => {});
}
```

> 注意：现代浏览器要求用户交互后才能播放音频。可在一个用户点击事件中先 `audio.play().catch()` 解锁。

### 7.3 图片加载失败兜底

```js
// 对目标区域的所有图片添加 onerror 兜底
function instrumentImages(root = document) {
  for (const img of root.querySelectorAll("img:not([data-fallback])")) {
    img.dataset.fallback = "1";
    img.addEventListener("error", function onErr() {
      // 替换为占位元素而非隐藏
      const ph = `<span class="img-fallback" style="
        display:inline-flex;align-items:center;justify-content:center;
        width:${this.width||48}px;height:${this.height||48}px;
        background:var(--bg-secondary);border-radius:var(--radius);
        color:var(--text-dim);font-size:12px;
      ">🖼</span>`;
      this.outerHTML = ph;
    }, { once: true });
  }
}

// 配合 MutationObserver 覆盖动态加载的图片
new MutationObserver(() => instrumentImages())
  .observe(document.body, { childList: true, subtree: true });
```

## 8. 注入路径对比

| 方案 | 安装难度 | CSS | JS | 持久化设置 | 分发 | 适用场景 |
|------|---------|-----|----|-----------|------|---------|
| **UserScript** (Tampermonkey) | 低 | ✅ | ✅ | GM_setValue | 手动分享脚本 | 个人使用、快速原型 |
| **Stylus** | 极低 | ✅ | ❌ | 无 | 分享到 userstyles.org | 纯视觉换肤 |
| **Chrome Extension** | 中 | ✅ | ✅ | chrome.storage | Chrome Web Store | 公开发布、团队分发 |
| **Bookmarklet** | 极低 | ✅ | ✅ | 无 | URL 分享 | 临时演示 |
| **DevTools Snippets** | 极低 | ✅ | ✅ | 无 | 无 | 本地开发调试 |

### 选择建议

- **只想改颜色** → Stylus
- **需要音效/动画/交互增强** → UserScript
- **要公开发布给其他人用** → Chrome Extension
- **调试/验证想法** → DevTools Console 直接粘贴

## 9. 完整 UserScript 模板

```js
// ==UserScript==
// @name         QQ2008 Theme for [Target Site]
// @namespace    https://github.com/yourname/qq2008-theme
// @version      1.0
// @description  QQ2008 retro theme for [Target Site]
// @author       You
// @match        https://target-site.com/*
// @grant        GM_addStyle
// @grant        GM_getValue
// @grant        GM_setValue
// @run-at       document-start
// ==/UserScript==

(() => {
  "use strict";

  // ============================================================
  //  Part 1: CSS Injection
  // ============================================================
  const themeCSS = `
    /* ---- Reset CSS Variables ---- */
    :root {
      --bg-primary: #dceef9 !important;
      --bg-secondary: #f1f7fb !important;
      --bg-card: #ffffff !important;
```

## 12. Chrome 内置工具 `chrome.devtools.inspectedWindow` 介绍

> 这是 Chrome DevTools Extension API，专为开发工具类扩展设计，功能远超普通 content script。核心区别：在**宿主页面上下文**中执行代码，`this` 指向宿主页面的 `window`。

### 核心 API

```js
// 1. 执行 JS（在宿主页面上下文中）
chrome.devtools.inspectedWindow.eval(
  "document.querySelectorAll('.card').length",
  (result, error) => {
    if (error) console.error(error);
    else console.log(`找到 ${result} 张卡片`);
  }
);

// 2. 获取当前面板选中的元素（$0）
chrome.devtools.inspectedWindow.eval("$0.className", console.log);

// 3. 重新加载页面（注入代码在 page load 前生效）
chrome.devtools.inspectedWindow.reload({ injectedScript: "..." });

// 4. 获取资源内容
chrome.devtools.inspectedWindow.getResources((resources) => {
  resources.forEach(r => console.log(r.url));
});

// 5. 监听导航
chrome.devtools.inspectedWindow.onResourceAdded.addListener((resource) => {
  console.log("新资源:", resource.url);
});
```

### 用它的场景

| 需求 | 普通 content script 能否实现 | 为什么需要 this API |
|------|---------------------------|-------------------|
| 获取页面 CSS 变量列表 | ✅ 可以 | — |
| 注入样式 | ✅ 可以 | — |
| 执行 `$0`（选中元素） | ❌ 不行 | `$0` 只在 DevTools 环境存在 |
| 在宿主 window 上下文中运行 | ❌ content script 是隔离环境 | `eval` 的 `this` 是宿主 window |
| 访问宿主全局变量 | ❌ 需要 `window.postMessage` | `eval("window.__SECRET__")` 直接读 |
| 页面加载前注入脚本 | ❌ `document_start` 有竞态 | `reload({ injectedScript })` 保证在 page load 前执行 |

### 典型用法：做一个"主题编辑器"DevTools 面板

```
theme-editor-devtools/
├── manifest.json
├── devtools.html          # DevTools 入口（创建面板）
├── panel.html             # 面板 UI
├── panel.js               # 面板逻辑（调用 eval 修改页面样式）
└── qq2008-preset.json     # QQ2008 预设色板
```

**`devtools.html`**：
```html
<script>
  chrome.devtools.panels.create("QQ2008 Theme", "", "panel.html");
</script>
```

**`panel.js`**（核心）：
```js
// 从页面读取 CSS 变量
function readVars() {
  chrome.devtools.inspectedWindow.eval(`
    [...new Set(
      [...document.styleSheets]
        .flatMap(s => { try { return [...s.cssRules] } catch { return [] } })
        .flatMap(r => r.style ? [...r.style].filter(p => p.startsWith('--')) : [])
    )].sort().map(v => [v, getComputedStyle(document.documentElement).getPropertyValue(v)])
  `, (result) => {
    if (result) renderVariableList(result);
  });
}

// 修改页面 CSS 变量（实时预览）
function setVar(name, value) {
  chrome.devtools.inspectedWindow.eval(`
    document.documentElement.style.setProperty('${name}', '${value}');
  `);
}

// 应用 QQ2008 预设
function applyPreset(presetColors) {
  chrome.devtools.inspectedWindow.eval(`
    const preset = ${JSON.stringify(presetColors)};
    Object.entries(preset).forEach(([k, v]) => {
      document.documentElement.style.setProperty(k, v, 'important');
    });
  `);
}

// 导出为 UserScript
function exportAsUserScript() {
  chrome.devtools.inspectedWindow.eval(`
    JSON.stringify(
      [...document.documentElement.style]
        .filter(p => p.startsWith('--') && document.documentElement.style.getPropertyPriority(p) === 'important')
        .map(p => [p, document.documentElement.style.getPropertyValue(p)])
    )
  `, (result) => {
    if (result) {
      const vars = JSON.parse(result);
      const css = `:root {\n${vars.map(([k,v]) => `  ${k}: ${v} !important;`).join('\n')}\n}`;
      console.log(css); // 复制粘贴到 UserScript
    }
  });
}
```

### 和普通 content script 的关系

```
普通 content script        chrome.devtools.inspectedWindow
──────────────────────     ──────────────────────────────
运行在隔离沙箱              运行在宿主页面上下文
无法访问 DevTools API       可访问 DevTools API
适用于"注入后就不管了"       适用于"交互式开发工具"
通过 DOM 操作间接改样式      通过 eval 直接改宿主全局变量
```

## 13. CSS 颜色体系设计

### 13.1 颜色阶梯：为什么需要多层 shade

仅设一个 `--accent` 是不够的。真实 UI 需要 4~5 层色阶：

```
--accent-lightest  → 浅色背景（hover 时微变色）
--accent-light     → 选中背景
--accent           → 默认强调色（按钮、链接）
--accent-dark      → hover 态
--accent-darkest   → active/pressed 态
```

**反例**：只设 `--accent` 和 `--accent-hover`，但列表项的选中态、输入框聚焦环、标签背景色都用了同一颜色，视觉层次扁平。

**正例**（本项目做法）：
```css
:root {
  --wb-bg-primary: #ffffff;       /* 卡片/内容区 */
  --wb-bg-secondary: #f1f7fb;     /* 次级背景 */
  --wb-bg-tertiary: #e4f0f7;      /* 三级背景 */
  --wb-bg-hover: #d9ebf6;         /* 悬停态 */
  --wb-bg-active: #c9e3f3;        /* 激活/选中态 */

  --wb-text-primary: #263d50;     /* 正文 */
  --wb-text-strong: #183a54;      /* 强调文字 */
  --wb-text-medium: #425f74;      /* 中等文字 */
  --wb-text-secondary: #617b8e;   /* 次要文字 */
  --wb-text-tertiary: #7b91a1;    /* 最弱文字 */

  --wb-border-default: #aac9dc;   /* 常规边框 */
  --wb-border-subtle: #d0e2ed;    /* 淡边框 */
  --wb-border-focus: #3187c7;     /* 聚焦边框 */
}
```

### 13.2 颜色语义化 vs 颜色值

❌ **按颜色值命名**（暗含具体颜色，换色系后语义错乱）：
```css
--color-blue: #2d83c6;
--color-light-blue: #dceef9;
```

✅ **按语义命名**（描述 UI 角色，换色系时只改值不改名）：
```css
--accent: #2d83c6;
--bg-sidebar: #dceef9;
```

### 13.3 图标颜色必须通过变量

图标用 `currentColor` 让颜色跟随文字：
```css
/* SVG 模板 */
<svg stroke="currentColor" fill="none">...</svg>

/* CSS 中控制颜色 */
.icon { color: var(--wb-icon-primary); }
.icon-muted { color: var(--wb-icon-muted); }
```

如果图标内部有多个颜色（如彩色 logo），用 CSS `filter` 或直接在 SVG 中用 CSS 变量。

### 13.4 渐变色系统

本项目大量使用 `linear-gradient` 模拟 3D 效果。要点：

- **顶部高光**：渐变上半部比下半部亮（`#f9fdff` → `#d7ecf8`）
- **inset 高光线**：`inset 0 1px rgba(255,255,255,.9)` 模拟顶光
- **多层 box-shadow**：外层做投影、内层做高光，组合起来才有立体感

```css

  background: linear-gradient(180deg, #f9fdff, #d7ecf8);  /* 渐变体感 */

    inset 0 1px rgba(255,255,255,.9),                     /* 顶部高光线 */
    0 1px 2px rgba(43,94,129,.12);                        /* 底部投影 */
```

## 15. 选择器优先级实战

### 15.1 为什么需要 `!important`

注入的主题 CSS 在页面原始 CSS **之后**加载，理论上自然有更高优先级。但以下情况会失效：

1. 原始 CSS 使用了更高特异性的选择器（如 `#root .card` vs 你的 `.card`）
2. 原始 CSS 本身就用了 `!important`
3. 内联样式（`style="..."` 属性）

因此**所有变量覆盖和视觉重写都加 `!important`** 是安全策略。

### 15.2 特异性战争与如何避免

当你发现自己的 `!important` 被另一个 `!important` 覆盖时，不要无脑叠特异性。按以下顺序排查：

1. **注入时机**：CSS 是否在目标样式表之后加载？（检查 `<head>` 中 `<style>` 的位置）
2. **选择器特异性**：`body.dark .card` > `.card`，调整你的选择器匹配其特异性
3. **`!important` 的来源**：是否是内联样式？（内联 `!important` 无法被外部 CSS 覆盖）
4. **Shadow DOM**：如果目标在 Shadow DOM 内，外部 CSS 无法穿透（除非用 `::part()` 或 `adoptedStyleSheets`）

### 15.3 选择器分组技巧

同类视觉区域用逗号分组，减少重复代码：

```css
/* ✅ 好：四个 topbar 变体共享样式 */
.conversation-list-topbar,
.claw-agent-chat-topbar,
.workbuddy-topbar,
.teams-top-bar {
  background: linear-gradient(...) !important;
  border-bottom: 1px solid #1265aa !important;
}

/* ✅ 好：用 :where() 降低特异性（不影响后面覆盖） */
.topbar :where(button, svg) {
  color: #ffffff !important;
}
```

`:where()` 的特殊性为 0，不会干扰后续更具体的选择器。

## 16. 状态机驱动的 UI 响应

### 16.1 设计原则

监听页面 DOM 变化来触发动作（音效/动画/通知），核心是**状态机 + 去重**：

```
状态变化检测 = 当前状态 !== 上次记录的状态
触发条件 = 从活跃态集合 → 目标态
```

### 16.2 readState() 的多层 fallback

页面状态可能以多种形式呈现（图标、CSS class、文字、属性），需要**按优先级逐层检测**：

```js
function readState(card) {
  // 第 1 层：专用 CSS class（最可靠）
  if (card.querySelector(".status-error")) return "failed";
  if (card.querySelector(".status-success")) return "working";

  // 第 2 层：CSS class 部分匹配（CSS Modules 动态类名）
  if (card.querySelector('[class*="_trailingStatus_"] path[fill="#F64041"]')) return "failed";

  // 第 3 层：文字内容正则（中文/英文多语言）
  const text = card.querySelector('[class*="_status_"]')?.textContent || "";
  if (/已完成|完成|completed|complete|done/.test(text)) return "completed";
  if (/失败|错误|failed|error/.test(text)) return "failed";

  // 第 4 层：推断（某些状态下没有明确的状态标记）
  if (card.querySelector(".trailing-icon")) return "pending";

  // 第 5 层：默认（某些组件完成后会移除所有状态标记）
  return "completed";
}
```

### 16.3 首次扫描静默（基线原则）

**这是最容易遗漏的 bug**：脚本加载时页面已有内容（历史任务、已有问题），如果直接根据当前状态触发动作，会导致"补播"。

```js
let initialized = false;
let previousState = false;

function scan() {
  const currentState = checkSomething();

  if (!initialized) {
    // 只记录基线，不触发任何动作
    initialized = true;
    previousState = currentState;
    return;
  }

  if (currentState && !previousState) {
    triggerAction(); // 仅新变化才触发
  }
  previousState = currentState;
}
```

### 16.4 防抖 vs 节流

- **防抖（debounce）**：事件停止触发 N ms 后才执行——用于"等 DOM 稳定后再扫描"
- **节流（throttle）**：每 N ms 最多执行一次——用于"限制音效频率"

本项目同时用了两者：
```js
// 防抖：MutationObserver 回调 → 60ms 后扫描（等 React 渲染完）
function scheduleScan() {
  clearTimeout(scanTimer);
  scanTimer = setTimeout(scan, 60);
}

// 节流：两次音效之间至少 1200ms
if (Date.now() - lastPlayedAt < 1200) return;
```

### 16.5 DOM 节点被重建时保持状态

React/Vue 等框架可能销毁并重建 DOM 节点（key 变化、列表重排）。不能用 DOM 引用追踪状态。

✅ **用业务 ID 作 key**：
```js
function getCardKey(card) {
  // 从最近的父级取业务 ID
  const host = card.closest("[data-conversation-id]");
  return host?.getAttribute("data-conversation-id") || card;
}
```

❌ **用 DOM 引用作 key**：
```js
const stateMap = new WeakMap(); // 节点重建后 WeakMap 丢失
stateMap.set(cardElement, state);
```

### 16.6 兜底定时扫描的必要性

`MutationObserver` 在极端情况下可能漏掉变化（批量更新合并回调、Shadow DOM 内部变化）。**1500ms 的 `setInterval` 兜底成本极低（一次 `querySelectorAll`），但能防止漏检。**

## 17. 持久化用户偏好

### 17.1 localStorage 存储模式

```js
const ENABLED_KEY = "THEME_SOUND_ENABLED";
const VOLUME_KEY = "THEME_SOUND_VOLUME";

function isEnabled() {
  // "false" 字符串才禁用，缺失/null 视为启用
  return localStorage.getItem(ENABLED_KEY) !== "false";
}

function getVolume() {
  const raw = localStorage.getItem(VOLUME_KEY);
  if (raw === null) return 0.8; // 默认值
  const val = Number(raw);
  return Number.isFinite(val) && val >= 0 && val <= 1 ? val : 0.8;
}
```

关键点：
- 用 `!== "false"` 而非 `=== "true"`（缺失时默认启用，更符合预期）
- 数值型做范围校验（`>=0 && <=1`），防止脏数据
- `Number.isFinite()` 比 `!isNaN()` 更严格（拒绝 `Infinity`）

### 17.2 暴露全局控制 API

```js
window.__THEME_API__ = {
  setEnabled(bool) {
    localStorage.setItem(ENABLED_KEY, String(Boolean(bool)));
  },
  setVolume(vol) {
    const v = Math.min(1, Math.max(0, Number(vol) || 0));
    localStorage.setItem(VOLUME_KEY, String(v));
  }
};
```

用户可以在 DevTools Console 中调用：
```js
window.__THEME_API__.setEnabled(false);  // 关闭音效
window.__THEME_API__.setVolume(0.3);     // 调低音量
```

### 17.3 浏览器扩展中的替代方案

如果用 Chrome Extension，可用 `chrome.storage.sync` 替代 `localStorage`，实现跨设备同步：

```js
// 写入
chrome.storage.sync.set({ themeSoundEnabled: false });

// 读取
chrome.storage.sync.get(["themeSoundEnabled"], (result) => {
  const enabled = result.themeSoundEnabled !== false;
});
```

## 18. 音效与多媒体处理

### 18.1 Audio 对象的正确初始化

```js
const audio = new Audio("./assets/sound.wav");
audio.preload = "auto"; // 预加载，避免首次播放延迟
```

### 18.2 浏览器自动播放限制

现代浏览器禁止未经用户交互的自动播放。解决策略：

**策略 A：利用首次用户交互解锁**
```js
let audioUnlocked = false;

document.addEventListener("click", function unlock() {
  if (audioUnlocked) return;
  // 播放一段静音来解锁 AudioContext
  const silent = new Audio("data:audio/wav;base64,..."); // 极短静音
  silent.play().then(() => { audioUnlocked = true; });
}, { once: true });
```

**策略 B：静默失败（本项目做法）**
```js
audio.play().catch((error) => {
  console.warn("播放失败，可能尚未获得音频播放权限。", error);
});
// 不阻塞 UI，用户点击页面后下次就能播放了
```

### 18.3 多音源管理

当有多个音效时，播放新音效前停止旧音效：

```js
function play(audio) {
  // 停止所有音效
  for (const a of allAudios) {
    a.pause();
    a.currentTime = 0;
  }
  // 播放目标音效
  audio.volume = getVolume();
  audio.play().catch(() => {});
}
```

### 18.4 文件命名策略

本项目中的 `qq-failure.wav` 实际用于"咳嗽"提示音（AI 选择题），但保留了旧文件名。原因：**避免升级时额外迁移资源文件**——用户可能已有旧版本安装，改文件名会导致旧引用 404。

> **教训**：一旦资源文件名对外发布，就应视为不可变的 API。功能变更时只改代码逻辑，不改文件名。

## 19. 兼容性与渐进增强

### 19.1 选择器兼容性检查清单

安装/启用主题前，验证关键选择器是否存在：

```js
// 在 Console 中快速检查
const criticalSelectors = [
  ".conversation-list-logo-row",
  ".user-menu-trigger--workbuddy",
  ".wb-scene-tabs",
  ".wb-home-composer__input-slot",
  '[class*="topRightSlotStandalone"]'
];

const missing = criticalSelectors.filter(s => !document.querySelector(s));
if (missing.length > 0) {
  console.warn("兼容性警告：以下选择器在当前版本中不存在：", missing);
}
```

### 19.2 多版本并存策略

如果目标应用有多个版本在运行，不要试图用一个主题 CSS 兼容所有版本。策略：

- **锁定版本**：主题 CSS 针对特定版本编写，版本号写入注释
- **兼容性检查**：安装前检查关键选择器，不匹配即停
- **版本升级后重建**：不从旧备份恢复，基于新版原包重新构建

### 19.3 渐进增强的 CSS 写法

对于不确定是否存在的选择器，用 `:where()` 降低特异性 + 组合多个可能的选择器：

```css
/* 覆盖所有可能的顶栏选择器（多版本兼容） */
:where(.topbar, .header, .navbar, [class*="top-bar"]) {
  background: linear-gradient(...) !important;
}
```

### 20.1 快速验证变量覆盖

```js
// 一键查看当前所有被覆盖的 CSS 变量
const overridden = [...document.documentElement.style]
```

### 20.2 对比注入前后

```js
// 在注入 CSS 前后各截一张 computed style 快照
function snapshotStyles(selector) {
  const el = document.querySelector(selector);
  if (!el) return null;
  const styles = getComputedStyle(el);
  const keys = ['backgroundColor', 'color', 'fontFamily', 'fontSize', 'borderRadius', 'boxShadow'];
  return Object.fromEntries(keys.map(k => [k, styles[k]]));
}

const before = snapshotStyles('.card');
// ... 注入 CSS ...
const after = snapshotStyles('.card');
console.table({ before, after });
```

### 20.3 隔离测试

在 Console 中创建临时样式标签来测试单个组件，避免污染全局：

```js
// 创建一个临时 <style>，5 秒后自动移除
const testStyle = document.createElement("style");
testStyle.textContent = `.card { background: red !important; }`;
testStyle.id = "temp-test-style";

```

### 20.4 捕获注入导致的异常

```js
// 包装 MutationObserver 以捕获回调中的异常
const originalObserve = MutationObserver.prototype.observe;
MutationObserver.prototype.observe = function (...args) {
  const callback = this._originalCallback || (fn => fn);
  const wrapped = function (mutations) {
    try {
      callback.call(this, mutations);
    } catch (e) {
      console.error("[Theme Debug] MutationObserver error:", e);
    }
  };
  return originalObserve.call(this, wrapped, ...args);
};
```

## 21. 性能优化

### 21.1 CSS 选择器性能

从快到慢：

1. **ID 选择器** `#header` — 最快
2. **Class 选择器** `.card` — 快
3. **属性选择器** `[data-view-id="sidebar"]` — 中等
4. **部分匹配** `[class*="_status_"]` — 慢
5. **通用选择器** `*` — 最慢，禁止

**实践**：主题 CSS 在页面加载时一次性应用，选择器性能通常不是瓶颈。但 JS 中 `querySelectorAll("[class*='...']")` 在大型 DOM 中可能慢——应限定在已知容器内查找：

```js
// ❌ 全局扫描
document.querySelectorAll('[class*="_status_"]');

// ✅ 限定在卡片容器内
card.querySelector('[class*="_status_"]');
```

### 21.2 MutationObserver 性能

`subtree: true` 在大型 SPA 中可能产生海量回调。优化：

- **去抖**：`setTimeout(scan, 60)` 而非直接 `scan()`
- **属性过滤**：`attributeFilter: ["class"]` 减少不相关回调
- **限定观察范围**：如果确定变化只发生在某个容器内，不要 observe `document.body`

### 21.3 CSS 文件大小

本项目 CSS 约 1120 行。对于注入场景（UserScript/扩展），建议：
- 压缩空格（但保留可读性以便调试）
- 合并重复的渐变定义（提取为变量）
- 不需要的组件样式直接删除

## 22. 常见陷阱与反模式

### 22.1 用 `display: none` 隐藏元素导致布局塌陷

```css
/* ❌ 可能影响 flex/grid 布局 */
.original-logo { display: none; }

/* ✅ 保持占位，只是不可见 */
.original-logo { visibility: hidden; width: 0; overflow: hidden; }

/* ✅ 或者用 opacity + pointer-events */
.original-logo { opacity: 0; pointer-events: none; }
```

### 22.2 硬编码 px 值而非相对单位

```css
/* ❌ 换字体大小后间距失调 */
.card { padding: 10px; font-size: 13px; }

/* ✅ 用 em/rem 或 CSS 变量 */
.card { padding: calc(var(--font-size) * 0.75); font-size: var(--font-size); }
```

### 22.3 修改全局样式忘记 `body` 和 `#root`

很多应用在 `body` 和 `#root` 上有全局背景色、字体设置，容易被忽略：

```css
/* 必须同时覆盖 */
html, body, #root {
  color: var(--text-primary) !important;
  background: var(--bg-primary) !important;
}
```

### 22.4 忘记 `input::placeholder` 和 `:empty::before`

```css
/* placeholder 颜色需要单独设置 */
input::placeholder,
textarea::placeholder,
[contenteditable="true"]:empty::before {
  color: var(--text-dim) !important;
}
```

### 22.5 `-webkit-background-clip: text` 在主题中的坑

如果原始 CSS 使用了渐变文字（`background-clip: text` + `text-fill-color: transparent`），覆盖文字颜色时需要用 `-webkit-text-fill-color`：

```css
/* 仅设 color 不够，需要同时重置 text-fill */
.gradient-text {
  color: var(--text-primary) !important;
  -webkit-text-fill-color: currentColor !important;
  background: none !important;
}
```

### 22.6 暗色模式下图片 filter 未调整

白色/亮色图片在暗色背景上可能过曝，需要降低亮度：

```css
body.dark img.bright-image {
  filter: brightness(.85);
}
```
