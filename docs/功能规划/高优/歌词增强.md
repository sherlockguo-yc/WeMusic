# 歌词增强

## 概述

LRC 滚动歌词和换源功能的基础上，逐步提升歌词详情页的交互体验和展示质量。

已完成：滚动交互优化、纯音乐检测、封面兜底链、点击跳转进度、翻译歌词支持、fetchLyric 接口重构、换源功能修复、桌面歌词。

## 功能方向

### 滚动歌词交互优化 ✅ 已完成

**实现时间**：2026-07-18

**需求**：
1. 引入"用户滚动状态"检测机制，区分主动滚动与自动滚动
2. 用户主动滚动时暂停自动跟随，允许自由浏览
3. 闲置 3 秒后平滑恢复自动跟随

**实现位置**：`src/lyrics.js`

**方案**：
- 4 个模块级状态变量：`_userScrolling`、`_scrollIdleTimer`、`_ignoreScrollUntil`、`SCROLL_IDLE_MS`
- `_ignoreScrollUntil` 时间戳区分程序滚动 vs 用户滚动（600ms 窗口）
- `_onLyricsScroll` 处理 `lpBody` scroll 事件：过滤程序滚动 → 标记用户滚动 → 启动 3s 闲置计时
- `_resumeAutoScroll` 闲置到期后调 `syncLyrics` 平滑滚回当前播放行
- `syncLyrics` 中 `_userScrolling` 为 true 时跳过 `scrollTo`

**状态重置点**：面板打开、关闭、新歌词加载时均重置为自动跟随模式。

### 纯音乐检测 ✅ 已完成

**实现时间**：2026-07-18

**需求**：自动判断纯音乐，跳过歌词搜索，避免无意义的 API 调用。

**判断依据**：歌名关键词（时长特征暂未启用，纯音乐时长分布太广，准确率不高）。

**实现位置**：
| 文件 | 改动 |
|------|------|
| `server/services/lyrics.js` | 新增导出 `isInstrumental(name)`，11 个正则模式 |
| `server/routes/stats.js` | `/api/stats/lyrics` 路由步骤 2 检测，命中则直接返回 `{ instrumental: true }` |
| `src/lyrics.js` | `doLoadLyrics` 处理 `instrumental` 响应，显示专用 UI |
| `public/css/style.css` | `.lp-instrumental` 音符图标 + 文字样式 |

**关键词列表**：
- 中文：`纯音乐`、`伴奏`、`无人声`、`背景音乐`、`器乐`（排除器乐团/器乐队）
- 英文/缩写：`instrumental`、`karaoke`、`off[-]vocal`、`orchestral`、`symphony`、`BGM`

**设计决策**：
- sourceId 优先：用户显式切换歌词源时跳过纯音乐检测
- 保守匹配：不匹配 `Inst.`（太模糊）、`acoustic`（通常有 vocal），宁漏不误
- 缓存：`instrumental: true` 也写入 24h 内存缓存，避免重复检测
- UI 区分：纯音乐显示音符图标 + "纯音乐，暂无歌词"，与普通"暂无歌词"视觉不同

**数据流**：
```
播放「纯音乐 - 天空之城」
  → loadLyrics() → GET /api/stats/lyrics?name=纯音乐 - 天空之城
    → isInstrumental("纯音乐 - 天空之城") → true
    → 返回 { instrumental: true } （0 次外部 API 调用）
  → 歌词面板显示 🎵 纯音乐，暂无歌词
```

### 封面三级兜底 ✅ 已完成

**实现时间**：2026-07-18

**需求**：歌词详情页在无专辑封面时不应显示空白圆圈，应降级使用歌手头像或默认图标。

**实现位置**：
| 文件 | 改动 |
|------|------|
| `src/lyrics.js` | `updateLyricsPanelMeta` 中新增 `tryCover()` 三级兜底函数 |
| `src/player.js` | `saveSession()` 序列化时补上 `singer_mid`，确保刷新页面后封面兜底不失效 |
| `public/css/style.css` | `.lp-cover-img[data-no-cover]` → `object-fit: contain` |

**三级兜底链**：
1. `song.album_mid` → 专辑封面（`albumCover()`）
2. `song.singer_mid` → 歌手头像（`singerAvatar()`），`img.onerror` 回退
3. `PLACEHOLDER_COVER_SVG` → 默认音乐图标（data URI SVG）+ 渐变背景

**设计决策**：
- 与播放器底栏 `updateNpCover()` 的三级兜底逻辑一致
- `session` 持久化增加 `singer_mid` 字段，刷新后不丢失
- 占位 SVG 使用 `encodeURIComponent` 编码原始 `#` 颜色值（避免双重编码导致解析失败）

### 点击歌词行跳转进度 ✅ 已完成

**实现时间**：2026-07-18

**需求**：在歌词详情页点击某行歌词，自动将播放进度调整到该行的时间位置。

**实现位置**：`src/lyrics.js` → `initLyrics()` 中 `lpBody` 上新增事件委托 click 处理。

**方案**：
- 通过 `closest('.lp-line')` + `data-i` 定位点击行，取 `lyricsLines[i].time`
- 调用 `player.js` 的 `seekTo(sec)` 跳转
- 同时重置 `_userScrolling = false`（点击是明确的导航意图）
- `.lp-line` 已自带 `cursor: pointer` + hover 变亮，无需额外 CSS

### 翻译歌词支持 ✅ 已完成

**实现时间**：2026-07-18

**需求**：如果歌词源提供了翻译（如网易云 `tlyric`），在歌词行下方展示翻译文本。

**实现位置**：

| 文件 | 改动 |
|------|------|
| `server/services/lyrics.js` | `fetchLyrics()` / `fetchLyricsById()` / `searchLyricsCandidates()` / `qqFetchLyric()` 返回 `{ lrc, tlyric }` 双字段；新增 `parseLrcWithTrans()` 按时间戳匹配原文与翻译 |
| `server/routes/stats.js` | `/api/stats/lyrics` 所有分支返回 `tlyric`；导入 `parseLrcWithTrans` 替代 `parseLrc` |
| `src/lyrics.js` | `renderLyricsLines()` 中追加 `<span class="lp-trans">` 翻译行 |
| `public/css/style.css` | `.lp-trans` 样式（字号 0.8em，颜色 `var(--text-dim)`，与原文区分） |

**匹配策略**：
- 翻译有时间戳 → 按时间匹配原文行
- 翻译无时间戳 → 按顺序逐行匹配

**设计决策**：
- 翻译是歌词源自带的（网易云 `tlyric`、QQ 音乐 `trans`），不依赖外部翻译 API
- 翻译为空时不显示 `<span class="lp-trans">`（不占 DOM），避免空白行
- 翻译文本也经过 `esc()` 转义，防止 XSS

### 换源功能修复 ✅ 已完成

**实现时间**：2026-07-18

**问题**：歌词换源功能失效，经历 3 轮修复才定位根因。

**根因**：`GET /api/stats/lyrics` 带 `sourceId` 的换源快速通道响应缺少 `candidates` 字段，前端 `lyricsCandidates = data.candidates || []` 把候选列表清空，下次打开弹窗重新拉取（不带 `sourceId`）走自动匹配覆盖用户选择。

**修复**：

| 文件 | 改动 |
|------|------|
| `src/lyrics.js` `restoreTitle()` | 加 null 检查（`candModal` 无 `<h3>` 元素，赋值 `null.textContent` 抛 TypeError） |
| `server/routes/stats.js` NE 换源分支 | `sourceId: Number(id)` → `sourceId`（保持字符串编码格式一致） |
| `src/lyrics.js` `doLoadLyrics()` | `lyricsCandidates = data.candidates \|\| []` → `if (data.candidates) lyricsCandidates = data.candidates`（仅在响应包含时更新） |

**教训**：已写入项目 Rule「服务端同一接口不同分支的响应字段一致性」，避免同类问题再次出现。

### 桌面歌词 ✅ 已完成

**实现时间**：2026-07-18

**需求**：迷你悬浮歌词窗口，始终置顶（Web 版用迷你窗口方案，完整支持需桌面客户端）。

**方案选择**：Document Picture-in-Picture API（Chrome/Edge/Firefox 桌面版）→ 系统级置顶。Safari 等不支持 PiP 的浏览器降级为 `window.open()` 迷你窗口。

**实现位置**：

| 文件 | 改动 |
|------|------|
| `src/desktop-lyrics.js` | **新建**：PiP 窗口创建/销毁、歌词渲染、设置面板、播放控制、降级弹窗、500ms 同步循环 |
| `public/index.html` | `.player-right-btns` 新增「桌面」按钮（Lucide Monitor 图标） |
| `src/player.js` | `initPlayer()` 绑定按钮事件，动态 import 桌面歌词模块 |
| `public/css/style.css` | `#dtLyricsBtn.active` 激活态样式（accent 绿底） |

**功能详情**：

- **双行模式（默认）**：当前行高亮 + 下一行预览。可切换为单行模式
- **三种背景风格**：毛玻璃（`backdrop-filter: blur(20px)`）、纯暗色（`rgba(0,0,0,0.85)`）、跟随主题（浅色/深色）
- **播放控制**：上一首 / 播放暂停 / 下一首，与主页面状态同步
- **设置面板**：齿轮图标 → popover，布局和背景偏好存 `localStorage`
- **歌词动态切换**：切歌时 body 自动重建（占位 ↔ 歌词行），纯音乐显示音符图标
- **窗口关闭自动清理**：PiP `pagehide` 事件 + popup `beforeunload` 事件
- **按钮激活态**：桌面歌词打开时按钮变绿，关闭恢复默认

**设计文档**：`docs/plans/2026-07-18-桌面歌词设计.md`
