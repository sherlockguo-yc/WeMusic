# 音量一致性

## 概述

不同 B 站视频源的音量差异巨大，用户连续播放多首歌时需要频繁手动调整音量。目标：自动标准化响度，让用户听到大致一致的音量。

## 架构约束

| 播放模式 | 音频来源 | 可控制程度 |
|---------|---------|-----------|
| 前台播放 | B 站 `<iframe>`（跨域） | 只能 postMessage 设整体音量，无法分析音频 |
| 后台播放 | `<audio>` 元素（服务端代理流） | 可引入 Web Audio API（AudioContext + GainNode），完整控制 |

**关键结论**：前台 iframe 无法做响度分析，真正的标准化只能在后台播放时实现。

## 技术方向（待讨论确认）

- **方案 A：前端实时分析** — 用 OfflineAudioContext 分析音频响度（EBU R128/ReplayGain），首播时分析 + 缓存 gain 值，后续直接应用
- **方案 B：服务端预分析** — 服务器首次代理时分析响度，将 gain 值写入元数据，前端直接读取
- **方案 C：简单响度估测** — 不分析音频，用视频平台元数据（如果有响度信息）或启发式规则估算
