# 管理功能

> 面向管理员的完整后台能力。当前实现极其单薄（仅用户数/反馈数/总时长 + 用户列表 + 反馈删除，权限仅靠环境变量白名单），需系统性补全。

---

## 一、现状诊断

| 维度 | 现状 | 差距 |
|------|------|------|
| 权限模型 | ✅ 四级角色（super_admin → admin → moderator → viewer），`SUPER_ADMIN` 环境变量兜底 | — |
| 数据看板 | ✅ 概览卡片 + 7 天趋势 + Top 20 | 趋势维度单一、无可视化图表、无导出 |
| 用户管理 | ✅ 列表 + 归档/恢复/删除 + 角色管理 | 无详情页、无批量操作、无备注标记 |
| 内容审核 | ✅ 反馈删除 + 屏蔽源查看 + 敏感词库增删 | 无审核工作流、无举报处理 |
| 系统配置 | ✅ 注册控制 + 功能开关（Toggle Switch 即时生效） | 无灰度发布、无公告管理 |
| 操作审计 | ✅ 操作日志记录与查询 | 无可配置保留策略 |
| 系统监控 | ✅ 内存/运行时间/数据库大小 | 无 CPU/磁盘/API 性能指标 |

## 二、角色与权限体系

### 角色定义

| 角色 | 核心能力 | 限制 |
|------|---------|------|
| **超级管理员** | 全部模块 + 管理管理员 + 备份恢复 | 环境变量兜底，不可被 Web 端降级/删除 |
| **管理员** | 除「权限管理」「备份恢复」「功能开关」外的全部模块 | 不可删除/降级超级管理员 |
| **内容审核员** | 仅内容审核 + 用户标记/备注 | 不可禁用/删除用户 |
| **只读观察员** | 数据看板 + 用户列表 + 审计日志（只读） | 不可执行任何修改操作 |

## 三、数据看板

- **实时概览卡片**：DAU/MAU、新增注册、活跃播放用户数、播放歌曲/歌单/反馈/屏蔽源总量
- **趋势图表**：用户增长折线图、播放时长时段分布热力图、注册→活跃转化漏斗、留存率、反馈趋势
- **内容分析**：热门歌曲 Top 50、热门歌手 Top 20、高频搜索词、播放源健康度、红心/收藏排行
- **用户画像**：活跃度分层、偏好分布、人均使用时长/播放数
- **数据导出**：CSV/JSON 导出、定时周报/月报自动推送、自定义时间范围

---

## 四、用户管理增强

- **详情页**：完整播放历史、歌单明细、反馈历史、登录历史（时间/IP/UA）、操作时间线
- **操作**：禁用/启用账号、重置密码、强制下线、编辑用户信息
- **归档与删除**：先将用户标记为「已归档」（无法登录，数据保留），在归档用户页面可彻底删除或恢复，须超级管理员二次确认
- **批量操作**：条件筛选、批量导出、批量禁用/启用、批量发送站内通知
- **标记与备注**：内部备注、状态标记（正常/观察中/已警告/已封禁）、按标记筛选
- **列表增强**：服务端分页、可排序列、用户名模糊搜索

---

## 五、内容审核

- **屏蔽源管理**：屏蔽源列表、批量取消屏蔽、屏蔽原因分类、自动标记规则、屏蔽源统计
- **歌单审核**：公开歌单审核队列、举报处理、官方精选标记/置顶/推荐位
- **评论审核**：审核队列、自定义敏感词库、用户禁言/解禁、举报处理
- **头像/昵称审核**：违规头像标记→强制重置、敏感昵称检测→强制修改
- **审核工作流**：待审核队列排序、审核记录、驳回必填理由、批量审核

---

## 六、系统配置

- **注册控制**：开放注册/仅邀请制/关闭注册、邀请码管理、IP 黑白名单
- **功能开关（Feature Flags）**：运行时启用/禁用模块、灰度发布（按百分比或用户组）
- **全局参数**：播放默认值、API 限频、缓存策略、推荐参数调整
- **公告管理**：创建/编辑/删除公告、定时发布+过期自动下架、目标用户范围筛选
- **备份与恢复**：手动/自动备份、从备份恢复（须超级管理员验证，自动创建快照）

---

## 七、操作审计

- **管理员操作日志**：操作人/目标/类型/时间/IP、变更前后对比、不可篡改
- **安全事件日志**：登录/登出记录、异地 IP 检测、异常时段操作、API 异常统计
- **日志查询与导出**：多条件筛选，CSV/JSON 导出
- **保留策略**：可配置保留天数，超期自动清理

---

## 八、系统监控

- **服务健康**：CPU/内存/磁盘使用率、Node.js 进程指标、数据库连接池、慢查询
- **API 性能**：P50/P95/P99 响应时间、QPS 趋势、错误率分布、第三方 API 成功率
- **后台任务**：任务面板、失败告警与重试、单次任务耗时
- **告警规则**：可配置阈值、通知方式（站内/邮件/Webhook/企业微信机器人）

---

## 九、核心交互流程

**内容审核闭环**：用户举报 → 系统聚合 → 审核员受理 → 确认屏蔽/驳回 → 通知举报人 → 写入审计日志

**运营日常巡检**：看板概览 → 发现异常 → 趋势详情 → 定位根因 → 导出周报

**功能灰度发布**：超级管理员启用功能开关 → 按比例灰度（20%→50%→100%）→ 观察指标 → 全量/回滚

**用户数据合规**：管理员搜索用户 → 导出数据 → 用户确认 → 二次验证删除

**备份与灾难恢复**：每日自动备份 → 恢复前校验 → 创建快照 → 二次验证 → 执行恢复

## 十、已确认决策

- [x] **管理面板形态**：独立路由 `/admin`，非 SPA 内弹窗
- [x] **用户删除机制**：采用「归档」机制（类似回收站）
  - 归档后用户无法登录，数据保留不删除
  - 提供「归档用户」页面，在此页面才能真正删除用户数据
  - 支持从归档恢复为正常账号
- [x] **敏感词库**：内置预置 + 管理员可手动增删，不对接第三方 API
  - 初始数据源：[konsheng/Sensitive-lexicon](https://github.com/konsheng/Sensitive-lexicon)（MIT，3.8k stars，持续更新）
- [x] **备份功能**：暂不开发，后续按优先级排期

## 十一、技术选型

- [x] **超级管理员初始化**：环境变量 `SUPER_ADMIN`（原 `ADMIN_USERNAME`），首次启动时自动将匹配用户 role 设为 `super_admin`
- [x] **前端文件拆分**：从第一版就拆分，6 个子模块各自独立文件，Vite 打包合并
- [x] **CSS**：新建独立 `admin.css`，不与 `style.css` 混在一起

## 十二、UI/UX 设计决策

- [x] **Logo 样式**：`We<span class="logo-accent">Music</span><span class="logo-suffix">管理</span>`，仅 `Music` 用主题色（绿色），对齐用户页面 logo 风格
- [x] **配置项控件**：使用 **开关按钮（Toggle Switch）** 而非下拉选择框
  - 选择即生效，零按钮，失败自动回弹
- [x] **弹窗体系**：禁止使用浏览器原生 `prompt()` / `confirm()`
  - 确认类 → 统一 `uiConfirm`
  - 输入类 → 统一 `uiPrompt`
  - 选项类 → 新增 `uiChoice`（单选列表 + 角色描述）
- [x] **布局方案**：管理面板用 `position: fixed; inset: 0` 覆盖整个视口，绕开原 `.app` 的 Grid 约束
- [x] **浏览器返回按钮**：`popstate` 监听中，离开 admin 时自动移除 `body.admin-view` 类，恢复侧栏/顶栏/播放器

## 十三、文件结构（骨架阶段）

```
server/routes/admin.js          # /api/admin/* 全部路由
server/middleware/admin.js       # requireRole(...roles) 中间件

src/admin-panel.js               # 管理面板入口（侧栏导航 + 动态加载 Tab）
src/admin/overview.js            # 概览：卡片统计 + 7 天趋势 + Top 20
src/admin/users.js               # 用户：活跃/归档切换 + 角色/归档/删除（uiChoice 弹窗）
src/admin/moderate.js            # 审核：反馈 + 屏蔽源 + 敏感词库增删
src/admin/config.js              # 配置：注册控制 + 功能开关（Toggle Switch）
src/admin/audit.js               # 审计：操作日志列表
src/admin/monitor.js             # 监控：内存/运行时间/数据库

public/css/admin.css             # 管理面板样式（独立）

scripts/seed-sensitive-words.js  # 敏感词库种子（konsheng/Sensitive-lexicon）
```
