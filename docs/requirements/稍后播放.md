# 下一首播放

## 用户场景

用户正在听歌时，浏览搜索结果、歌单、发现页，看到想听的歌但不希望打断当前播放。点击「下一首播放」将该歌曲插入到当前歌曲之后，当前歌曲播完后自动切到该歌曲。

**典型流程**：
1. 用户正在听「晴天」→ 在搜索页搜「七里香」→ 右键 → 下一首播放 → toast 提示 → 继续浏览
2. 当前歌曲播完后自动播放「七里香」

## 设计决策（2026-07-10 讨论）

| 决策 | 选择 |
|------|------|
| 菜单项数量 | 1 个：「下一首播放」 |
| 行为 | 插入到当前歌曲之后（`splice(queueIndex+1, 0, song)`），不是队列末尾 |
| 队列空时 | 仍显示「下一首播放」，行为 = 作为 queue[0]，不自动播放 |
| 替换队列 | 队列有待播歌时 toast 提醒，不弹确认框 |
| 业界参考 | 简化版（参考 Spotify 单选项模式 + Apple Music 的插队行为） |

## 现状分析

### 当前队列行为

- 点击歌曲行 / 右键「播放」→ `playFromList(songs, index)` **替换整个队列**（`state.queue = songs`）
- 没有「追加到现有队列」的能力
- 右键菜单项：添加到歌单 / 分享 / 复制 Bilibili 链接 / 在 Bilibili 搜索此歌 / 从歌单移除

### 数据模型（现有，无需改动）

```js
// state.js
state.queue = [];       // 播放队列数组
state.queueIndex = -1;  // 当前播放索引
```

## 功能设计

### 入口：右键菜单（所有歌曲行通用）

```
┌──────────────────────────┐
│ 下一首播放               │  ← 新增
├──────────────────────────┤
│ 添加到歌单               │
│ 分享                     │
│ 复制 Bilibili 链接       │
│ 在 Bilibili 搜索此歌     │
├──────────────────────────┤
│ 从歌单移除               │  ← 仅在歌单内时显示
└──────────────────────────┘
```

### 视觉反馈

| 触发时机 | 反馈方式 |
|----------|----------|
| 正在播放，添加成功 | Toast：「「歌名」将下一首播放」（2s 自动消失） |
| 队列空，添加成功 | Toast：「「歌名」已添加到播放队列」 |
| 队列有待播歌，执行替换播放 | Toast：「已替换播放队列」 |
| 队列按钮 | 增加队列数量 badge（如 `3`），已有队列数量显示时更新数字 |
| 队列提醒 | 添加后 0.5s 内队列按钮短暂脉冲动画一次（`transform: scale(1.1)`） |

### 行为细节

- **队列为空且无正在播放时**：添加到 queue[0]，不自动播放
- **不去重**：用户可以同一首歌添加多次（有意为之，比如想循环听某首歌 N 遍）
- **跨视图**：在搜索结果页 / 歌单页 / 发现页 / 专辑页均可触发
- **持久化**：添加后自动 `saveSession()`，刷新页面队列不丢失
- **右键菜单选中后立即关闭菜单**
- **`playFromList` 替换队列时**：如果队列中有待播歌（用户手动加过「下一首播放」），toast 提醒「已替换播放队列」

## 实现要点

### 核心函数（新增 `src/queue.js`）

```js
// 插入到下一首
function enqueueNext(song) {
  if (state.queueIndex >= 0) {
    state.queue.splice(state.queueIndex + 1, 0, song);
    toast(`「${song.name}」将下一首播放`);
  } else {
    state.queue.push(song);
    toast(`「${song.name}」已添加到播放队列`);
  }
  saveSession();
  renderQueue();
  updateQueueBadge();
  pulseQueueBtn();
}
```

## 验收标准

1. 正在播放时，右键任意歌曲行 → 出现「下一首播放」
2. 点击「下一首播放」→ toast 提示 "「歌名」将下一首播放" + 当前歌曲播完立即切到该歌
3. 连续「下一首播放」3 首不同歌 → 按添加顺序依次播放
4. 队列为空时右键 → 「下一首播放」可见，点击后 toast "已添加到播放队列"
5. 刷新页面 → 队列中通过「下一首播放」加的歌仍存在
6. 从搜索页 / 歌单页 / 发现页 / 专辑页均可触发
7. 队列中有待播歌时点击播放新歌 → toast "已替换播放队列"
