# 稍后播放

## 用户场景

用户正在听歌时，浏览搜索结果、歌单、发现页，看到想听的歌但不希望打断当前播放。点击「稍后播放」将该歌曲追加到播放队列末尾，当前歌曲播完后自动切到该歌曲。

**典型流程**：
1. 用户正在听「晴天」→ 在搜索页搜「七里香」→ 右键 → 稍后播放 → toast 提示已添加 → 继续浏览
2. 当前队列播完后自动播放「七里香」

## 现状分析

### 当前队列行为

- 点击歌曲行 / 右键「播放」→ `playFromList(songs, index)` **替换整个队列**（`state.queue = songs`）
- 没有「追加到现有队列末尾」的能力
- 右键菜单项：添加到歌单 / 分享 / 复制 Bilibili 链接 / 在 Bilibili 搜索此歌 / 从歌单移除
- 缺少类似 Spotify「Add to Queue」、网易云「下一首播放」的功能

### 数据模型（现有，无需改动）

```js
// state.js
state.queue = [];       // 播放队列数组
state.queueIndex = -1;  // 当前播放索引
```

### 队列操作（现有）

| 函数 | 文件 | 行为 |
|------|------|------|
| `playFromList(songs, index)` | player.js | 替换 queue，从 index 开始播放 |
| `removeFromQueue(i)` | queue.js | 从 queue 中移除第 i 首 |
| `playNext()` | player.js | 播放 queue[queueIndex+1] |
| `renderQueue()` | queue.js | 刷新队列抽屉 UI |

## 功能设计

### 方案：两个入口 + 两个位置

**两个放置位置**：

| 位置 | 含义 | 效果 |
|------|------|------|
| **下一首播放** | 插入到当前歌曲之后 | `queue.splice(queueIndex + 1, 0, ...songs)` |
| **稍后播放** | 追加到队列末尾 | `queue.push(...songs)` |

**设计理由**：网易云音乐 / QQ 音乐 / Apple Music / YouTube Music 均同时提供这两个选项。各自场景不同：
- 下一首播放：等当前这首听完马上听它（高紧迫）
- 稍后播放：先排着，队列里前面的播完了再说（低紧迫）

**「下一首播放」仅在队列中有正在播放的歌曲时可用**（`queueIndex >= 0`），否则与「稍后播放」等价（都是加到最后），此时隐藏「下一首播放」避免歧义。

### UI 入口

#### 入口 1：右键菜单（所有歌曲行通用）

```
┌──────────────────────────┐
│ 下一首播放               │  ← 新增（queueIndex >= 0 时显示）
│ 稍后播放                 │  ← 新增
├──────────────────────────┤
│ 添加到歌单               │
│ 分享                     │
│ 复制 Bilibili 链接       │
│ 在 Bilibili 搜索此歌     │
├──────────────────────────┤
│ 从歌单移除               │  ← 仅在歌单内时显示
└──────────────────────────┘
```

#### 入口 2：歌曲行「更多」按钮菜单（…）

与右键菜单内容一致。

#### 入口 3：搜索页直接按钮（可选，低优先级）

在搜索页每首歌曲行增加一个「⏭」图标按钮，点击直接执行「下一首播放」（有当前歌曲时）或「稍后播放」（无当前歌曲时），无需打开菜单。

### 视觉反馈

| 触发时机 | 反馈方式 |
|----------|----------|
| 成功添加 | Toast：「已添加到稍后播放」（2s 自动消失） |
| 队列按钮 | 增加队列数量 badge（如 `3`），已有队列数量显示时更新数字 |
| 队列提醒 | 添加后 0.5s 内队列按钮短暂脉冲动画一次（`transform: scale(1.1)`） |

### 行为细节

- **支持多首歌曲同时添加**：选中多首歌（如果将来支持多选），或专辑/歌单的「全部稍后播放」
- **去重检查**：不检查重复。用户可以同一首歌添加多次（有意为之，比如想循环听某首歌 N 遍）
- **队列为空且无正在播放时**：添加首首歌后提示「队列为空，是否立即播放？」→ 否：只添加到队列，需手动点击播放；是 → 直接播放（可选，可后续迭代）
- **跨视图**：在搜索结果页 / 歌单页 / 发现页 / 专辑页均可触发
- **持久化**：添加后自动 `saveSession()`，刷新页面队列不丢失
- **右键菜单选中后立即关闭菜单**

```

                    │   queueIndex >= 0    │
                    │   (正在播放中)        │

                             │

              ▼              ▼              ▼
```

## UI 层级

右键菜单 / 更多菜单均为现有 `.ctx-menu`（z-index: 200），无需新增弹窗。

## 边界条件

| 场景 | 行为 |
|------|------|
| 队列为空 + 稍后播放 1 首 | 添加到 queue，不自动播放。「队列」按钮 badge 显示 1 |
| 队列为空 + 用户点「播放」 | 保持现有行为（`playFromList` 替换队列并播放） |
| 正在播放最后一首 + 稍后播放 | 追加到末尾，播完当前最后这首后切到新歌 |
| 正在播放 + 下一首播放 | 插到当前歌曲之后，下一首就是它 |
| 正在播放 + 连续加 3 首「下一首播放」 | 按添加顺序（先加的先播），LIFO？不，FIFO。第三首 > 第二首 > 第一首（从 queueIndex+1 开始依次 insert） |
| 队列为空 + 下一首播放 | 「下一首播放」选项不显示（与稍后播放等价，无需冗余） |
| 同一首歌多次稍后播放 | 允许，不做去重 |
| 删除队首（当前播放） | 现有逻辑不变（`removeFromQueue` 处理） |
| 用户清空队列后稍后播放 | 等同「队列为空」场景 |

## 实现要点

### 核心函数（新增）

```js
// src/queue.js 新增

// 添加到队列末尾
function enqueueLater(songs) {
  state.queue.push(...songs);
  saveSession();
  renderQueue();
  updateQueueBadge();
  toast(`${songs.length > 1 ? songs.length + ' 首' : '「' + songs[0].name + '」'}已添加到稍后播放`);
  pulseQueueBtn();
}

// 插入到下一首
function enqueueNext(songs) {
  if (state.queueIndex < 0) {
    // 无当前歌曲，等同于稍后播放
    return enqueueLater(songs);
  }
  for (const s of songs.reverse()) {
    state.queue.splice(state.queueIndex + 1, 0, s);
  }
  saveSession();
  renderQueue();
  updateQueueBadge();
  toast(`${songs.length > 1 ? songs.length + ' 首' : '「' + songs[0].name + '」'}将下一首播放`);
  pulseQueueBtn();
}
```

### 修改点

| 文件 | 改动 |
|------|------|
| `src/queue.js` | 新增 `enqueueLater()`、`enqueueNext()`、`updateQueueBadge()`、`pulseQueueBtn()` |
| `src/ui.js` | `openSongMenu()` 右键菜单新增「下一首播放」「稍后播放」两项 |
| `src/playlist-ui.js` | 歌曲行「…」更多按钮菜单新增同样两项 |
| `public/css/style.css` | 队列按钮 badge 样式、脉冲动画 |
| `public/index.html` | 队列按钮增加 badge 容器 |

### 不需要改动的

- `state.js` — 数据模型不变
- `player.js` — 播放逻辑不变（`playNext()` 读的是 `queue[queueIndex+1]`，自然生效）
- `server/` — 无需服务端改动

## 验收标准

1. 正在播放时，右键任意歌曲行 → 出现「下一首播放」和「稍后播放」
2. 点击「下一首播放」→ toast 提示 + 当前歌曲播完立即切到该歌
3. 点击「稍后播放」→ toast 提示 + 队列末尾新增该歌曲
4. 队列为空时右键 → 不显示「下一首播放」，只显示「稍后播放」
5. 连续「稍后播放」3 首 → 队列按添加顺序排列，队列按钮显示 badge `3`
6. 刷新页面 → 队列仍存在（持久化生效）
7. 从搜索页 / 歌单页 / 发现页 / 专辑页均可触发
8. 队列按钮脉冲动画正常（添加时短暂放大再回弹）
9. 多首同时添加（如专辑「全部稍后播放」）→ toast 显示「N 首已添加到稍后播放」

## 不在本需求范围内

- 多选歌曲后批量操作（这是更上层功能，需先实现多选交互）
- 队列内拖拽排序
- 「全部稍后播放」按钮（作为独立小需求后续追加）
- 添加到队列后是否自动滚动到队列抽屉中的对应位置
