# 自定义主题色

## 用户场景

用户在设置中除了 10 个预设主题色外，可以创建自己调配的颜色作为主题强调色（`--accent`）。

## 数据模型

### 自定义颜色对象

```json
{
  "id": "abc123",
  "name": "暖橙",
  "color": "#ff6b35",
  "createdAt": 1720300000
}
```

- `id`: 随机字符串，唯一标识
- `name`: 用户起的名字，可空字符串
- `color`: HEX 颜色值 `#rrggbb` 或 `#rgb`
- 每个用户最多 8 个自定义颜色

### 存储

自定义颜色列表存在 `user_preferences.data` JSON 的 `customPalettes` 字段中，与现有偏好共存。

### API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/auth/custom-palettes` | 获取当前用户的自定义颜色列表 |
| POST | `/auth/custom-palettes` | 创建自定义颜色 `{name, color}` |
| DELETE | `/auth/custom-palettes/:id` | 删除指定自定义颜色 |

## 交互方案

**方案 D：系统预设 + HSL 滑块 + 原生取色器 + HEX 输入**

### 系统预设
- 10 个预设色块保持不变（绿/蓝/红/橙/黄/粉/紫/青/靛/灰）
- 预设颜色不可删除

### 自定义编辑工具
- HSL 三通道滑块：Hue（色相 0-360）、Saturation（饱和度 0-100%）、Lightness（亮度 0-100%）
- HEX 输入框：支持直接输入/粘贴色号
- 原生取色器：`<input type="color">` 调用系统调色盘
- 名字输入框：可空

### 行为
- 拖动滑块/输入 HEX → 实时预览（`--accent` 即时变化）
- 点击「保存」→ 写入服务端，新增色块出现在列表中
- 点击 × 删除 → 如果该颜色当前正在使用，回退到绿色
- 已达 8 个上限时，「+ 自定义」按钮禁用

## 状态机

| 状态 | 表现 |
|------|------|
| 预览态 | 默认：10 个系统色块 + 「+ 自定义」按钮 |
| 编辑态 | 点击「+ 自定义」展开 HSL 滑块 + HEX + 取色器 + 名字 |
| 上限态 | 8 个已满，「+ 自定义」灰掉 |
| 删除确认 | hover 自定义色块出现 × → 点击 → 确认 → 删除 |

## 边界条件

- 删掉当前使用的颜色 → 回退到默认绿色
- HEX 格式校验：`#rrggbb` / `#rgb` 两种格式
- 名字可为空（仅靠色块识别）
- 编辑态在切换到其他色块或关闭设置面板时自动收起

## 验收标准

1. 打开设置 → 看到 10 个系统预设色块 + 「+ 自定义」按钮
2. 点击「+ 自定义」→ 展开编辑工具 → 拖动滑块主题色实时变化
3. 调好颜色，输入名字 → 点保存 → 新色块出现并被选中
4. Hover 自定义色块 → 出现 × → 点击 → 确认后删除
5. 删除当前使用的颜色 → 回退到默认绿色
6. 已达 8 个自定义时 → 「+ 自定义」禁用
7. 关闭重新打开 → 自定义颜色仍在
8. 换设备登录 → 自定义颜色同步过来
