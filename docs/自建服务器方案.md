# 自建服务器方案

> 评估日期：2026-07-06
> 目标：搭建长期低成本、高配置的个人服务器，部署 WeMusic 及未来更多服务，对外提供稳定域名访问
> 原则：纯自建，不引入任何云服务器

---

## 一、网络环境验证

| 项目 | 结果 |
|------|------|
| 公网出口 IP | `120.229.206.232`（广东移动） |
| 是否 CGNAT 大内网 | ❌ 否（非 100.64.x.x 段） |
| 端口转发 | ✅ 可用（已在小米路由器测试通过） |
| 入站连接 | ✅ 可达（手机蜂窝数据成功访问 18989 端口） |
| **80 / 443 端口** | ❌ **被移动封锁**（运营商层面拦截） |
| 其他端口 | ✅ 可用（如 18989、8443 等） |
| IP 类型 | 动态公网 IP（重启光猫会变） |

**结论**：具备自建服务器条件，80/443 被封但非标端口可用，需 DDNS 处理 IP 变动。

---

## 二、方案：N150 小主机 + DDNS + 非标端口

### 链路（当前阶段：直接非标端口）

```
用户 → https://域名:8443 → 路由器端口转发(8443) → N150 → Nginx → Docker 服务
```

### 链路（未来升级：Cloudflare DNS Proxy）

```
用户 → https://域名(标准443) → CF 边缘节点 → N150:8443 → Nginx → Docker 服务
```

### 端口方案

| 阶段 | 用户访问 | 实际端口 | 说明 |
|------|---------|---------|------|
| 当前 | `https://域名:8443` | 8443 | 直接非标端口，纯自建，国内直连 |
| 未来 | `https://域名` | 8443 | CF DNS Proxy 端口重写，域名干净 |

### 成本（5 年）

| 项目 | 费用 | 说明 |
|------|------|------|
| N150 小主机（16G+512G） | ~1000 元 | 一次性，4C4T x86，可扩 32G |
| 电费 | 250 元 | ~50 元/年，待机 7-10W |
| 域名 | 300 元 | ~60 元/年 |
| **5 年总计** | **~1550 元** | 年均 ~310 元 |
| 升级 32G 内存 | +300 元 | 可选 |
| **含升级总计** | **~1850 元** | |

### 优势

- 长期成本最低，用得越久越划算（5 年比云服务器便宜一半以上）
- 4C4T x86，可跑十几个 Docker 容器
- 非标端口阶段完全国内直连，零中间节点
- 数据完全自己掌控
- 配置可扩展（内存 32G、双硬盘）

### 劣势

- 停电/断网影响服务（个人项目可接受）
- URL 带端口号（当前阶段），未来 CF DNS Proxy 可解决
- 需要自己维护硬件和系统

## 三、硬件选购

N100 在 2026 年已普遍缺货停产，当前购买推荐代替品 **N150**（同架构、同 6W 功耗、性能略高 5-8%）。

### 推荐型号

**1. 大唐 MAX N150**（性价比最高）
- 准系统 ~559 元 + 自配 16G DDR5 + 512G NVMe
- 双 2.5G 网口、内置电源
- 配齐约 **~960 元**

**2. 中柏 JNG11（N150）**（品牌正规，京东有售）
- 整机含内存硬盘，约 **800-1000 元**
- USB 3.2 + Type-C、双网口

**规格要求**
- **CPU**：Intel N150，4 核 4 线程，TDP 6W，睿频 3.6GHz
- **内存**：16G DDR5 起步（方便后续扩 32G）
- **存储**：512G NVMe SSD 起步（建议选双硬盘位型号）
- **网口**：双 2.5G 网口

## 五、常见问题

### 80/443 端口被封怎么办？

**当前**：使用非标端口（8443），URL 访问 `https://域名:8443`。

**未来升级**：Cloudflare DNS Proxy — 将域名的 DNS 设为 Cloudflare 代理模式（橙色云朵），配置 Origin Rule 使 CF 监听标准 443，回源到 N150 的 8443 端口。用户访问标准 `https://域名`，无需端口号。

### IP 变了怎么办？

DDNS 客户端每 5 分钟检查一次公网 IP，发现变化后自动更新 DNS 记录。域名始终正确，用户无感知。

### 停电/断网怎么办？

个人项目可接受偶尔中断。如需更高可用性，可加个 UPS（不间断电源），~200 元。

### 安全措施

- SSH 密钥登录 + 禁用密码
- ufw 防火墙只开放必要端口
- 服务间通过 Docker 内网通信
- 定期更新系统和 Docker 镜像
