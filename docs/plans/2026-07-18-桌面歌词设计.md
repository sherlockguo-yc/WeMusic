# 桌面歌词 — 设计文档

> 日期：2026-07-18
> 状态：已实现
> 来源：`docs/功能规划/高优/歌词增强.md`

## 概述

在播放器底栏新增入口，点击后打开系统级置顶的迷你歌词窗口。主方案使用 Document Picture-in-Picture API（Chrome 116+ / Edge 116+ / Firefox 151+），Safari 降级为 `window.open()` 迷你窗口。

## 方案选择

| 方案 | 技术 | 系统置顶 | 浏览器支持 |
|------|------|---------|-----------|
| **主方案** | Document PiP API | ✅ | Chrome/Edge 116+, Firefox 151+ |
| **降级** | `window.open()` | ❌ | 所有浏览器 |

运行时特征检测：`'documentPictureInPicture' in window` → PiP，否则 → popup。

## 窗口内容

### 默认布局（双行模式 B）

```
┌──────────────────────────────────┐
│                            ⚙      │  ← 设置齿轮（右上角）
│  🎵  当前歌词行（高亮，大字）    │
│     下一行歌词（灰色，小字预览） │
│  ⏮  ⏯  ⏭                      │  ← 播放控制
└──────────────────────────────────┘
```

- 窗口尺寸：**固定 `340×130`**，打开后不再调用 `resizeTo` 改变尺寸（用户手动拖拽的宽高始终保留）
- 当前行：`font-weight: 700`，`font-size: 17px`（中号，可字号调节）
- 下一行：灰度透明，`font-size: 12px`
- 播放按钮：Lucide skip-back / play-or-pause / skip-forward，16px

### 单行模式（A）

- **不改变窗口尺寸**，仅通过 CSS `[data-layout="single"] .dt-next{display:none}` 隐藏第二行预览
- 设置中切换，偏好存 localStorage

### 背景风格（三种，用户可选）

| 风格 | 效果 | CSS |
|------|------|-----|
| **毛玻璃**（默认） | 半透明 + 模糊，桌面壁纸可见 | `backdrop-filter: blur(20px)` + `rgba(0,0,0,0.4)` |
| **纯暗色** | 固定暗色底，任何壁纸清晰 | `rgba(0,0,0,0.85)` + 白色歌词 |
| **跟随主题** | 浅色白底黑字 / 深色黑底白字 | `var(--bg)` / `var(--text)` |

偏好键：`wemusic_desktop_lyrics_bg` → `'blur'` | `'dark'` | `'theme'`

### 设置入口

窗口右上角齿轮图标，点击弹出设置面板。面板使用 **`position: fixed`** 定位（脱离 `.dt-root` 的 `overflow:hidden` 裁剪范围，否则 130px 高的窗口会把面板内容裁掉）。面板内 3 组 toggle：

- **布局**：双行 / 单行（`data-layout` → CSS 隐藏下一行）
- **背景**：毛玻璃 / 暗色 / 主题（`data-bg` → CSS 切换背景）
- **字号**：小 / 中 / 大（`data-size` → CSS 切换当前行与下一行字号）

所有设置变更**只更新 DOM 属性 / CSS class，不调用 `resizeTo`**，避免窗口尺寸被强制重置。不弹模态框。

### 播放控制

三个按钮：⏮ 上一首 / ⏯ 播放暂停 / ⏭ 下一首。
- 按钮状态从主页面 `player.js` 同步
- 点击通过同一 origin 直接调用主页面的 `playPrev()` / `togglePause()` / `playNext()`

## 模块架构

```
src/desktop-lyrics.js     ← 新文件：桌面歌词全部逻辑（独立 chunk，11KB / 4KB gzip）
public/index.html          ← 修改：.player-right-btns 新增按钮
src/player.js              ← 修改：initPlayer() 绑定按钮点击（动态 import）
public/css/style.css       ← 修改：#dtLyricsBtn.active 激活态样式
```

### 实际实现与设计的差异

- **`lyrics.js` 无需修改**：桌面歌词的同步循环在 `desktop-lyrics.js` 内部独立运行（500ms interval），直接读取 `lyrics.js` 导出的 `lyricsLines` 和 `state.current`，无需 lyrics.js 主动 push
- **按钮图标**：使用 Lucide Monitor 图标（`<rect>` + `<line>`），而非 `monitor-speaker`
- **加载方式**：`player.js` 通过 `await import('./desktop-lyrics.js')` 动态加载，首屏不打包

### 模块职责

- **`desktop-lyrics.js`**：PiP 窗口生命周期、HTML/CSS 注入、500ms 同步循环、body 重建（占位 ↔ 歌词行）、设置面板（布局/背景/字号）、播放控制事件、popup 降级、清理
- **`player.js`**：`$('dtLyricsBtn').onclick` → 动态 import → `openDesktopLyrics()` 或 `closeDesktopLyrics()`
- **`public/index.html`**：静态按钮元素 `id="dtLyricsBtn"`
- **`public/css/style.css`**：`#dtLyricsBtn.active { background: var(--accent); }`

## 数据流（实际实现）

```
desktop-lyrics.js 内部 setInterval(500ms)
  │
  ├→ import { lyricsLines } from './lyrics.js'     ← 直接读导出值
  ├→ import { state } from './state.js'            ← state.current
  ├→ const _playerP = import('./player.js')        ← elapsed, timerPaused, autoTimer
  │
  ├→ 计算当前行 index → render(pipDoc, state)
  │    ├→ dtCurrent.textContent = line[idx].text
  │    ├→ dtNext.textContent = line[idx+1]?.text
  │    └→ dtPlayBtn.innerHTML = paused ? PLAY : PAUSE
  │
  └→ 检测 lyricsLines.length 变化 → rebuildBody()（占位 ⇄ 歌词行）

**通信方式**：主页面和 PiP 窗口是同一 origin，直接访问 `pipWindow.document` 操作 DOM。PiP 窗口内无独立 JS bundle。

**偏好存储**：`localStorage`
- `wemusic_desktop_lyrics_layout` → `'double'` | `'single'`
- `wemusic_desktop_lyrics_bg` → `'blur'` | `'dark'` | `'theme'`
- `wemusic_desktop_lyrics_size` → `'sm'` | `'med'` | `'lg'`（新增加字号偏好）

## 触发入口

播放器底栏（`.player`）按钮组中新增桌面歌词按钮。Lucide Monitor 图标：

```svg
<rect x="2" y="3" width="20" height="14" rx="2"/>
<line x1="8" y1="21" x2="16" y2="21"/>
<line x1="12" y1="17" x2="12" y2="21"/>
```

## 边界情况

| 场景 | 处理 |
|------|------|
| PiP 窗口被用户手动关闭 | `pagehide` 事件 → 更新按钮状态为「未激活」 |
| 浏览器标签页刷新 | PiP 自动关闭（生命周期绑定），刷新后按钮恢复默认态 |
| 纯音乐（无歌词） | PiP 窗口显示「纯音乐，暂无歌词」+ 音符图标 |
| 切歌 | 新歌词加载完成后 push 新 lines 数组 |
| PiP API 不可用 → popup 降级 | 功能正常，仅失去系统置顶 |
| popup 被浏览器拦截 | toast 提示「请允许弹出窗口」 |
| 重复点击按钮 | 检查窗口是否已存在 → 已存在则聚焦，不重复打开 |
| 歌词为空 / 加载中 | 显示「加载中…」placeholder |

## 实现后验证

1. Chrome/Edge：PiP 窗口置顶 → 点开 VS Code 看看歌词还在不在上面 ✅
2. Safari：降级为 popup → 功能正常（未测试，无 Safari 环境）
3. 切歌 → 窗口内容更新 ✅（`rebuildBody` 自动检测 `lyricsLines.length` 变化）
4. 播放/暂停 → 按钮图标同步 ✅（同步循环 500ms 更新）
5. 单行/双行切换 → 仅隐藏/显示下一行（CSS `data-layout`），窗口尺寸不变 ✅（已移除 `resizeTo`，避免覆盖用户手动拖拽的窗口尺寸）
6. 三种背景切换 → 视觉正确 ✅
7. 关闭窗口 → 按钮恢复未激活态 ✅（`pagehide` / `beforeunload` → `cleanup()`）
8. 纯音乐 → 显示专用 UI ✅
