# 桌面歌词 — 设计文档

> 日期：2026-07-18
> 状态：设计完成，待实现
> 来源：`docs/功能规划/中优/歌词增强.md`

## 概述

在播放器底栏新增入口，点击后打开系统级置顶的迷你歌词窗口。主方案使用 Document Picture-in-Picture API（Chrome 116+ / Edge 116+ / Firefox 151+），Safari 降级为 `window.open()` 迷你窗口。

## 方案选择

| 方案 | 技术 | 系统置顶 | 浏览器支持 |
|------|------|---------|-----------|
| **主方案** | Document PiP API | ✅ | Chrome/Edge 116+, Firefox 151+ |
| **降级** | `window.open()` | ❌ | 所有浏览器 |

运行时特征检测：`'documentPictureInPicture' in window` → PiP，否则 → popup。

## 窗口内容

### 默认布局（双行模式 B）

```
┌──────────────────────────────────┐
│  🎵  当前歌词行（高亮，大字）    │ ⚙ │  ← 设置齿轮
│     下一行歌词（灰色，小字预览） │    │
│  ⏮  ⏯  ⏭                      │    │  ← 播放控制
└──────────────────────────────────┘
```

- 窗口尺寸：`320×120`
- 当前行：`var(--text)`，`font-weight: 700`，`font-size: 1.1em`
- 下一行：`var(--text-dim)`，`font-size: 0.85em`
- 播放按钮：Lucide skip-back / play-or-pause / skip-forward，16px

### 单行模式（A）

- 去掉第二行预览，窗口 `320×72`
- 设置中切换，偏好存 localStorage

### 背景风格（三种，用户可选）

| 风格 | 效果 | CSS |
|------|------|-----|
| **毛玻璃**（默认） | 半透明 + 模糊，桌面壁纸可见 | `backdrop-filter: blur(20px)` + `rgba(0,0,0,0.4)` |
| **纯暗色** | 固定暗色底，任何壁纸清晰 | `rgba(0,0,0,0.85)` + 白色歌词 |
| **跟随主题** | 浅色白底黑字 / 深色黑底白字 | `var(--bg)` / `var(--text)` |

偏好键：`wemusic_desktop_lyrics_bg` → `'blur'` | `'dark'` | `'theme'`

### 设置入口

窗口右上角齿轮图标，点击弹出 3 个 toggle 选项（布局 / 背景），使用 CSS popover 或简单下拉，不弹模态框。

### 播放控制

三个按钮：⏮ 上一首 / ⏯ 播放暂停 / ⏭ 下一首。
- 按钮状态从主页面 `player.js` 同步
- 点击通过同一 origin 直接调用主页面的 `playPrev()` / `togglePause()` / `playNext()`

## 模块架构

```
src/desktop-lyrics.js     ← 新文件：桌面歌词全部逻辑
src/player.js              ← 修改：底栏添加触发按钮
src/lyrics.js              ← 修改：syncLyrics 时 push 状态到 PiP
public/css/style.css       ← 修改：按钮样式 + PiP 窗口内部样式
```

### 模块职责

- **`desktop-lyrics.js`**：打开/关闭 PiP 窗口、渲染内容、处理设置切换、播放按钮事件
- **`player.js`**：新增 `#dtLyricsBtn` 按钮的 click 处理 + 状态同步
- **`lyrics.js`**：`syncLyrics()` 中增加 push 分支，每秒推送当前行 index + paused 状态

## 数据流

```
lyrics.js syncLyrics(sec)         ← 每秒触发
  │
  ├→ 更新主歌词面板 (lpBody)
  │
  └→ if PiP 窗口打开:
       desktopLyrics.pushState({
         idx,                    // 当前行 index
         lines,                  // lyricsLines 数组
         paused,                 // 播放/暂停
         songName, singer        // 当前歌曲信息
       })
         │
         ▼
       renderPipContent()        // 直接操作 PiP window.document 的 DOM
```

**通信方式**：主页面和 PiP 窗口是同一 origin，直接访问 `pipWindow.document` 操作 DOM。PiP 窗口内无独立 JS bundle。

**偏好存储**：`localStorage`
- `wemusic_desktop_lyrics_layout` → `'double'` | `'single'`
- `wemusic_desktop_lyrics_bg` → `'blur'` | `'dark'` | `'theme'`

## 触发入口

播放器底栏（`.player`）按钮组中新增桌面歌词按钮。Lucide `monitor-speaker` 或 `picture-in-picture2` 图标。

## 边界情况

| 场景 | 处理 |
|------|------|
| PiP 窗口被用户手动关闭 | `pagehide` 事件 → 更新按钮状态为「未激活」 |
| 浏览器标签页刷新 | PiP 自动关闭（生命周期绑定），刷新后按钮恢复默认态 |
| 纯音乐（无歌词） | PiP 窗口显示「纯音乐，暂无歌词」+ 音符图标 |
| 切歌 | 新歌词加载完成后 push 新 lines 数组 |
| PiP API 不可用 → popup 降级 | 功能正常，仅失去系统置顶 |
| popup 被浏览器拦截 | toast 提示「请允许弹出窗口」 |
| 重复点击按钮 | 检查窗口是否已存在 → 已存在则聚焦，不重复打开 |
| 歌词为空 / 加载中 | 显示「加载中…」placeholder |
