# WeMusic 架构概览

> **维护策略**：结构约束由 `scripts/check-deps.js` + CI 自动验证。
> 本文档只记录「设计意图和约定」（为什么这么做）。

## 模块依赖

```
utils.js → 所有模块（共享图标/工具函数，禁止被任何模块循环依赖）
state.js → 全局状态（被所有模块读/写）
main.js  → 启动入口，导入所有模块
  └→ player / queue / search / lyrics / ui / playlist-ui / settings / stats / admin
```

**硬约束（CI 自动检查）**：禁止 `main.js` 被其他模块 `import ... from './main'` 静态导入。所有 cross-module 导航必须使用 `await import('./main.js').then(m => m.navPush(...))`。

## Vite 构建

- `base: '/dist/'` — HTML 引用与 chunk 路径必须一致
- `manualChunks` 拆分 player / playlist / lyrics / stats / search
- 共享常量放在 `utils.js`（不在 chunked 模块，避免 TDZ）
- 生产构建保留 `console.error`/`console.warn`（`drop: ['console.log']`）

## 数据存储

| 类型 | 存储 | 跨设备 |
|------|------|:--:|
| 偏好（主题/字体/字号/色板/音量） | localStorage + 服务端 `user_preferences` | ✅ |
| 歌单 / 歌曲 / 红心 / 专辑收藏 | 服务端 SQLite | ✅ |
| 播放队列 + 当前进度 | localStorage + `GET/PUT /auth/session` | ✅ |
| 最近播放 | `play_logs` → `GET /stats/history` | ✅ |
| 播放统计 | `POST /stats/log` → `play_logs` | ✅ |
| bvid 缓存 | `GET/PUT /stats/bvid/:id` | ✅ |
| 播放模式 / 侧边栏宽度 | 仅 localStorage | ❌ |

## 播放器状态机

```
未播放 → [点播放] → 解析 bvid → startVideo → 播放中
         [切后台] → bgAudio 接管 → [回前台] → mountVideoAt
播放中 → [暂停] → 暂停中 → [继续] → 播放中
```
