# 图片封面兜底

## 设计目标

专辑封面、歌手头像等外链图片在加载失败时（404/网络错误），**不应隐藏留空**，而应 fallback 到默认占位图标。

## 问题背景

2026-07-08「为你推荐」页面发现：部分热门歌曲有 `album_mid` 字段，但 y.qq.com 实际未收录对应专辑图片 → 图片 404 → `onerror="this.style.display='none'"` → img 元素隐藏 → 该行封面位置**空白**（无默认图标）。

根因：渲染逻辑只处理了「无 `album_mid`」这一种无封面场景，没有处理「有 `album_mid` 但图片 404」。

## 实现方案

### 正确模式：`data-fb` + `outerHTML` 替换

```js
// src/playlist-ui.js（已修复）
const ph = `<span class="song-cover-ph-sm"><svg>...</svg></span>`;
const coverUrl = albumCover(s.album_mid, 150);
// 图片正常→显示封面；404→outerHTML 替换为默认图标
coverHtml = `<img class="song-cover-sm" src="${coverUrl}" loading="lazy"
  data-fb="${esc(ph)}"
  onerror="this.outerHTML=this.dataset.fb" />`;
```

**原理**：
1. 把默认图标的 HTML 存在 `data-fb` 属性（`esc` 转义 HTML 特殊字符）
2. 图片正常加载 → 什么都不做
3. 图片 404 → `onerror` 触发 → `this.outerHTML` 把 `<img>` 整替换为 `<span class="song-cover-ph-sm">`
4. 默认图标和封面图尺寸一致（28×28），替换后 flex 布局不跳

### 反模式：`onerror` 仅隐藏

```js
// ❌ 禁止：图片消失留空
onerror="this.style.display='none'"
onerror="this.style.visibility='hidden'"
```
