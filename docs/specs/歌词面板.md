# 歌词面板

## 概述

`.lyrics-panel` 是全屏歌词视图，包含封面轮播、歌曲信息、操作按钮、歌词内容区。

## 结构

```
.lyrics-panel（fixed, scale 动画展开）
├── .lp-bg（毛玻璃背景）
├── .lp-bg::after（渐变遮罩）
├── .lp-content（z-index: 2, flex column）
│   ├── .lp-top（标题栏）
│   ├── .lp-main（flex row）
│   │   ├── .lp-cover-side（左列，flex: 0 0 380px）
│   │   │   ├── .lp-cover-wrap（封面圆形容器）
│   │   │   └── .lp-song-info（居中对齐, width:100%; overflow:hidden）
│   │   │       ├── .lp-song-name（最多 2 行）
│   │   │       ├── .lp-song-singer（最多 2 行，支持换行）★
│   │   │       └── .lp-like-row（Grid 3 列按钮区，margin-top: 20px）
│   │   ├── .lp-body（右列，歌词滚动区）
│   │   └── .lp-bg-overlay（背景详情浮层）
│   └── .lp-ctrl-bar（底部播放控制）
```

## 歌曲信息区（`.lp-song-info`）★

**宽度约束**：`.lp-song-info` 必须设置 `width: 100%; overflow: hidden`，这是 2026-07-11 新增的约束。`.lp-cover-side`（380px 列 + 左右各 40px padding）内容区仅 300px 宽。若不约束宽度，子元素文本（歌名、歌手名）会溢出容器覆盖下方按钮。

### 歌手名换行（`.lp-song-singer`）★

**背景**（2026-07-11）：合唱歌曲歌手用 `/` 分隔（如"李克勤 / 林晓峰 / 张碧晨 / … / 炎明熹"），15 个歌手名约 800px 宽，远超 300px 可用宽度。此前使用 `white-space: nowrap; text-overflow: ellipsis` 单行截断，不仅大部分歌手不可见，且由于 `.lp-song-info` 缺少宽度约束，文本溢出容器与下方 `.lp-like-row` 按钮行视觉重叠。

**方案选择**：
- 方案 A（滚动）：复用 `checkMarquee` 左右滚动。占用 1 行但需 hover 交互，与 `.lp-song-name`（已用 `line-clamp: 2`）行为割裂
- **方案 B（换行，选中）**：`display: -webkit-box; -webkit-line-clamp: 2; -webkit-box-orient: vertical; word-break: break-word`，最多 2 行，超出用 `text-overflow: ellipsis` 兜底。信息一目了然，与歌名行为一致，改动最小

**实现**：`public/css/style.css` 中 `.lp-song-singer` 样式。

## 按钮网格（`.lp-like-row`）★

**布局**：`display: grid; grid-template-columns: repeat(3, auto); justify-content: center; justify-items: center; gap: 12px; margin-top: 20px`

按钮按 grid 自动流排列，每行最多 3 个：
- 第 1 行：♡（列 1）| 歌单（列 2）| 歌词（列 3）
- 第 2 行：背景（如果换行）

`margin-top: 20px`（2026-07-11 从 14px 上调）确保按钮行与上方歌手名之间有足够间距，避免合唱歌曲多行歌手名与按钮重叠。

**关键**：「背景」按钮必须在 JS 中显式声明 `grid-column` 以对齐「歌单」按钮所在列：

```js
const bgCol = song.song_mid ? 2 : 1; // 有 ♡ 时歌单在第 2 列，否则在第 1 列
$('lpLikeRow').innerHTML = `
  ${likeBtn}                                        // 列 1（可能不存在）
  <button id="lpAddBtn">歌单</button>                // 列 2（或列 1）
  <button id="lpSwitchBtn">歌词</button>             // 列 3（或列 2）
  <button id="lpBgBtn" style="grid-column:${bgCol}">背景</button> // 显式列绑定
`;
```

`justify-items: center` 确保同列元素 centerX 精确对齐。

**历史教训**：曾尝试用 `flex-wrap + rAF + getBoundingClientRect() + transform: translateX()` 的 JS 补丁方案，失败根因是 `.lyrics-panel` 展开有 0.55s scale 动画，动画期间测量数据不可靠。最终用 Grid 声明式方案根治。见 `.codebuddy/rules/CSS布局混用禁止.mdc` 第 7 条。

### 按钮切换触发

- ♡：`toggleLike(song)`，存在 song_mid 时才渲染
- 歌单：`import('playlist-ui').addSongs([song])`
- 歌词：`openLyricsSwitchModal(song)`（网易云/B站候选）
- 背景：切换 `#lpBgOverlay` 的 `display`

## 展开/关闭动画

- 进入：set `--lp-origin-x/y`（封面在播放条中的位置），`classList.add('show')` → `transform: scale(1)`
- 退出：`classList.remove('show')` → `transform: scale(0)` 缩回封面位置
- 过渡：0.55s cubic-bezier

动画期间可见性靠 `pointer-events: none/auto` + `visibility: hidden/visible` 控制。

## 响应式

720px 断点以下（移动端）：
- `.lp-main` 切换为 `flex-direction: column`
- `.lp-cover-side` 改为横向排列，封面缩至 80x80
- `.lp-song-info` 增加 `overflow: hidden`（约束歌手名宽度），`flex: 1; min-width: 0`
- `.lp-like-row` 切回 `justify-content: flex-start`（按钮左对齐，Grid 语义不变），`margin-top: 10px`（比桌面端小，移动端空间有限）
