# 歌手写真

## 用户场景

用户在歌手页点击「写真」tab，浏览歌手头像 + 全部专辑封面的画廊。点击图片进入全屏 lightbox，支持键盘 ← → 切换、Esc 关闭、自动轮播（6 秒/张 + 双图层 cross-fade 过渡）。

## 数据流

- **来源**：`/music/artist` 返回的 `albums` 列表（已有，无需新 API）
- **图片**：
  - 歌手头像：`T001R800x800M000{mid}.jpg`
  - 专辑封面：`T002R800x800M000{albumMid}.jpg`
- **空值**：无专辑 → 只显示歌手头像 + "暂无写真"
- **异常**：API 失败 → tab 内错误信息；个别封面 404 → `onerror` 移除该卡片

## UI 布局

- **画廊网格**：`repeat(auto-fill, minmax(260px, 1fr))`，自适应网格
- **item**：1:1 正方形，`object-fit: cover`，hover 放大 + 阴影
- **歌手头像**：与专辑封面相同尺寸，无特殊样式区分
- **Lightbox**：`z-index: 200`，全屏 92% 黑色遮罩，双图层叠加（Grid 同一区域）
- **Cross-fade**：双图叠放，切图时下一张后台加载 → 加载完后 0.6s opacity 交叉淡入淡出
- **自动播放**：右下角 ▶/⏸ 按钮，6 秒/张循环；手动切图自动暂停
- **移动端**（≤720px）：`minmax(160px, 1fr)`，lightbox 边距缩窄

## 状态机

- **空态**：无专辑 → 头像 + 空提示
- **加载态**：与歌曲/专辑同步，切换 tab 即时渲染
- **正常态**：头像 + 专辑封面网格
- **全屏浏览态**：双图层 cross-fade 切换 + 手动/自动轮播 + 底部名称 + 页码

## 影响面

| 文件 | 改动 |
|------|------|
| `src/search.js` | 统一导入 `singerAvatar`；`openArtist()` 新增写真 tab + 画廊 + lightbox + 双图层 cross-fade + 自动播放 |
| `public/css/style.css` | `.photo-gallery` / `.photo-item` / `.lightbox-img-wrap` 双图层 / `.lightbox-play` 等 + 响应式 |

服务端无需修改。

## 边界

- 专辑多于 100 张：全部展示，滚动浏览
- 图片 404：`onerror` 移除卡片，lightbox 从 DOM 实时收集可见图片
- 移动端：grid 自适应收缩
- 键盘事件只绑定一次（`document._lbKeyBound`）
- lightbox DOM 只创建一次（`ensureLightbox`）
- 自动播放定时器在关闭/手动切图/退出页面时清除
- 首次打开 lightbox 直接显示（不做 cross-fade），后续切换才有过渡

## 验收标准

1. 歌手页三个 tab：歌曲 / 专辑 / 写真
2. 写真 tab 显示头像 + 全部专辑封面网格，大小一致
3. 点击封面进入全屏 lightbox，双图层 cross-fade 过渡
4. 支持 ← → 切换、Esc 关闭、点击遮罩关闭
5. 右下角播放按钮，6 秒/张自动轮播，手动切图自动暂停
6. 底部显示名称 + 页码
7. 窗口 < 720px 网格自适应
