# 分享海报系统

> 更新时间：2026-07-06
> 相关代码：`shared/poster-template.js`、`server/services/poster.js`、`server/routes/stats.js`、`src/report.js`

## 架构概述

```
┌─────────────┐     POST /api/stats/poster     ┌───────────────┐
│  Browser     │ ────────────────────────────── │  Server       │
│  src/report  │   { theme, format, data }      │  Puppeteer    │
└─────────────┘                                  └───────────────┘
                                                       │
                                                shared/poster-template.js
                                                ┌─────┴─────┐
                                                │ Desktop    │  Mobile
                                                │ 690×auto   │  430×820
                                                │ DSF 2      │  DSF 3
                                                │ 1 张 PNG   │  4 张 PNG
                                                └───────────┘
```

- **模板层** `shared/poster-template.js`：纯字符串模板，不依赖 DOM/Node API，可在浏览器和 Node ESM 通用
- **渲染层** `server/services/poster.js`：Puppeteer 无头浏览器渲染 HTML → 截图 PNG
- **路由层** `server/routes/stats.js`：desktop 返回 `image/png`，mobile 返回 JSON `{pages: [...]}`
- **前端** `src/report.js`：用户选择主题 + 格式 → 调用 API → 下载

## 主题系统

| Key | 名称 | 风格 |
|-----|------|------|
| `mint` | 清新简约 | 白底深字，浅色调色板 |
| `dark` | 暗夜律动 | 黑底浅字 + 封面模糊背景 |
| `sunset` | 日落橙粉 | 渐变紫橙 |
| `ocean` | 深海蓝调 | 渐变蓝紫 |

每个主题定义 `text`、`textDim`、`textSub`、`accent`、`cardBg`、`cardBorder`、`chipBg` 等 Token。

## 桌面版

- **尺寸**：690px 宽，高度随内容自适应
- **像素密度**：`deviceScaleFactor: 2`
- **布局**：杂志式横排，`wpp-*` 类名前缀
- **内容**：品牌 → 标题 → 日期 → Hero（播放数字）→ 4 列拼贴 → 听歌人格 → 洞察 → TOP 5 歌曲 → TOP 歌手 → 统计 → Footer
- **返回格式**：单张 PNG（`image/png` Content-Type）

## 手机版（4 页 Story）

### 设计原则

- 每页 **430×820** 固定尺寸，恰好占一个手机屏幕（iPhone 14 Pro Max 比例）
- **3× 像素密度**适配 Retina 屏
- **类名前缀**：`wpm-*`（HTML）、`.wmpN-*`（CSS），与桌面版 `wpp-*` 完全隔离
- 每页右上角页码标签 `WEEKLY XX / 04`
- 每页底部点状页码指示器
- 顶部 4px 主题色渐变条
- **禁止使用"上滑查看"等垂直滑动引导文案**（用户拿到多张图后会左右滑动）

### 4 页内容

| 页 | 文件名后缀 | 内容 |
|----|-----------|------|
| 1 | `cover` | PRO 徽章 + 品牌 + 标题 + 日期 + 3 列速览（播放/歌曲/天数）+ TOP 1 歌曲/歌手卡片 + 装饰波形 |
| 2 | `stats` | 大数字播放量 + 4 个 stat 卡（不重复歌曲/听歌天数/日均播放/听歌歌手）+ 人格卡 + 洞察引用 |
| 3 | `albums` | 2×2 封面拼贴网格，每个 cell 内：封面图 + 播放次数 badge + 专辑名 + 歌手名 |
| 4 | `rankings` | TOP 5 歌曲（含封面+排名+歌名+歌手+播放次数）+ 最爱歌手标签（显示播放次数）+ Footer |

### API

```
POST /api/stats/poster
Body: { theme: "dark", format: "mobile", data: {...posterData} }
Response: {
  format: "mobile",
  pages: [
    { name: "cover",   dataUrl: "data:image/png;base64,..." },
    { name: "stats",   dataUrl: "data:image/png;base64,..." },
    { name: "albums",  dataUrl: "data:image/png;base64,..." },
    { name: "rankings",dataUrl: "data:image/png;base64,..." },
  ]
}
```

### 数据字段

传递给 `posterHTMLMobile` 的数据包含：

```js
{
  appName, title, dateRange,
  plays, duration, uniqueSongs, days,
  persona: { icon, label, desc },
  insightText,
  topSongs: [{ name, singer, albumMid, playCount }],   // playCount 新增
  topArtists: [{ name, playCount }],                     // playCount 新增
  topAlbums: [{ name, singer, albumMid, playCount }],   // playCount 新增
  generatedAt
}
```

## 注意事项

### QQ 音乐封面尺寸限制

y.qq.com 图片服务器只支持以下尺寸：**90 / 150 / 300 / 500 / 800**。使用其他尺寸（如 400）会 404。

```js
// ✅ 正确
coverUrl(albumMid, 500)
coverUrl(albumMid, 150)

// ❌ 错误 — 400 不在支持列表中，图片加载失败但 onerror 隐藏后只留空白框
coverUrl(albumMid, 400)
```

### 拼贴墙封面数量不足 4 张

通过 `buildCollageCovers()` 自动填充：优先 TopAlbums → 不足用 TopSongs 补足（去重）→ 仍不足用空 slot 占位。

### 修改共享代码后必须重启

`shared/poster-template.js`、`server/services/poster.js` 等文件被服务端引用，修改后必须 `bash restart.sh`。
