# 管理面板

## 架构

### 入口与框架

`src/admin-panel.js` 负责渲染管理面板的框架（侧边导航 + 内容区），各 Tab 模块按需动态 `import()` 加载。

```
admin-panel.js
├── openAdminPanel(initialTab)  — 渲染 admin-layout + 侧栏 + 内容区
├── loadTab(tab)                — 按需加载 Tab 模块，含角色查询与权限检查
├── renderNav()                 — 根据 cachedRole 过滤 TABS 渲染侧栏导航
├── refreshCachedRole(newRole)  — 供外部模块在角色变更后同步缓存
└── cleanupCurrentTab()         — Tab 切换时清理资源（如监控定时器）
```

### 角色体系

| 角色 | 权限 |
|------|------|
| super_admin | 全部 Tab + 管理其他管理员 + 归档用户 |
| admin | 除超级管理员专属外的全部模块 |
| moderator | 概览、用户列表（只读）、审核 |
| viewer | 概览、审计、监控（只读） |

角色缓存 `cachedRole`：
- 首次进入管理面板时通过 `GET /api/admin/me` 获取
- 缓存后不再重复请求（同一次管理面板会话内有效）
- 离开管理面板（返回主站）时清空

### Tab 模块

`TABS` 数组定义所有导航项，每个 Tab 有 `id`、`label`、`roles`（允许访问的角色列表）、`icon`（Lucide SVG）。

Tab 模块文件：
- `src/admin/overview.js` — 概览（数据看板、趋势图、热门歌曲）
- `src/admin/users.js` — 用户管理（角色修改、归档/恢复/删除）
- `src/admin/moderate.js` — 内容审核
- `src/admin/config.js` — 配置
- `src/admin/audit.js` — 审计日志
- `src/admin/monitor.js` — 实时监控

## 角色修改与侧栏空白 Bug

### 症状

在用户管理页面修改任意用户的角色后，左侧侧边栏完全变为空白。

### 根因

`src/admin/users.js` 在角色修改回调中**无条件**调用了 `refreshCachedRole()`：

```js
// 修改前（有 bug）
await api(`/admin/users/${id}/role`, { method: 'PUT', body: { role: newRole } });
refreshCachedRole();  // ← 无参数调用，无论修改谁的角色都触发
```

`refreshCachedRole()` 无参数时会将 `cachedRole` 设为 `null`，然后调用 `renderNav()`。`renderNav()` 用 `cachedRole` 过滤 `TABS`：

```js
const visibleTabs = TABS.filter((t) => t.roles.includes(cachedRole));
// cachedRole = null → 没有 tab 的 roles 包含 null → 侧栏为空
```

**修复（2026-07-10）**：

1. `src/admin/users.js`：只在修改**自己**角色时调用 `refreshCachedRole(newRole)`
2. `src/admin-panel.js`：`refreshCachedRole()` 改为 `async`，无参数时重新从服务端获取角色而非直接置 `null`

### 防御规则

1. **`refreshCachedRole()` 只在修改自己角色时调用**，修改他人角色不需要刷新导航
2. **`renderNav()` 的任何调用必须保证 `cachedRole` 不为 `null`**，否则所有 Tab 被过滤掉
3. **修改 `cachedRole` 的代码路径必须覆盖两种场景**：有参数（直接更新）和无参数（重新获取），禁止出现「无参数时直接置 null 后渲染」的代码
