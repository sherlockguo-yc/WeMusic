# 管理面板

## 架构

### 入口与框架

`src/admin-panel.js` 负责渲染管理面板的框架（侧边导航 + 内容区），各 Tab 模块按需动态 `import()` 加载。

```
admin-panel.js
├── openAdminPanel(initialTab)  — 渲染 admin-layout + 侧栏 + 内容区
├── loadTab(tab)                — 按需加载 Tab 模块，含角色查询与权限检查
├── renderNav()                 — 根据 cachedRole 过滤 TABS 渲染侧栏导航
├── refreshCachedRole(newRole)  — 供外部模块在角色变更后同步缓存
└── cleanupCurrentTab()         — Tab 切换时清理资源（如监控定时器）
```

### 角色体系

| 角色 | 权限 |
|------|------|
| super_admin | 全部 Tab + 管理其他管理员 + 归档用户 |
| admin | 除超级管理员专属外的全部模块 |
| moderator | 概览、用户列表（只读）、审核 |
| viewer | 概览、审计、监控（只读） |

角色缓存 `cachedRole`：
- 首次进入管理面板时通过 `GET /api/admin/me` 获取
- 缓存后不再重复请求（同一次管理面板会话内有效）
- 离开管理面板（返回主站）时清空

### Tab 模块

`TABS` 数组定义所有导航项，每个 Tab 有 `id`、`label`、`roles`（允许访问的角色列表）、`icon`（Lucide SVG）。

Tab 模块文件：
- `src/admin/overview.js` — 概览（数据看板、趋势图、热门歌曲）
- `src/admin/users.js` — 用户管理（角色修改、归档/恢复/删除）
- `src/admin/moderate.js` — 内容审核
- `src/admin/config.js` — 配置
- `src/admin/audit.js` — 审计日志
- `src/admin/monitor.js` — 实时监控

## 内容审核 Tab（src/admin/moderate.js）

入口函数 `renderModerate(container, role)`。内部通过 `admin-tabs` 切换三个子标签：**用户反馈** / **屏蔽源** / **敏感词库**。子标签内容渲染在 `#adminModerateContent` 容器内，`activeSubTab` 记录当前子标签，`loadSubTab()` 负责按当前子标签分发渲染。

### 权限可见性

| 角色 | 敏感词库新增/删除 | 反馈删除 |
|------|:---:|:---:|
| super_admin / admin | ✓ | ✓ |
| moderator | 只读（新增框与删除按钮不渲染） | 只读 |
| viewer | 不显示该 Tab | — |

> `currentRole !== 'moderator'` 作为是否渲染写入控件的判断条件。moderator 仅能查看敏感词列表，不能增删。

### 分页加载（无限滚动，2026-07-15）

不再一次性加载全部（原 `?limit=500`），改为滚动到底部自动加载下一页：

- 每页 `LIMIT = 50`，后端 `GET /admin/sensitive-words?page=N&limit=50` 已支持 `page`/`limit`（offset = (page-1)*limit）。
- 前端状态：`page`（当前页）、`total`（总数）、`loading`、`done`。首屏 `await loadPage()` 加载第 1 页。
- `#swScroll` 为 `IntersectionObserver` 的 `root`，`#swSentinel` 为观察目标，`rootMargin: '120px'`（提前 120px 预加载）。
- 哨兵进入视口 → `loadPage()`：追加 `data.words` 到 `#swTbody`，`page++`；若返回数 `< LIMIT` 则 `done=true`，哨兵显示"已加载全部"。
- 加载中哨兵显示旋转 loading 动画（`.sw-sentinel.loading::after`）；失败显示错误文案。
- `swObserver` 为模块级变量，`renderSensitiveWords` 入口先 `disconnect()` 旧 observer，避免 Tab 切换/增删重渲染时泄漏。
- 新增/删除成功后调用 `loadSubTab()` 整体重渲染（从第 1 页重新加载）。

> 注意：`.sw-table-wrap` 必须有固定 `max-height` + `overflow-y:auto`，否则无独立滚动容器、`IntersectionObserver` 的 root 退化为视口，逻辑仍工作但表头不再吸顶。

### 分类标签颜色映射

`catColors` / `catLabels` 在前端硬编码：

| category | 颜色 | 中文标签 |
|----------|------|---------|
| political | `#e0556a` | 政治 |
| adult | `#e08050` | 涉黄 |
| spam | `#d4a017` | 垃圾 |
| other | `#6b7280` | 其他 |

> 后端 `POST /admin/sensitive-words` 的默认 `category` 为 `'other'`（代码写死，前端未提供分类选择 UI）。如需支持手动选择分类，需额外开发。

### 交互行为

- **新增**：点击「添加」按钮或输入框按 `Enter` 触发 `doAdd()`，调用 `POST /admin/sensitive-words`，成功后 `toast('已添加')` 并重渲染。
- **删除**：点击 `.sw-del-btn` 先弹 `uiConfirm('确认删除此敏感词？\n\n匹配该词的内容将不再被过滤。')`，确认后 `DELETE /admin/sensitive-words/:id`，成功后 `toast('已删除')` 并重渲染。
- **空状态**：`.sw-empty` 显示搜索图标 + "暂无敏感词" 文案（替换原单行 `<td>` 文本）。

### 设计约束（遵循项目 Rules）

- 全部视觉常量使用 CSS 变量（`var(--bg-card)` / `var(--border)` / `var(--accent)` 等），未硬编码 px 字号。
- 图标统一 Lucide（plus / trash-2 / search），`stroke-width="2"`、`currentColor`。
- 删除按钮 hover/active 通过 CSS 实现，未仅硬编码在 JS 中。

## 角色修改与侧栏空白 Bug

### 症状

在用户管理页面修改任意用户的角色后，左侧侧边栏完全变为空白。

### 根因

`src/admin/users.js` 在角色修改回调中**无条件**调用了 `refreshCachedRole()`：

```js
// 修改前（有 bug）
await api(`/admin/users/${id}/role`, { method: 'PUT', body: { role: newRole } });
refreshCachedRole();  // ← 无参数调用，无论修改谁的角色都触发
```

`refreshCachedRole()` 无参数时会将 `cachedRole` 设为 `null`，然后调用 `renderNav()`。`renderNav()` 用 `cachedRole` 过滤 `TABS`：

```js
const visibleTabs = TABS.filter((t) => t.roles.includes(cachedRole));
// cachedRole = null → 没有 tab 的 roles 包含 null → 侧栏为空
```

**修复（2026-07-10）**：

1. `src/admin/users.js`：只在修改**自己**角色时调用 `refreshCachedRole(newRole)`
2. `src/admin-panel.js`：`refreshCachedRole()` 改为 `async`，无参数时重新从服务端获取角色而非直接置 `null`

### 防御规则

1. **`refreshCachedRole()` 只在修改自己角色时调用**，修改他人角色不需要刷新导航
2. **`renderNav()` 的任何调用必须保证 `cachedRole` 不为 `null`**，否则所有 Tab 被过滤掉
3. **修改 `cachedRole` 的代码路径必须覆盖两种场景**：有参数（直接更新）和无参数（重新获取），禁止出现「无参数时直接置 null 后渲染」的代码
