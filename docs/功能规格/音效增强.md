# 音效增强

> 状态：✅ 全部完成 · 2026-07-20

## 概述

基于 Web Audio API 的三项音效后处理，全部在浏览器端实现，无需服务端改动。它们共享同一个 AudioContext 和管线。

## 功能列表

| 功能 | 规格文档 | 技术 | 实施日期 |
|------|---------|------|---------|
| 音量归一化 | [音量归一化.md](音量归一化.md) | EBU R128 + ffmpeg 服务端分析 | 2026-07-13 |
| 淡入淡出 | [淡入淡出.md](淡入淡出.md) | 双 `<audio>` + 双 GainNode + linearRamp | 2026-07-20 |
| 预设 EQ | [均衡器.md](均衡器.md) | 5 段 BiquadFilterNode | 2026-07-20 |

## 音频管线

```
bgAudio → _gainNode ──┐
                        ├──→ AnalyserNode → [60Hz]→[250Hz]→[1k]→[4k]→[12k] → destination
_nextAudio → _nextGain ──┘
```

- **音量归一化**：`_gainNode.gain` 乘积（`safeGain × _volume`）
- **淡入淡出**：双 GainNode 的 `linearRampToValueAtTime` 反向交叉渐变
- **EQ**：合并后统一经过 5 段 BiquadFilter 链
- **音频可视化**：AnalyserNode（FFT 256）插在 gain 合并点与 EQ 链之间，`lyrics.js` 通过 `getAnalyserNode()` 读取频谱数据，在歌词详情页底部渲染实时频谱

## 不适用场景

- 展开视频（`mode === 'iframe'`）：音频由 B 站 iframe 播放器独立输出，GainNode / Filter 链路无法介入
- 目前仅 bg 模式有效
