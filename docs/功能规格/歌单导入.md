# 歌单导入

## 概述

音乐用户的歌单沉淀在多个平台，支持从外部平台导入歌单降低迁移壁垒。

## 数据流

```
用户粘贴分享链接 → POST /api/music/parse-playlist
  → 服务端解析链接 → 调用对应平台的歌单 API → 返回歌曲列表
  → 前端展示 → 用户确认 → 批量写入 songs 表 + 关联到歌单
```

## 已实现

- **QQ 音乐歌单导入**：解析分享链接（`https://y.qq.com/n/ryqq/playlist/...`），提取歌曲列表。（`POST /api/music/parse-playlist` → `server/services/qqmusic.js` → 调 QQ 音乐 `uniform_get_Dissinfo` 接口）
- **网易云音乐歌单导入**：解析分享链接（`https://music.163.com/playlist?id=...`），提取歌曲列表。（同上端点 → `server/services/netease.js` → 调网易云歌单 API）
- **JSON 文件导入/导出**：WeMusic 自身格式的备份与恢复。导出将当前歌单序列化为 JSON 文件下载，导入解析 JSON 并写入数据库。

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/music/parse-playlist` | 解析外部歌单链接，返回歌曲列表预览 |
| POST | `/api/music/import-playlist` | 确认导入，将歌曲列表写入指定歌单 |
| GET | `/api/playlists/:id/export` | 导出指定歌单为 JSON 文件 |

## 边界条件

- 链接格式不合法 → 返回错误提示"无法识别该链接格式，请检查是否完整复制"
- 外部 API 超时/不可用 → 返回错误提示，建议用户稍后重试
- 歌单为空 → 提示"该歌单没有歌曲"
- 部分歌曲解析失败（如歌曲已在平台下架） → 导入成功的部分，跳过的在日志中记录
- 重复导入同一歌单 → 不做去重，由用户决定是否清理重复歌曲
- JSON 文件格式不匹配 → 返回"文件格式不正确，请选择 WeMusic 导出的 JSON 文件"

## 涉及文件

| 文件 | 职责 |
|------|------|
| `server/routes/music.js` | `POST /api/music/parse-playlist` 路由 |
| `server/services/qqmusic.js` | QQ 音乐歌单 API 调用（`uniform_get_Dissinfo`） |
| `server/services/netease.js` | 网易云歌单 API 调用 |
| `src/playlist-ui.js` | 导入弹窗 UI、文件选择、批量写入 |
| `public/index.html` | 导入弹窗 HTML 骨架 |
| `public/css/style.css` | 弹窗样式 |

## 已知限制

- QQ 音乐歌单解析依赖 `disstid`（从链接中正则提取），不支持加密/私有歌单
- 网易云歌单解析依赖公开 API，可能因网易云接口变动而失效
- 不支持 Spotify / Apple Music 歌单导入（这些平台的 API 需要 OAuth 授权）
