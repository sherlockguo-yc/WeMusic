# 离线缓存

## 概述

目前所有音频都依赖 Bilibili 在线流，每次播放都需要请求音频代理。如果 B 站视频被下架或网络不稳定，歌曲就无法播放。支持离线缓存能大幅提升可用性。

## 已确认设计决策

- **缓存触发**：自动缓存所有播放过的歌曲（听过即存，set-and-forget）；同时提供手动「取消离线」能力（列表/详情里单独移除某首）。
- **技术方案**：不引入 PWA / Service Worker。使用浏览器前端存储（IndexedDB 或 OPFS）存原始音频字节，由 JS 逻辑控制命中与降级。
- **降级策略**：双重降级 —— ① `navigator.onLine === false`（断网）直接切缓存；② 在线但音频请求超时/失败时自动回退缓存并轻提示。
- **写入时机**：边播边后台落盘。播放仍走现有直连/代理流（立刻出声），同时后台 `fetch` 完整字节存盘，存完打标记；用户无感、不拖慢首播。
- **存储上限**：默认 2GB（设置里可调，如 5GB），达到上限按 LRU 自动淘汰最旧的缓存。
- **管理 UI**：侧边栏独立页面（`src/offline-page.js`），入口在侧边栏「离线缓存」导航项（位于「我的专辑」下方）。
  - 页面级视图，不占弹窗，与歌单页面同级。
  - 用量行：`X 首歌曲 · X 首主动缓存 · X 首自动缓存 · 已用 X.XX GB / X GB`（`view-sub` 副标题）。
  - 操作栏：存储上限选择器 +「清空自动缓存」和「清空全部」按钮（`offline-page-bar` flex `space-between`）。
  - 缓存列表：`section-head` 标题「已缓存歌曲」+ 全量列表（不再折叠）。
  - 列表项：显示「歌名 - 歌手」主行 + 「歌手 · B站音频 · 歌词{xxx} · 本地缓存/自动缓存」副行（含主动/被动状态标签），每行右侧一个 `×` 移除按钮。
  - 歌名多级回退链：显式 `song.name` → 歌词包内 `lyrics.song.name` → 视频源标题 → BV 号（兜底）。
- **缓存分类（主动 / 被动）**：两类数据需区分对待：
  - **被动缓存（自动）**：播放时自动落盘的音频 + 视频源 + 歌词。受 2GB 上限约束，超出按 LRU 淘汰最旧的。
  - **主动缓存（钉住）**：用户在歌曲上右键「缓存到本地」显式固定的离线包（音频 + 视频源 + 歌词）。**不受 LRU 淘汰影响，只能由用户手动删除**。
  - **主动钉住范围**：右键「缓存到本地」固定整包（音频 + 视频源 + 歌词），确保离线可完整播放且能看词。
  - **一键清空**：设置页提供两个按钮 —— 「清空自动缓存」（只删被动项，保留主动钉住）与「清空全部」（含主动钉住，需二次确认弹窗）。
  - **钉住随源迁移**：用户在已钉住的歌曲上换源时，「钉住」状态自动迁移到新源 —— 后台重新下载并钉住新源音频+视频源+歌词，同时释放旧源缓存；始终只保留当前源的钉住版。

## 缓存数据结构（草稿）

每首歌的离线包以「歌曲」为键（建议用 `bvid` 或 `bvid+cid`，音频由 cid 决定），包含：

| 字段 | 说明 |
|------|------|
| `key` | 歌曲标识（bvid / bvid+cid） |
| `audio` | 原始音频字节（Blob 或 OPFS 文件引用） |
| `videoSource` | 解析出的 B 站视频源（bvid/cid），离线回放用 |
| `lyrics` | 歌词源缓存（含 `lines`/`candidates`/`sourceId`/`song`/`artist`） |
| `song` | 持久化的歌曲信息 `{ name, singer }`，用于设置页列表展示（**2026-07-14 新增**，旧条目无此字段） |
| `pinned` | 是否主动钉住（true=主动缓存，false=被动） |
| `lastAccessed` | 最后访问时间（LRU 排序用，仅被动项参与淘汰） |
| `size` | 该包占用字节数 |
| `createdAt` | 落地时间 |

**LRU 淘汰规则**：仅对 `pinned=false` 的条目，按 `lastAccessed` 升序删除，直到总占用 ≤ 上限。主动钉住项永不自动删除。

## 旧条目迁移（2026-07-14 引入 `song` 字段）

`song` 字段是在「设置页极简化」时新增的。在此之前已落盘的旧条目没有 `song`，列表只能回退显示 BV 号（或「未知歌曲」）。

- 新增 `offline.migrateOldEntries()`：打开离线缓存页面时**自动执行一次**。
- 规则：遍历所有条目，已有 `song.name` 跳过；否则尝试从 `lyrics.song` 提取回填 `song = { name, singer }`；**无法回填（歌词包内也无歌名）→ 直接删除该条目**。
- 返回 `{ updated, deleted }`，删除时会 toast「已清理 N 条无歌名旧缓存」。
- 后续新缓存（`fetchAndStore` 主动/被动落盘、`migratePin` 源迁移）均已写入 `song` 字段，不再依赖迁移。

## 与现有播放架构的衔接

- 当前 `src/player.js` 的 `_getPlayableUrl` 有两条路径：直连 B站 CDN（优先，URL 带 `deadline` 过期）和服务端代理 `/api/play/stream`。
- **两种路径下载下来的字节都是原始音频**，与 URL 是否过期无关，因此缓存「字节」即可，回放 url 用 `URL.createObjectURL(blob)` / OPFS 文件路径喂给 `bgAudio`，兼容现有播放器。
- 播放前：先查离线包命中 → 命中则直接用本地字节（更新 `lastAccessed`）；未命中走在线流并后台落盘。
- 右键菜单（`.ctx-menu`，z-index 200）新增「缓存到本地」项，触发主动钉住下载（需先确保已解析 bvid/视频源，并抓取歌词）。

## UI 表现（已确认）

- **歌曲行状态角标（两态，视觉须明显区分）**：
  - **主动钉住（已下载）**：强调样式（如实心/填充图标 + accent 色），表示用户特意留存的离线收藏，不会被 LRU 删。
  - **被动缓存（已临时缓存）**：弱化样式（如描边/灰度图标 + 次要色），表示自动临时缓存、可能被 LRU 淘汰。
  - 二者**不可显示成同一效果**，须一眼可区分。
- **离线播放提示**：降级到缓存播放时，在播放栏状态区（`.player-status`）显示「离线播放」并附**原因**，如「离线」「网络不佳，已切换缓存」。
