# 淡入淡出（Crossfade）

> 状态：✅ 已实现 · 实施日期：2026-07-20

## 概述

切歌时通过双 `<audio>` 引擎 + 双 `GainNode` 实现平滑音频过渡——当前曲目淡出，同时下一曲目从 0 音量淡入，消除切歌的"断崖感"。

与音量归一化的关系：淡入淡出仅在自动切歌时执行，归一化增益在过渡期和过渡后始终生效。两根管线互不干扰。

## 行为矩阵

| 场景 | 行为 | 说明 |
|------|------|------|
| 自动切歌（歌曲播完） | 触发淡入淡出 | `autoAdvance()` 内启动 |
| 手动切歌（上一首/下一首按钮） | 立即取消过渡，直接切换 | `playNext()` / `playPrev()` 首行调 `_cancelCrossfade()` |
| 单曲循环 | 不触发 | `playMode === 'single'` 先于淡入淡出分支判断 |
| 展开视频（mode === 'iframe'） | 不触发 | 条件 `mode === 'bg'` 阻断了淡入淡出 |
| 定时停止（播完当前曲） | 不触发 | `sleepAfterSong` 分支先于淡入淡出 |
| 开关关闭（toggle off） | 退化为常规 `playNext(true)` | `_crossfadeDuration === 0` |

### 过渡期间的状态

| 关注点 | 过渡期间（0 → `duration` 秒） | 过渡结束后 |
|--------|------------------------------|-----------|
| 音频 | `bgAudio` 淡出，`_nextAudio` 淡入 | `bgAudio` 接管播放，`_nextAudio` 静音 |
| 歌词（歌词面板 + 桌面歌词） | 保持显示旧歌歌词 | `_completeCrossfade` 调用 `loadLyrics` 加载新歌 |
| 播放器 UI | 已切换到新歌信息（歌名/封面/进度条） | 无变化 |
| 进度条 | 从 0 重新开始 | 正常推进 |
| 播放历史 | 已记录新歌 | 无变化 |

## 音频管线

### 静态结构

```
bgAudio ──→ MediaElementSource ──→ _gainNode（归一化 + 音量）──┐
                                                                  ├──→ AudioContext.destination
_nextAudio ──→ _nextSource ──→ _nextGainNode（过渡用）──────────┘
```

两个 `<audio>` 元素各绑定一个 `MediaElementSource` + `GainNode`（`createMediaElementSource` 规范限制：每个 `<audio>` 只能调用一次，因此全部在 `_initAudioCtx()` 中一次性创建）。

`_nextGainNode` 初始 `gain.value = 0`，非过渡期间始终静音。

### 过渡期间的增益变化

使用 `linearRampToValueAtTime` 实现音频线程级插值（非 JS 定时器离散跳变）：

```
时刻 now（过渡开始）
  _gainNode.gain      = 当前正常音量（含归一化增益）
  _nextGainNode.gain  = 0

时刻 now + duration（过渡结束）
  _gainNode.gain      = 0                                            ← linearRampToValueAtTime
  _nextGainNode.gain  = skipNorm ? _volume : (safeNextGain × _volume) ← linearRampToValueAtTime
```

同一时刻 `cancelScheduledValues(now)` → `setValueAtTime` → `linearRampToValueAtTime` 确保从当前值平滑过渡，不跳变。

### 过渡结束后：bgAudio 接管

`_completeCrossfade(url)` 的核心逻辑：

1. 取消所有 `linearRampToValueAtTime` 调度
2. `bgAudio.src = url`（此 URL 已被 `_nextAudio` 缓存到浏览器，加载极快）
3. 监听 `canplay` → 同步 `currentTime` → `bgAudio.play()`
4. `_gainNode` 恢复到正常音量（`_applyNormVol()`）
5. `_nextGainNode.gain = 0` → `_nextAudio.pause()`
6. 250ms 重叠期（`_nextAudio` 延迟暂停）消除音频断档
7. 1.5 秒兜底超时：如果 `canplay` 迟迟不触发，强制切换避免 `_nextAudio` 一直播放

## 关键函数

| 函数 | 位置 | 职责 |
|------|------|------|
| `_initAudioCtx()` | initPlayer | 创建 AudioContext + 双管线（bgAudio / _nextAudio） |
| `_startCrossfade(bvid)` | player.js | 获取下一首 URL → 设置增益 ramp → 播放下一首 → 启动结束定时器 |
| `_completeCrossfade(url)` | player.js | 淡入淡出结束：bgAudio 接管播放 → 暂停 _nextAudio → 加载新歌歌词 |
| `_cancelCrossfade()` | player.js | 立即取消增益 ramp → 恢复 bgAudio 正常音量 → 停止 _nextAudio |
| `_fetchGainForNext(bvid)` | player.js | 异步查询下一首归一化增益（fire-and-forget，不阻塞过渡） |
| `_switchCurrentSongUI(song)` | player.js | 更新播放器 UI（歌名/封面/进度条/like/dislike 按钮），playCurrent 和 autoAdvance 共用 |

## 触发链路

```
bgAudio 播放结束 → 'ended' 事件
  └─ autoAdvance()
       ├─ sleepAfterSong? → stopPlayback（不淡入淡出）
       ├─ playMode === 'single'? → playCurrent（不淡入淡出）
       ├─ computeNextIndex()
       └─ _crossfadeDuration > 0 && nextSong.bvid?
            ├─ Yes: _switchCurrentSongUI(nextSong) → _startCrossfade(bvid)
            │    └─ _getPlayableUrl(bvid) → 设置 ramp → _nextAudio.play()
            │         └─ setTimeout(duration × 1000) → _completeCrossfade(url)
            └─ No:  playNext(true)（常规切歌）
```

### 手动切歌打断

```
用户点击下一首 → playNext(auto=false)
  └─ _cancelCrossfade()          ← 第一步：立即停止过渡
  └─ computeNextIndex() → playCurrent()
       └─ startVideo() → 直接设置 bgAudio.src（无过渡）
```

## 歌词方案 B 详解

### 为什么选方案 B

过渡期间两首歌同时在响——旧歌淡出、新歌淡入。如果切歌瞬间就切歌词（方案 A），用户看到的词和听到的人声会有 5 秒对不上。如果过渡期间隐藏歌词（方案 C），歌词面板空白 5 秒体验差。

方案 B：过渡期间歌词跟随旧歌同步衰减（歌词同步定时器仍在读旧歌的 `lyricsLines`），过渡结束自然切换到新歌。

### 实现

- `autoAdvance` 的淡入淡出分支**不调用** `loadLyrics(newSong)`，只更新 UI（`_switchCurrentSongUI`）
- `_completeCrossfade` 的最后一步调用 `import('./lyrics.js').then({ loadLyrics, setLyricsFor })` ，清空 `lyricsFor` 缓存强制重新加载
- 桌面歌词（PiP）依赖同一个 `lyricsLines` / `lyricsFor` 全局变量，自动跟随

## 与音量的交互

| 操作 | 过渡期间 | 过渡结束后 |
|------|---------|-----------|
| 调节音量滑块 | `_applyVol()` 不调用 `_applyNormVol()`（避免覆盖 ramp） | `_applyVol()` 正常调用 |
| 切换归一化开关 | 同上 | `_applyNormVol()` 正常 |
| 静音/取消静音 | `bgAudio.muted` 联动，但不调 gain | 正常 |

过渡期间 `_gainNode` 由 `linearRampToValueAtTime` 驱动，禁止任何 `gain.value =` 直接赋值。

## 设置

| localStorage key | 默认值 | 类型 | 说明 |
|-----------------|--------|------|------|
| `wemusic_crossfade_enabled` | `'0'` | `'0'` / `'1'` | Toggle 开关 |
| `wemusic_crossfade_duration` | `'5'` | `'3'` / `'5'` / `'8'` / `'10'` | 过渡时长（秒） |

UI 结构：Toggle switch + 4 个时长按钮（开关开启时显示，关闭时隐藏）。

`crossfade_changed` 自定义事件通知 player.js 实时读取新值。`_crossfadeDuration = 0` 时 `autoAdvance` 走常规 `playNext(true)` 路径。

## 边界情况与设计决策

| 边界情况 | 决策 | 理由 |
|---------|------|------|
| 下一首无 bvid（prefetch 未完成） | 退化为常规切歌 | `nextSong.bvid` 为 falsy 时直接 `playNext(true)` |
| 过渡期间用户开视频 | 不触发（`mode === 'bg'` 条件已阻断） | iframe 模式下音频由 B 站播放器独立输出，GainNode 无效 |
| 歌曲时长 < 过渡时长 | 过渡按设定时长进行（旧歌已播完时静音） | 用户配置优先；短歌极端罕见 |
| 下一首归一化 gain 尚未缓存 | `_nextNormGain` 默认为 1.0，淡入目标仍正确 | `_fetchGainForNext` 异步拉取，第二次播放生效 |
| 下一首获取 URL 失败 | `_crossfading = false`，状态恢复，不切歌 | 避免卡在过渡态 |
| 同一首歌重复播放（playQueue 只有一首） | `computeNextIndex()` 返回当前 index → 不走淡入淡出 | `i !== state.queueIndex` 条件已过滤 |
